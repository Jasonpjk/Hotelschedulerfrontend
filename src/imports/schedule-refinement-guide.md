현재 만든 근무표 메인 화면, 공정성 리포트, 운영 밸런스 리포트, 우측 버전 요약 패널, 범례, 날짜 헤더까지 아래 기준으로 다시 정리해줘.

중요:
- 로그인/대시보드와 동일한 프리미엄 호텔 운영 SaaS 톤앤매너는 유지
- 디자인 시스템, 컬러, 타이포그래피, 브랜드 무드는 유지
- 지금 시안은 한국어 기준으로 통일
- 기능 UI는 모두 한국어
- 레이아웃은 더 정돈되고 실무적으로 개선

핵심 로직 수정:
1. 공정성 기준은 “1~14일 / 15~28일”처럼 고정된 날짜 구간으로 자르지 말 것
2. 이전 달과 다음 달까지 연결된 “롤링 14일 기준”으로 판단하는 구조처럼 보여줄 것
3. 즉, 월 경계를 넘어도 14일 기준으로 연속적으로 판단하는 개념이 드러나야 함
4. 요일 기준 블록으로 나누는 방식은 사용하지 말 것
5. 휴무가 한 번에 3~4일 붙을 수 있어도, 전체적으로 연속 근무가 6일을 넘지 않도록 공정성에 반영할 것

공정성 리포트 수정:
현재 공정성 리포트를 아래 항목 중심으로 재구성해줘.

반드시 포함:
- 롤링 14일 기준 휴무 4회 충족 여부
- 최대 연속 근무일
- 6일 초과 연속 근무 위험 여부
- A13 후 다음날 M07 위험 조합 건수
- 주말 휴무 순환 공정성
- REQ 신청 휴일 반영률
- 야간 종료 후 2일 휴무 보장 여부

중요:
- “1~14일”, “15~28일”처럼 고정 구간 표는 제거
- 대신 직원별로 다음과 같은 구조가 좋음:
  - 최근 14일 기준 상태
  - 최대 연속 근무일
  - 신청 휴일 반영 여부
  - 주말 휴무 상태
  - 위험 조합 여부
  - 상태 배지(정상 / 주의 / 조정 필요)

추가 조건:
- 휴무가 3~4일 연속으로 붙더라도, 연속 근무가 6일 초과되지 않게 배분되는지를 공정성에서 보여줘
- “최대 연속 근무일”은 매우 중요한 항목으로 보여줘
- 관리자 입장에서 누가 규칙에 걸리는지 바로 보이게 해줘

버전 요약 패널 수정:
버전 요약에는 아래만 남기고 다시 정리해줘.

유지할 항목:
- 스케줄 전체 확정 상태
- 커버리지
- 신청 휴일 반영률
- 오후→오전 위험 건수
- 14일 휴무 규칙 상태
- 야간 전환 2일 휴무 위반 건수
- 조별 역할 균형 상태
- 중간조 보완 상태
- 최대 연속 근무일 경고 상태

삭제할 항목:
- 확정 완료 비율
- 미확정 셀
- 개인별 확정/미확정 느낌의 모든 정보

중요:
- 개인 셀 확정 여부는 필요 없음
- 스케줄 전체가 초안인지 확정인지 상태만 유지
- 버전 요약은 공정성 + 운영 밸런스 핵심 지표를 한눈에 보여주는 요약판처럼 바꿔줘

운영 밸런스 리포트 수정:
운영 밸런스는 지금보다 더 단순하고 실무적으로 보여줘.

핵심 기준:
- 오전조(M07), 오후조(A13), 야간조(N22)에 인차지 매니저 1명 이상 배치 여부
- 선임 / 담당 / 인턴 구성 균형
- 오전조와 오후조의 역할 구성 균형
- 중간조(C08/C09/C10/C11)가 어느 시간대 부족을 보완했는지
- 체크인 수가 많을 때는 늦은 중간조(C10/C11)가 보완
- 체크아웃 수가 많을 때는 이른 중간조(C08/C09)가 보완

표현 방식:
- 복잡한 능력치 숫자 나열보다
  - 조별 역할 구성 상태
  - 인차지 필수 충족 여부
  - 중간조 보완 성공/부족 여부
  - 피크일 대응 상태
  를 카드 + 짧은 요약 문장 + 상태 배지 중심으로 보여줘

근무표 화면 수정:
- 월간 근무표는 유지
- 주말/공휴일 색상 체계 유지
- 날짜 헤더 높이는 모든 날짜 셀에서 완전히 동일하게 맞춰줘
- 토/일/공휴일이라도 헤더 높이가 달라 보이지 않게
- 전체적으로 달력 헤더 라인이 정돈돼 보이게

범례 수정:
- 근무코드 범례와 휴무/비근무 범례에서 색이 들어간 사각형 칩의 크기를 전부 동일하게 맞춰줘
- M07, A13, N22, C08, C09, C10, C11, REQ, OFF, HOL, VAC, SL 모든 칩의 박스 크기/높이/여백을 통일
- 텍스트 길이가 달라도 칩 크기가 들쭉날쭉해 보이지 않게 정돈

운영 경고 패널 수정:
- 현재 운영 경고는 유지 가능
- 다만 경고 문구는 더 짧고 관리자용으로 정리
- 예:
  - 3월 22일 최소 커버리지 2명 부족
  - A13→다음날 M07 위험 조합 3건
  - 롤링 14일 기준 휴무 미충족 직원 1명
  - 최대 연속 근무일 7일 직원 발생
이런 식으로 간결하게

최종 목표:
- 공정성 리포트는 날짜 반으로 자른 관리가 아니라 월 경계를 넘는 롤링 14일 기준으로 보이게
- 연속 근무 6일 초과 위험이 명확히 드러나게
- 버전 요약은 공정성/운영 밸런스 핵심 요약판이 되게
- 범례 칩 크기와 날짜 헤더 높이는 완전히 통일
- 전체적으로 더 실무적이고 더 정돈된 호텔 운영 관리자 근무표 화면으로 개선해줘
