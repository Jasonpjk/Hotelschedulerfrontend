방금 완성한 로그인 페이지, 대시보드, 근무표 페이지와 동일한 디자인 시스템과 톤앤매너를 유지해서,
호텔 프런트 운영 스케줄 시스템의 “수요 예측” 페이지를 디자인해줘.

이 페이지는 단순 숫자 입력 화면이 아니라,
전달 월말에 다음 달 체크인/체크아웃 수요를 예측하고,
관리자가 그 근거를 검토하고,
필요시 이벤트/연휴/단체 예약을 반영해 수동 보정한 뒤,
최종 반영값을 확정해서 월간 스케줄 생성에 넘기는 핵심 관리자 워크스페이스다.

전체 공통 방향:
- 일반 소비자용 예약 사이트가 아니라 호텔 운영 관리자용 SaaS 화면
- 로그인/대시보드/근무표와 동일한 프리미엄 호텔 브랜드 무드 유지
- 롯데호텔 홈페이지가 주는 고급스럽고 정제된 분위기를 참고
- 딥 네이비, 차콜, 웜 화이트, 절제된 골드 포인트 유지
- 넓은 여백, 얇은 선, 세련된 타이포그래피, 정돈된 정보 구조
- 차트가 있더라도 과하지 않게
- 복잡한 데이터가 많아도 관리자 입장에서 이해하기 쉽게
- 데이터 과학 도구처럼 보이지 말고, 운영 판단용 관리자 화면처럼 보여야 함
- 데스크톱 우선 디자인
- 지금 시안은 한국어 기준으로 통일
- 브랜드명, 서비스명, 호텔명, 사용자명 같은 고유명사는 현재 표기 유지 가능
- 언어 선택 UI는 있어도 되지만, 현재 화면 자체는 한국어 기준으로 보여줘
- 향후 영어, 일본어, 러시아어, 베트남어로 확장 가능한 구조처럼 설계

정보 구조:
기존 “수요 입력” 페이지를 “수요 예측” 페이지로 확장한 구조처럼 보여줘.
새로운 독립 서비스가 아니라, 근무표 생성 전에 사용하는 같은 제품 내부의 예측 워크스페이스처럼 보여야 한다.

페이지 목적:
- 전달 말에 다음 달 체크인/체크아웃 수를 예측
- 예측 근거를 확인
- 공휴일/연휴/이벤트/단체 예약 반영
- 관리자가 수동 보정
- 최종 반영값을 확정
- 그 결과가 근무표 생성으로 이어지는 구조를 보여줘

예측 로직이 화면에서 자연스럽게 드러나야 하는 핵심 규칙:
- 전년 동일 “일자”가 아니라 전년 동일 “요일”과 유사 주차 기준으로 비교
- 올해 현재 예약 수를 기준으로 시작
- 전년 동일 요일의 생성 시점 예약 수와 실제 체크인/체크아웃 수를 비교
- 작년에 늘어난 만큼 올해도 늘어난다고 보는 픽업 증가분 로직이 반영되어야 함
- 체크인과 체크아웃 모두 각각 예측
- 주말과 평일 패턴 차이를 반영해야 함
- 추석, 설, 대체공휴일, 연휴 길이, 주말 포함 여부 같은 공휴일 구조가 반영되어야 함
- 공휴일이 작년과 올해 달/길이/주말 포함 여부가 달라져도 반영되는 구조처럼 보여야 함
- 과거 데이터가 없는 대형 이벤트도 수동 보정할 수 있어야 함
- 예: BTS 무료 공연, 대형 컨벤션, 국제행사, 스포츠 이벤트, 단체 예약 등

페이지 전체 레이아웃:
1. 상단 헤더
- 브랜드/서비스명
- 현재 페이지 타이틀: 수요 예측
- 호텔 선택 드롭다운
- 예측 대상 월 선택
- 현재 버전 또는 예측 기준 버전 표시
- 언어 선택 UI, 알림, 사용자 프로필
- 로그인/대시보드/근무표 페이지와 동일한 스타일 유지

2. 좌측 사이드바
- 대시보드
- 근무표
- 직원 관리
- 휴무 관리
- 수요 예측
- 호텔 설정
- 현재 “수요 예측” 메뉴가 선택 상태로 보이게
- 기존 페이지와 동일한 네비게이션 디자인 유지

3. 상단 요약 카드 영역
반드시 포함:
- 예상 총 체크인
- 예상 총 체크아웃
- 피크 체크인일
- 피크 체크아웃일
- 연휴 영향일 수
- 이벤트 반영 건수
- 예측 신뢰도 또는 검토 필요 건수

카드 디자인 조건:
- 숫자는 명확히 보이게
- 상태 배지 또는 보조 문구가 함께 보이게
- 너무 화려한 KPI 대시보드처럼 말고, 운영 판단용 요약 카드처럼 정돈되게

4. 핵심 탭 구조
반드시 탭으로 구성:
- 예측 결과
- 예측 근거
- 이벤트 보정
- 최종 반영값

각 탭의 역할이 분명해야 함.

탭 1: 예측 결과
목표:
- 날짜별 예상 체크인 수 / 체크아웃 수를 한눈에 확인
- 피크일과 리스크일을 파악
- 근무표 인력 배치에 영향을 주는 날짜를 바로 알 수 있게

반드시 포함:
- 날짜별 테이블 또는 달력형 리스트
- 날짜
- 요일
- 예상 체크인 수
- 예상 체크아웃 수
- 피크일 강조
- 과소 배치 위험 또는 과다 배치 위험 요약
- 주말과 공휴일 강조
- 월 전체를 훑기 쉬운 구조

디자인 조건:
- 숫자가 많아도 읽기 쉽게
- 스프레드시트 느낌은 있되 훨씬 더 정제되고 세련되게
- 피크일은 과장 없이 명확하게 강조

탭 2: 예측 근거
목표:
- 이 수치가 왜 나왔는지 관리자에게 설명
- 블랙박스처럼 보이지 않고, 예측 논리를 검토할 수 있게

반드시 포함:
- 올해 현재 예약 수
- 전년 동일 요일 기준 예약 수
- 전년 동일 요일 실제 체크인 수
- 전년 동일 요일 실제 체크아웃 수
- 전년 픽업 증가분
- 올해 최종 예측값
- 주말/평일 구분
- 유사 주차 기준 비교
- 참고한 과거 패턴 요약

표현 방식:
- 날짜별 표 또는 카드
- “현재 예약 + 전년 픽업 증가분 = 예상 체크인” 같은 계산 흐름이 직관적으로 보이게
- 기술적으로 복잡한 수식 대신 운영자가 이해할 수 있는 구조
- 필요하면 작은 비교 그래프나 비교 막대 허용
- 하지만 차트 남발 금지

탭 3: 이벤트 보정
목표:
- 과거 데이터가 없는 이벤트나 특수 상황을 수동으로 반영
- 공휴일 구조 변화와 대형 이벤트를 예측에 반영

반드시 포함:
- 이벤트 목록 카드 또는 테이블
- 이벤트명
- 날짜
- 영향 기간
- 예상 수요 영향 방향
- 예상 증가/감소 수준
- 체크인 중심인지 체크아웃 중심인지
- 신뢰도 또는 참고 수준
- 수동 조정값 입력
- 메모/사유 입력

반드시 표현되어야 하는 이벤트 유형:
- 추석 / 설 / 대체공휴일 / 샌드위치 연휴
- 주말 포함 연휴
- 대형 공연
- 컨퍼런스
- 단체 예약
- 지역 축제
- 과거 데이터가 없는 신규 이벤트

중요:
- 추석 같은 명절은 단순 공휴일 1개가 아니라 “연휴 구조”로 반영되는 느낌이 나야 함
- 예: 작년은 3일 연휴, 올해는 주말 포함 5일 연휴 같은 차이를 반영할 수 있어야 함
- 과거 데이터 없는 이벤트도 “수동 uplift”처럼 반영 가능해야 함

탭 4: 최종 반영값
목표:
- 스케줄 생성에 실제로 사용되는 확정 수요값을 보여줌
- 관리자 승인 후 근무표 생성으로 넘어가는 구조

반드시 포함:
- 날짜별 최종 체크인 반영값
- 날짜별 최종 체크아웃 반영값
- 예측값 대비 수동 조정 여부
- 최종 반영 사유 요약
- “근무표 생성에 반영” 버튼 또는 확정 액션
- 이 값이 월간 스케줄 생성에 전달되는 흐름이 느껴지게

중요 운영 원칙 표현:
- 스케줄은 월말에 한 번 생성하고 직원에게 공지되므로, 자주 바꾸지 않는 구조
- 예측 품질을 높여 최초 배치 안정성을 높이는 방향
- M/A/N 기본 배치는 안정적으로 가고, 변동 흡수는 C조 시작시간 조정으로 대응하는 구조가 자연스럽게 보이게
- 예측 페이지가 근무표 페이지의 입력값을 만드는 역할을 한다는 점이 드러나게

근무표와의 연결 요약 카드 또는 보조 패널:
- 예상 체크인 피크일
- 예상 체크아웃 피크일
- 중간조 보완 필요일
- 최소 커버리지 위험일
- 과소 배치 위험
- 과다 배치 위험
- 관리자에게 “이 예측이 근무표에 어떤 영향을 주는지” 바로 읽히게

수동 보정 UX:
- 이벤트 카드 수정
- 관리자 조정값 입력
- 조정 사유 입력
- 적용 전/후 비교
- 보정 이력 요약
- 너무 기술적인 관리자 콘솔처럼 보이지 않게
- 프리미엄 운영툴 안에 자연스럽게 녹아든 구조로 설계

한국어 UI 기준 예시:
- 수요 예측
- 예측 결과
- 예측 근거
- 이벤트 보정
- 최종 반영값
- 예상 체크인
- 예상 체크아웃
- 피크일
- 연휴 영향
- 이벤트 반영
- 현재 예약 수
- 전년 예약 수
- 전년 실제 체크인
- 전년 실제 체크아웃
- 픽업 증가분
- 수동 보정
- 최종 반영
- 근무표 생성 반영
- 검토 필요
- 신뢰도
- 조정 사유
- 중간조 보완 필요

타이포그래피:
- 로그인/대시보드/근무표와 같은 프리미엄 호텔 브랜드 무드 유지
- 정확히 특정 폰트를 복제하려고 하지 말고, 유사한 고급스럽고 정제된 타이포그래피 분위기 유지
- 제목은 절제된 품격
- 숫자와 날짜, 보조 정보는 가독성 우선
- 표와 카드 모두 읽기 편해야 함
- 향후 다국어 전환에도 구조가 무너지지 않게

금지사항:
- 소비자용 예약 사이트처럼 보이게 하지 말 것
- 여행 감성 이미지, 객실 사진, 프로모션 배너 금지
- 데이터 과학 툴처럼 지나치게 기술적인 느낌 금지
- 차트만 가득한 화면 금지
- 단순 숫자 입력 폼으로만 보이게 하지 말 것
- 공휴일/이벤트 차이를 무시하는 단순 전년 비교처럼 보이게 하지 말 것
- 근무표 페이지의 부속 화면처럼 빈약하게 보이지 말 것

최종 목표:
- 기존 수요 입력 페이지가 “수요 예측 워크스페이스”로 승격된 느낌
- 전년 동일 요일 비교, 연휴 구조 반영, 이벤트 보정, 최종 반영값 확정까지 한 흐름으로 보이는 관리자용 예측 화면
- 체크인/체크아웃 예측이 실제 근무표 인원 배치와 연결되는 실무형 호텔 운영 SaaS 페이지처럼 보여주는 디자인
