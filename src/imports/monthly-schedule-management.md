방금 완성한 로그인 페이지와 대시보드 페이지와 동일한 디자인 시스템, 톤앤매너, 브랜드 무드를 유지해서,
호텔 프런트 운영 스케줄 시스템의 핵심 메인 페이지인 “월별 근무표 관리 화면”을 디자인해줘.

이 화면은 서비스에서 가장 중요한 핵심 작업 화면이다.
호텔 관리자와 스케줄 담당자가 실제로 매일 확인하고 생성하고 수정하고 확정하는 업무용 메인 페이지처럼 보여야 한다.

전체 공통 방향:
- 일반 소비자용 예약 사이트가 아니라 호텔 운영 관리자용 SaaS 화면
- 로그인 페이지와 대시보드의 프리미엄 호텔 브랜드 무드 유지
- 롯데호텔 홈페이지가 주는 고급스럽고 정제된 분위기 참고
- 딥 네이비, 차콜, 웜 화이트, 절제된 골드 포인트 유지
- 넓은 여백, 얇은 선, 세련된 타이포그래피, 정돈된 정보 구조
- 엑셀보다 훨씬 세련되고 실무 효율이 높은 관리자 화면
- 데스크톱 우선 디자인
- 표가 복잡해도 지저분하지 않게
- 소비자용 예약/여행 사이트처럼 보이지 않게
- 개발자용 백오피스처럼 차갑고 딱딱하게 보이지 않게
- 실제 호텔 운영 관리자 툴처럼 고급스럽고 실무적으로 설계

현재 시안의 언어 규칙:
- 지금 디자인 시안은 한국어 기준으로 통일
- 화면의 모든 기능 UI 텍스트는 100% 한국어
- 브랜드명, 서비스명, 호텔명, 사용자명 같은 고유명사는 현재 표기 유지 가능
- 언어 선택 UI는 있어도 되지만 지금 시안은 한국어 기준 화면
- 향후 영어, 일본어, 러시아어, 베트남어로 자연스럽게 확장 가능한 구조처럼 설계
- 각 언어로 바뀌어도 버튼 폭, 탭 이름, 상태 배지, 카드 제목이 버틸 수 있게 충분한 여백 확보

프로젝트 맥락:
- 이 서비스는 호텔 프런트 직원 스케줄을 생성, 조회, 수정, 확정하는 운영 시스템
- 로그인 후 대시보드에서 이 페이지로 들어옴
- 이 페이지에서 월 선택, 버전 확인, 스케줄 생성, 수동 수정, AI 조정, 확정, 다운로드까지 핵심 업무가 이뤄짐
- 이 화면은 서비스의 중심 업무 화면

근무조 표시 규칙(중요):
- 오전조는 Morning의 M으로 표기하고 07:00 시작
- 오후조는 Afternoon의 A로 표기하고 13:00 시작
- 야간조는 Night의 N으로 표기하고 22:00 시작
- 중간조는 Morning과 충돌하지 않도록 MD 코드로 표기
- 중간조 시작 시간은 08:00, 09:00, 10:00, 11:00가 각각 따로 존재
- 권장 표기:
  - M07 = 오전조 07:00 시작
  - A13 = 오후조 13:00 시작
  - N22 = 야간조 22:00 시작
  - MD08 = 중간조 08:00 시작
  - MD09 = 중간조 09:00 시작
  - MD10 = 중간조 10:00 시작
  - MD11 = 중간조 11:00 시작
- 색만으로 구분하지 말고 코드와 시간 기반 표기를 함께 사용
- 야간 전담, 야간 블록, 재진입 없음 같은 규칙은 화면에서 강조하지 말 것
- 그런 규칙은 실제 코딩 로직에서 처리할 예정이므로 디자인 시안에서는 일반 근무 결과 표기 중심으로 보여줘

휴무/비근무 표기 규칙(매우 중요):
쉬는 유형은 서로 구분되어 보여야 한다.
반드시 서로 다른 코드와 시각적 구분을 표현해줘.

권장 표기:
- REQ = 본인 신청 휴일
- OFF = 일반 휴무
- HOL = 공휴일
- VAC = 휴가
- SL = 여성보건휴가

조건:
- 각 비근무 유형은 코드가 다르게 보이게
- 색상 또는 배경 스타일도 적절히 구분
- 하지만 너무 복잡하거나 난잡하지 않게
- 관리자와 직원이 한눈에 구분 가능해야 함

날짜/캘린더 표현 규칙:
- 토요일과 일요일은 날짜 색을 평일과 다르게 보여줘
- 공휴일도 별도 색상 또는 강조 표시로 보이게
- 주말과 공휴일이 표 상단 날짜 행에서 한눈에 구분되어야 함
- 오늘 날짜 강조가 있으면 좋음

9. 운영 밸런스 리포트 탭(핵심 추가)
이 서비스의 중요한 특징으로 다음 내용을 보여줘야 한다.

반드시 포함:

- 오전조(M07), 오후조(A13), 야간조(N22)의 각 근무능력 합계가 균형을 이루는지 확인하는 리포트
- 어느 조가 상대적으로 강하거나 약한지 요약
- 중간조(MD08/MD09/MD10/MD11)가 부족한 시간대를 얼마나 잘 보완했는지 리포트
- 특정 날짜 또는 시간대에 중간조 투입으로 커버가 개선된 내용 표현
- “중간조가 필요한 시간대를 잘 채웠는지”가 한눈에 보이게

표현 방식 예시:

- 밸런스 상태 배지(균형 / 주의 / 부족)
- 시간대 커버리지 요약
- 중간조 보완 기여도 요약
- 특정 날짜 경고 및 보완 성공 사례

중요:
- 이 리포트는 단순 통계가 아니라 운영 판단용 화면처럼 보여야 함
- 관리자 입장에서 “어느 조가 약한지 / 중간조가 잘 메웠는지”를 빠르게 읽을 수 있어야 함
