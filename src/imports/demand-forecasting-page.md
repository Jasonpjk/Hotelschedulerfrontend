현재 서비스의 정보 구조를 아래 기준으로 수정해줘.

중요 판단:
- 스케줄은 전달 말에 월 단위로 작성해서 직원들에게 공지되므로, 중간에 자주 바꾸지 않는 운영 원칙을 반영해야 해
- 따라서 체크인/체크아웃 숫자는 스케줄 생성 전에 예측해서 확정하는 흐름이 필요해
- 이 기능은 근무표 화면 안에 넣지 말고, 기존 “수요 입력” 페이지를 확장한 “수요 예측” 페이지로 재구성해줘
- 즉, 별도의 완전히 새로운 독립 메뉴를 추가하기보다 기존 수요 입력 메뉴를 “수요 예측” 중심 페이지로 바꿔줘

메뉴 구조 수정:
- 대시보드
- 근무표
- 직원 관리
- 휴무 관리
- 수요 예측
- 호텔 설정

수요 예측 페이지 목적:
- 전달 월말에 다음 달의 체크인/체크아웃 수를 예측
- 그 예측값을 기준으로 월간 근무표 생성
- 이후 스케줄은 자주 변경하지 않고, 최초 예측 품질을 높이는 구조

예측 로직이 화면에서 자연스럽게 드러나게 해줘:
- 전년 동일 “일자”가 아니라 전년 동일 “요일”과 유사 주차 기준으로 비교
- 전년 동일 요일의 생성 시점 예약 수와 실제 체크인 수를 비교
- 작년에 늘어난 만큼 올해도 늘어난다고 가정하는 픽업 방식
- 올해 현재 예약 수 + 작년 동일 요일 기준 픽업 증가분 = 올해 예상 체크인 수
- 체크아웃 숫자도 같은 방식으로 예측
- 주말은 수요가 크게 늘어나므로 요일 중심 비교가 중요하다는 점이 드러나게
- 공휴일/이벤트/단체 예약 같은 예외 요인도 수동 보정 가능하게

수요 예측 페이지 내부 구조:
탭 구조로 설계해줘.

1. 예측 결과
- 날짜별 예상 체크인 수
- 날짜별 예상 체크아웃 수
- 피크일 강조
- 인력 과소/과다 배치 위험 요약

2. 예측 근거
- 올해 현재 예약 수
- 전년 동일 요일 기준 예약 수
- 전년 동일 요일 실제 체크인/체크아웃 수
- 전년 픽업 증가분
- 최종 예측값 산출 근거
- 예측 신뢰도 또는 참고 수준 표시 가능

3. 수동 보정
- 이벤트 반영
- 단체 예약 반영
- 관리자 조정값 입력
- 조정 사유 기록

4. 최종 반영값
- 실제 스케줄 생성에 쓰일 최종 체크인/체크아웃 예측값
- “근무표 생성 반영” 버튼 또는 확정 흐름이 느껴지게

디자인 방향:
- 로그인/대시보드/근무표와 같은 프리미엄 호텔 운영 SaaS 톤앤매너 유지
- 데이터가 많아도 깔끔하고 신뢰감 있게
- 예측 결과를 보는 화면이지 복잡한 데이터 과학 도구처럼 보이지 않게
- 운영 관리자도 이해할 수 있는 구조
- 차트가 있더라도 과하지 않게
- 카드, 표, 탭, 요약 패널 중심

근무표 페이지와의 연결도 반영:
- 근무표 페이지에서는 예측 결과를 입력값으로 소비하는 구조처럼 보여줘
- 예를 들어:
  - 피크 체크인일
  - 피크 체크아웃일
  - 예상 수요 기준 적정 인원 충족 여부
  - 과소 배치 위험
  - 과다 배치 위험
  이런 정보가 요약으로 연결되게

운영 원칙도 시각적으로 반영:
- 스케줄은 월 단위 공지 후 자주 바꾸지 않음
- 예측 품질을 높여서 최초 배치의 안정성을 높이는 구조
- 중간조가 변동 흡수 역할을 하는 방향이 자연스럽게 연결되게

최종 목표:
- 기존 “수요 입력” 페이지를 “수요 예측” 중심 페이지로 재구성
- 예측 → 검토 → 수동 보정 → 최종 반영 → 근무표 생성 흐름이 한눈에 보이게
- 체크인/체크아웃 예측이 실제 스케줄 인원 배정과 연결되는 실무형 관리자 화면처럼 보이게
