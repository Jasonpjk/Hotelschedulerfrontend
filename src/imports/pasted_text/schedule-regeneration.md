현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS의 “수요 재배치” 화면을
“스케줄 재생성” 화면으로 단순화해줘.

이번 작업의 목적은
수요예측 변경이나 기타 운영 사유로 근무표를 다시 짤 때
복잡한 탭 구조 없이 핵심 정보만 입력하고
스케줄 재생성 버튼을 누르면 실제 월별 근무표와 연간 야간조 계획에 반영되도록 만드는 것이야.

--------------------------------------------------
[현재 문제]
--------------------------------------------------

현재 “수요 재배치” 화면은 아래처럼 기능이 너무 많이 나뉘어 있음.

- 수요예측 재업로드
- 영향 분석 & 재배치
- 잠금 관리
- 야간조 재설정

이 구조는 실제 스케줄러가 사용하기에 너무 복잡함.

또한 첫 번째 화면의 우측 하단 “재배치 설정으로 이동” 버튼이 작동하지 않음.
이 버튼을 눌렀을 때 다음 설정 영역으로 자연스럽게 이동하거나 펼쳐져야 하는데 현재 dead-end처럼 보임.

수정 방향:
- “수요 재배치”를 “스케줄 재생성”으로 변경
- 여러 탭을 없애거나 최소화
- 한 화면에서 위에서 아래로 진행하는 단순한 흐름으로 변경
- 스케줄 재생성 결과가 실제 월별 근무표와 연간 야간조 계획에 반영되게 수정

--------------------------------------------------
[화면명 변경]
--------------------------------------------------

좌측 메뉴명과 화면 제목을 아래처럼 바꿔줘.

기존:
- 수요 재배치

변경:
- 스케줄 재생성

설명 문구 예:
“수요예측 변경이나 운영 사유 발생 시, 변경 기준일 이후의 일정만 다시 생성하고 새 버전으로 저장합니다.”

--------------------------------------------------
[전체 화면 구조]
--------------------------------------------------

기존의 복잡한 탭 구조를 없애고 아래 흐름으로 단순화해줘.

1. 재생성 기준
2. 변경 기준일
3. 영향 기간
4. 보호 일정
5. 야간조 조정
6. 스케줄 재생성
7. 결과 미리보기
8. 새 버전 저장

각 영역은 카드형 섹션으로 구성하고,
사용자가 위에서 아래로 자연스럽게 진행할 수 있게 해줘.

--------------------------------------------------
[1. 재생성 기준]
--------------------------------------------------

재생성 기준은 복잡하게 여러 사유를 나누지 말고 아래 2개만 제공해줘.

- 수요예측 변경
- 기타 사유

수요예측 변경을 선택한 경우:
- 최신 수요예측 업로드
- 기존 수요예측과 비교
- 영향 날짜 자동 분석
- 체크인 / 체크아웃 변화 표시
- 중간조 추가 필요 여부 표시

기타 사유를 선택한 경우:
- 사유 직접 입력
- 영향 기간 입력

기타 사유 예:
- 시설 점검
- 단체 행사
- VIP 행사
- 갑작스러운 객실 판매 증가
- 장기 부재
- 퇴사
- 교육 / 병가 반영
- 기타 운영 이슈

중요:
- 기타 사유를 다시 여러 유형으로 쪼개지 말 것
- “시설 점검”, “행사”, “장기 부재”, “퇴사” 등을 별도 입력 폼으로 나누지 말 것
- 기타 사유는 텍스트 입력 + 영향 기간 입력만으로 충분함

--------------------------------------------------
[2. 변경 기준일]
--------------------------------------------------

변경 기준일은 단순하게 하나만 둬.

의미:
- 이 날짜 이전 스케줄은 유지
- 이 날짜부터 이후 일정만 재생성

문구 예:
“변경 기준일 이전 일정은 유지하고, 이후 일정만 다시 생성합니다.”

중요:
- 다음 달 스케줄을 만드는 상황이므로 “오늘 이전 날짜 잠금” 같은 복잡한 설명은 넣지 말 것
- 변경 기준일이 핵심 기준임
- 변경 기준일 이전 일정은 자동 재생성 대상에서 제외

--------------------------------------------------
[3. 영향 기간]
--------------------------------------------------

재생성할 영향 기간을 선택할 수 있게 해줘.

수요예측 변경 선택 시:
- 업로드한 최신 수요예측을 기준으로 영향 날짜를 자동 제안
- 사용자가 필요하면 영향 기간 수정 가능

기타 사유 선택 시:
- 사용자가 영향 시작일 / 종료일을 직접 입력

예:
- 영향 기간: 2026-05-12 ~ 2026-05-14

--------------------------------------------------
[4. 보호 일정]
--------------------------------------------------

보호 일정은 별도 “잠금 관리” 탭으로 분리하지 말고,
스케줄 재생성 화면 안의 하나의 카드로 제공해줘.

보호 일정은 아래 2단계로 구분해줘.

1) 절대 보호
2) 우선 보호

--------------------------------------------------
[절대 보호]
--------------------------------------------------

절대 보호는 스케줄 재생성 시 절대 변경하면 안 되는 일정임.

절대 보호 대상:
- 승인된 교육
- 승인된 병가
- 직원이 신청했고 승인 안내까지 나간 휴일
- 직원에게 승인 안내가 나간 연차
- 직원에게 승인 안내가 나간 SL
- 직원에게 승인 안내가 나간 HO
- 신혼여행
- 장기 병가
- 장기 교육
- 퇴사 이후 기간
- 스케줄러가 직접 절대 보호로 지정한 일정

중요:
- 직원에게 이미 승인 안내가 나간 일정은 직원 신뢰와 직결되므로 절대 보호로 처리
- 자동 재생성 시 절대 보호 일정은 변경 금지
- 절대 보호 일정은 재생성 후보에서 제외

--------------------------------------------------
[우선 보호]
--------------------------------------------------

우선 보호는 가능하면 유지해야 하지만,
운영상 꼭 필요하면 관리자 확인 후 변경할 수 있는 일정임.

우선 보호 대상:
- SL
- 연차
- HO
- 승인 전 또는 안내 전인 휴무 후보
- 아직 직원에게 공지되지 않은 OFF
- 스케줄러가 우선 보호로 지정한 일정

정리:
- SL / 연차 / HO 자체는 기본적으로 우선 보호
- 단, 직원에게 승인 안내가 이미 나간 경우는 절대 보호로 승격

--------------------------------------------------
[근무표에서 보호 일정 선택]
--------------------------------------------------

추가 보호 일정은 별도 캘린더 입력이나 수기 입력으로 하나씩 추가하지 말고,
기존 월별 근무표를 보면서 직접 선택할 수 있게 해줘.

버튼:
- 근무표에서 보호 일정 선택

동작:
1) 버튼 클릭
2) 기존 월별 근무표를 선택 모드로 열기
   - 같은 화면 안의 모달 또는 전체화면 선택 모드 권장
   - 완전히 다른 페이지로 이동했다가 돌아오는 구조는 상태가 꼬일 수 있으므로 지양
3) 스케줄러가 보호할 셀을 여러 개 클릭해서 다중 선택
4) 선택된 셀 개수 표시
5) 보호 수준 선택
   - 절대 보호
   - 우선 보호
6) “보호 일정으로 저장” 클릭
7) 선택한 보호 일정이 스케줄 재생성 화면의 보호 일정 카드에 반영

중요:
- 여러 사람의 일정을 하나씩 날짜 입력해서 보호하는 방식은 불편하므로 사용하지 말 것
- 근무표를 보면서 셀을 직접 다중 선택하는 방식이 핵심
- 선택한 보호 일정은 스케줄 재생성 시 반드시 반영
- 기존 근무표 컴포넌트를 재사용하되 “보호 일정 선택 모드”로 동작하게 해줘

--------------------------------------------------
[5. 야간조 조정]
--------------------------------------------------

야간조 변경은 특정 사건이 있을 때만 나타나는 기능이 아니라,
스케줄러가 필요하다고 판단하면 언제든 사용할 수 있어야 함.

따라서 스케줄 재생성 화면에 아래 옵션을 항상 제공해줘.

옵션:
[ ] 야간조 계획도 함께 조정

체크하지 않으면:
- 월별 근무표만 재생성
- 연간 야간조 계획은 변경하지 않음

체크하면:
- 월별 근무표 재생성 결과와 함께
- 해당 월의 연간 야간조 계획도 같이 조정

야간조 조정 방식:
- 자동 재설정
- 직접 우선 대상자 선택
- 직접 제외 대상자 선택

기본값:
- 자동 재설정

중요:
- “야간조 변경 필요” 카드가 특정 사건이 있을 때만 뜨면 안 됨
- 스케줄러가 원할 때 직접 체크해서 실행할 수 있어야 함
- 대체 후보 목록을 별도로 보여줄 필요 없음
- 야간조 대체 후보는 전체 직원 중 운영정책에 맞는 사람을 시스템이 찾으면 됨

--------------------------------------------------
[연간 야간조 계획 단순화]
--------------------------------------------------

근무표의 “연간 야간조 계획” 화면과 월별 야간조 계획 팝업에서
예비 대상자 개념은 제거해줘.

기존:
- 우선 대상자
- 예비 대상자
- 제외 대상자

변경:
- 우선 대상자
- 제외 대상자

의미:
- 우선 대상자: 해당 월 야간조에 실제 투입하고 싶은 직원
- 제외 대상자: 장기 병가, 신혼여행, 퇴사, 운영상 제외가 필요한 직원

중요:
- 예비 대상자를 별도로 관리하지 말 것
- 대체가 필요하면 시스템이 전체 직원 중 조건에 맞는 사람을 자동으로 찾도록 처리
- 기존 연간 야간조 계획 기능 자체는 유지하되 구조만 단순화

--------------------------------------------------
[야간조 운영 원칙]
--------------------------------------------------

야간조는 해당 월에 오전조 / 오후조 / 중간조와 섞이지 않는 것을 원칙으로 해줘.

원칙:
- 야간조 대상자는 해당 월에 N22 중심으로 배치
- M07 오전조 배정 금지
- A13 오후조 배정 금지
- C08 / C09 / C10 / C11 중간조 배정 금지
- OFF / 휴식일은 가능

야간조 계획을 함께 조정하는 경우:
- 변경 기준일 이후 구간만 조정
- 절대 보호 일정은 변경하지 않음
- 우선 보호 일정은 가능하면 유지
- 변경 결과는 연간 야간조 계획 해당 월에도 반영

--------------------------------------------------
[6. 스케줄 재생성]
--------------------------------------------------

“스케줄 재생성” 버튼을 누르면 실제로 스케줄이 다시 만들어져야 함.

버튼:
- 스케줄 재생성

동작:
1) 재생성 기준 확인
2) 변경 기준일 확인
3) 영향 기간 확인
4) 절대 보호 / 우선 보호 일정 반영
5) 야간조 계획 조정 여부 확인
6) 운영정책 기준으로 영향 구간만 재생성
7) 결과 미리보기 생성

운영정책 반영:
- 최소 조별 인원
- 각 조별 인차지 1명 이상
- A13 다음날 M07 지양
- 5일 / 6일 이상 연속근무 경고
- 전월 근무강도 회복 반영
- 야간조 전환 OFF 정책
- 수요예측 피크 반영
- 중간조 C08 / C09 / C10 / C11 우선 활용
- 보호 일정 유지

중요:
- 스케줄 재생성 버튼이 단순 미리보기만 만들고 끝나면 안 됨
- 이후 새 버전 저장 시 실제 월별 근무표에 반영되어야 함
- 야간조 계획도 함께 조정 옵션이 켜져 있으면 연간 야간조 계획에도 반영되어야 함

--------------------------------------------------
[7. 결과 미리보기]
--------------------------------------------------

스케줄 재생성 후 결과 미리보기를 보여줘.

결과 미리보기에 포함할 항목:
- 월별 근무표 변경 건수
- 변경된 직원 / 날짜 / 기존 코드 / 변경 코드
- 중간조 추가 또는 조정 내역
- 절대 보호 유지 건수
- 우선 보호 유지 건수
- 우선 보호 변경 필요 건수
- 야간조 계획 변경 여부
- 연간 야간조 계획 변경 내역
- 남은 운영 경고
- 새 버전명

예:
- 월별 근무표 변경: 14건
- 중간조 추가: C09 2건, C10 3건
- 보호 일정 유지: 9건
- 야간조 계획 변경: 있음
- 5월 야간조 계획: 박지호 제외, 이서연 우선 대상자 추가
- 남은 경고: 1건

--------------------------------------------------
[8. 새 버전 저장]
--------------------------------------------------

스케줄 재생성 결과는 기존 버전을 덮어쓰지 말고 새 버전으로 저장해줘.

버튼:
- 새 버전으로 저장

저장 시 실제 반영 대상:
1) 월별 근무표
2) 연간 야간조 계획
3) 변경 이력
4) 대시보드 상태
5) 운영 경고
6) 공정성 리포트

새 버전에는 아래 정보를 기록:
- 재생성 사유
- 변경 기준일
- 영향 기간
- 수요예측 버전
- 보호 일정 유지 여부
- 월별 근무표 변경 내역
- 야간조 계획 변경 내역
- 남은 경고
- 생성자
- 생성 시각

예:
v4.1 스케줄 재생성

생성 사유: 5월 12일 시설 점검 및 야간조 조정
변경 기준일: 2026-05-12
영향 기간: 2026-05-12 ~ 2026-05-14
월별 근무표 변경: 14건

--------------------------------------------------
[재배치 설정으로 이동 버튼 수정]
--------------------------------------------------

현재 첫 번째 화면의 우측 하단 “재배치 설정으로 이동” 버튼이 작동하지 않음.

수정:
- 버튼명을 “스케줄 재생성 설정”으로 변경
- 클릭 시 같은 화면 아래의 재생성 설정 영역으로 스크롤
또는
- 재생성 설정 카드가 바로 펼쳐지게 처리

중요:
- 클릭해도 아무 일도 안 일어나는 dead-end 버튼 금지
- 별도 탭 이동보다 같은 화면 안에서 다음 단계가 열리는 방식 권장

--------------------------------------------------
[수요예측 변경 선택 시]
--------------------------------------------------

재생성 기준에서 “수요예측 변경”을 선택한 경우에만 수요예측 업로드 영역을 보여줘.

표시 항목:
- 최신 수요예측 업로드
- 기존 Forecast 버전
- 신규 Forecast 버전
- 체크인 변화
- 체크아웃 변화
- 영향 날짜
- 중간조 추가 필요 여부

수요예측 변경이 아닌 기타 사유를 선택한 경우에는
수요예측 업로드를 강제하지 말 것.

- 기타 사유는 텍스트 입력으로 충분함

--------------------------------------------------
[데이터 연결]
--------------------------------------------------

스케줄 재생성 결과는 다음 화면들과 실제로 연결되어야 함.

1) 월별 근무표
- 변경된 근무 코드가 실제 셀에 반영

2) 연간 야간조 계획
- 야간조 조정 옵션이 켜져 있으면 해당 월 계획에 반영
- 우선 대상자 / 제외 대상자 구조로 반영
- 예비 대상자는 사용하지 않음

3) 대시보드
- 근무표 버전
- 운영 경고
- 공정성 리포트
- 마지막 반영 시각 갱신

4) 변경 이력
- 스케줄 재생성으로 변경된 내역 기록

5) 버전 관리
- 새 버전으로 저장
- 기존 버전은 덮어쓰지 않음
