현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS의 “스케줄 재생성” 화면이 만들어졌지만,
화면이 빈 아코디언 목록처럼 보여서 사용자가 무엇을 해야 하는지 알기 어렵습니다.

이번 작업은 기능을 새로 추가하는 것이 아니라
현재 만들어진 “스케줄 재생성” 화면의 UX와 레이아웃을 실제 사용할 수 있는 마법사형 흐름으로 정리하는 작업입니다.

--------------------------------------------------
[현재 문제]
--------------------------------------------------

현재 화면에는 아래 단계가 카드처럼 나열되어 있습니다.

1. 재생성 기준
2. 변경 기준일
3. 영향 기간
4. 보호 일정
5. 야간조 조정
6. 스케줄 재생성 실행

하지만 각 단계가 모두 접힌 상태이거나 내용이 거의 보이지 않아
첫 화면이 빈 목록처럼 보입니다.

문제점:
- 첫 번째 단계 내용이 기본으로 열려 있지 않음
- 사용자가 무엇부터 입력해야 하는지 알 수 없음
- 각 단계의 현재 입력 상태 요약이 없음
- 핵심 CTA 버튼이 첫 화면에서 보이지 않음
- 화면이 너무 비어 있고 여백이 커서 미완성처럼 보임
- 스케줄 재생성 결과가 실제 월별 근무표와 연간 야간조 계획에 반영되는 흐름이 보이지 않음

--------------------------------------------------
[수정 목표]
--------------------------------------------------

“스케줄 재생성” 화면을 단순한 접힘 목록이 아니라
실제로 사용할 수 있는 단계형 마법사 화면으로 정리해줘.

핵심 목표:
- 첫 번째 단계는 기본으로 펼쳐진 상태
- 각 단계는 입력 상태를 요약해서 보여줌
- 사용자가 위에서 아래로 자연스럽게 진행
- 핵심 버튼은 명확하게 표시
- 스케줄 재생성 결과가 월별 근무표와 연간 야간조 계획에 실제 반영되는 구조가 보이게 함

--------------------------------------------------
[화면 구조 재정리]
--------------------------------------------------

현재 단계는 유지하되, 아래처럼 표시 방식을 바꿔줘.

1. 재생성 기준
- 기본으로 열려 있어야 함
- 사용자가 바로 선택할 수 있어야 함
- 선택지:
  - 수요예측 변경
  - 기타 사유

2. 변경 기준일
- 재생성 기준을 선택하면 바로 아래에 표시
- 설명:
  “이 날짜 이전 일정은 유지하고, 이후 일정만 다시 생성합니다.”

3. 영향 기간
- 수요예측 변경이면 자동 분석 결과를 제안
- 기타 사유이면 영향 시작일 / 종료일 입력

4. 보호 일정
- 자동 보호 일정 요약 표시
- 근무표에서 보호 일정 선택 버튼 제공

5. 야간조 조정
- 체크박스 형태로 단순 표시
- “야간조 계획도 함께 조정” 옵션 제공

6. 스케줄 재생성 실행
- 모든 필수값이 입력되면 실행 가능
- 실행 후 결과 미리보기 표시
- 최종적으로 새 버전 저장

--------------------------------------------------
[첫 화면 기본 표시]
--------------------------------------------------

첫 화면에서 최소한 아래 내용은 바로 보여야 합니다.

상단 요약:
- 현재 적용 호텔
- 대상 월
- 현재 근무표 버전
- 현재 수요예측 버전
- 재생성 상태: 설정 전 / 분석 완료 / 미리보기 완료 / 저장 완료

첫 번째 카드:
- “재생성 기준”은 기본으로 펼쳐진 상태
- 수요예측 변경 / 기타 사유 선택 버튼 표시

하단 또는 우측:
- 현재 진행 단계 표시
- 다음 단계로 이동 버튼

--------------------------------------------------
[아코디언 동작 수정]
--------------------------------------------------

현재처럼 모든 카드가 닫혀 있고 빈 영역처럼 보이면 안 됩니다.

수정:
- 1단계는 기본 펼침
- 사용자가 1단계를 입력하면 2단계 자동 펼침
- 2단계를 입력하면 3단계 자동 펼침
- 완료된 단계는 접혀도 요약 정보가 보여야 함

예:
1. 재생성 기준
요약: 수요예측 변경 선택됨

2. 변경 기준일
요약: 2026-05-12부터 재생성

3. 영향 기간
요약: 2026-05-12 ~ 2026-05-14

4. 보호 일정
요약: 절대 보호 5건 / 우선 보호 8건

5. 야간조 조정
요약: 야간조 계획 함께 조정 안 함

6. 스케줄 재생성 실행
요약: 미리보기 전

--------------------------------------------------
[재생성 기준 카드]
--------------------------------------------------

재생성 기준 카드는 아래처럼 단순하게 구성해줘.

선택지:
1) 수요예측 변경
- 최신 수요예측을 업로드하거나 기존 예측과 비교해서 재생성

2) 기타 사유
- 시설 점검, 행사, 장기 부재, 기타 운영 사유를 직접 입력해서 재생성

기타 사유 선택 시:
- 사유 입력 필드
- 영향 시작일
- 영향 종료일

중요:
- 시설 점검 / 행사 / 장기 부재 / 퇴사 등을 별도 복잡한 폼으로 나누지 말 것
- 기타 사유는 텍스트 입력으로 처리

--------------------------------------------------
[변경 기준일 카드]
--------------------------------------------------

변경 기준일 카드에는 아래만 간단히 표시해줘.

- 변경 기준일 date picker
- 설명:
  “변경 기준일 이전 일정은 유지하고, 이후 일정만 다시 생성합니다.”

불필요한 설명 제거:
- 오늘 이전 날짜 잠금 설명 제거
- 과거 날짜 관련 복잡한 설명 제거
- 다음 달 스케줄 생성 기준에서는 변경 기준일만 핵심으로 사용

--------------------------------------------------
[영향 기간 카드]
--------------------------------------------------

영향 기간은 단순하게 구성해줘.

수요예측 변경 선택 시:
- 업로드된 예측 기준 영향 날짜 자동 제안
- 사용자가 수정 가능

기타 사유 선택 시:
- 영향 시작일
- 영향 종료일

--------------------------------------------------
[보호 일정 카드]
--------------------------------------------------

보호 일정 카드는 아래처럼 단순하게 보여줘.

요약:
- 절대 보호 0건
- 우선 보호 0건

버튼:
- 근무표에서 보호 일정 선택

동작:
- 기존 월별 근무표를 선택 모드로 열기
- 셀 다중 선택 가능
- 보호 수준 선택:
  - 절대 보호
  - 우선 보호
- 보호 일정으로 저장
- 저장 후 스케줄 재생성 화면으로 돌아와 요약에 반영

중요:
- 사용자가 여러 사람의 일정을 날짜 입력으로 하나씩 추가하게 하지 말 것
- 근무표를 보면서 클릭 선택하는 방식으로 처리
- 기존 월별 근무표 컴포넌트를 재사용하되 “보호 일정 선택 모드”로 열기

자동 절대 보호 대상:
- 승인된 교육
- 승인된 병가
- 직원이 신청했고 승인 안내까지 나간 휴일
- 직원에게 승인 안내가 나간 연차
- 직원에게 승인 안내가 나간 SL
- 직원에게 승인 안내가 나간 HO
- 신혼여행
- 장기 병가
- 장기 교육
- 퇴사 이후 기간

우선 보호 대상:
- SL
- 연차
- HO
- 승인 전 또는 안내 전인 휴무 후보
- 아직 직원에게 공지되지 않은 OFF
- 스케줄러가 우선 보호로 지정한 일정

--------------------------------------------------
[야간조 조정 카드]
--------------------------------------------------

야간조 조정은 항상 선택할 수 있게 하되 단순하게 보여줘.

체크박스:
[ ] 야간조 계획도 함께 조정

체크하지 않으면:
- 월별 근무표만 재생성
- 연간 야간조 계획은 변경하지 않음

체크하면:
- 월별 근무표 재생성 결과와 함께
- 해당 월 연간 야간조 계획도 함께 조정

체크 시 표시:
- 자동 재설정
- 직접 우선 대상자 선택
- 직접 제외 대상자 선택

기본값:
- 자동 재설정

중요:
- 특정 사건이 있을 때만 야간조 조정 카드가 나타나면 안 됨
- 스케줄러가 원할 때 언제든 선택할 수 있어야 함
- 대체 후보 목록을 별도로 강제 표시하지 말 것

--------------------------------------------------
[스케줄 재생성 실행 카드]
--------------------------------------------------

마지막 카드에는 실제 실행 버튼을 명확히 보여줘.

버튼:
- 스케줄 재생성

버튼 활성 조건:
- 재생성 기준 선택 완료
- 변경 기준일 입력 완료
- 영향 기간 입력 완료

클릭 시:
- 운영정책 기반으로 영향 기간만 재생성
- 보호 일정 반영
- 야간조 조정 옵션 반영
- 결과 미리보기 생성

중요:
- 버튼 클릭 후 아무 일도 안 일어나면 안 됨
- 단순 toast만 띄우고 끝내면 안 됨
- 결과 미리보기가 실제로 생성되어야 함

--------------------------------------------------
[결과 미리보기]
--------------------------------------------------

스케줄 재생성 후 결과 미리보기를 같은 화면 하단에 표시해줘.

결과 미리보기 포함 항목:
- 월별 근무표 변경 건수
- 변경된 직원 / 날짜 / 기존 코드 / 변경 코드
- 중간조 추가 또는 조정 내역
- 보호 일정 유지 건수
- 보호 일정 변경 필요 건수
- 야간조 계획 변경 여부
- 연간 야간조 계획 변경 내역
- 남은 운영 경고
- 새 버전명

버튼:
- 새 버전으로 저장
- 다시 조정
- 취소

--------------------------------------------------
[새 버전 저장]
--------------------------------------------------

“새 버전으로 저장”을 누르면 실제 데이터에 반영되어야 함.

반영 대상:
1) 월별 근무표
2) 연간 야간조 계획
3) 변경 이력
4) 대시보드 상태
5) 운영 경고
6) 공정성 리포트
7) 버전 관리

중요:
- 기존 버전을 덮어쓰지 말 것
- 새 버전으로 저장
- 월별 근무표 셀이 실제로 변경되어야 함
- 야간조 계획도 함께 조정 옵션이 켜져 있으면 연간 야간조 계획도 실제로 변경되어야 함

현재 화면은 핵심 버튼이 보이지 않아 사용자가 막히는 느낌이 있음.
