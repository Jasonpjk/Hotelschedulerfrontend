현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “근태 신청” 화면을 아래 기준으로 수정해주세요.

이번 수정은 일반 직원용 근태 신청 화면의 사용성을 높이고,
자동 스케줄 엔진과 연결된 실제 반영 결과를 더 명확하게 보여주기 위한 보완입니다.

중요:
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 유지
- 넓은 여백, 얇은 선, 정돈된 그리드, 실무형 UI 유지
- 직원용 화면이지만 전체 시스템 안에서 동일한 브랜드/운영 UI처럼 보여야 함

[1. 휴가 / 연차 잔여 표시 방식 변경]
현재 상단의 휴가 잔여 / 연차 잔여는 단순히 남은 일수만 보이지 말고,
반드시 “총 일수 중 남은 일수” 형태로 표시해주세요.

표시 예시:
- 휴가 잔여 15일 중 5일
- 연차 잔여 15일 중 12일

또는 더 간결한 표기:
- 휴가 15일 중 5일 남음
- 연차 15일 중 12일 남음

중요:
- 직원이 본인 잔여 현황을 직관적으로 이해할 수 있어야 함
- 관리자 화면과 연계될 수 있도록 일관된 수치 구조로 보이게 해주세요

[2. 승인 완료 후에도 수정 / 재신청 / 삭제 가능]
현재 신청 건이 승인 완료 또는 자동 반영 완료 상태가 되었더라도
직원이 다시 수정하거나, 변경해서 재신청하거나, 삭제할 수 있는 기능을 추가해주세요.

필수 기능:
- 기존 신청 내역 상세 보기
- 기존 신청 내용 수정
- 수정 후 재신청
- 신청 삭제

중요:
- 이미 자동 반영 완료된 건이라도 사용자가 다시 조정 요청할 수 있어야 함
- 단, 실제 운영상 확정 단계와 충돌할 수 있으므로
  상태에 따라 “수정 가능”, “재신청 가능”, “삭제 가능” 조건이 다르게 보일 수 있음
- 이 조건은 너무 복잡하게 보이지 말고, 직원 입장에서는 자연스럽게 이해되게 구성

권장 UX:
- 각 신청 내역의 “보기” 버튼을 누르면 상세 모달 또는 상세 페이지 오픈
- 그 안에서 아래 버튼 제공
  - 수정
  - 다시 신청
  - 삭제
- 수정 후 저장하면 기존 건을 업데이트하거나 재신청 프로세스로 연결
- 삭제 시 확인 모달 노출

[3. 여러 날짜 일괄 신청 기능 추가]
근태 신청 시 하루씩 따로 신청하는 방식이 아니라,
여러 날짜를 동시에 선택해서 한 번에 일괄 신청할 수 있는 기능을 추가해주세요.

반드시 가능한 흐름:
1. 달력에서 여러 날짜 선택
2. 유형 선택
   - 휴가
   - SL
   - 연차
3. 사유 입력(선택)
4. 일괄 신청 제출

중요:
- 직원이 연속 날짜 또는 여러 개의 개별 날짜를 동시에 선택할 수 있어야 함
- 예:
  - 4월 10일, 4월 11일, 4월 15일 선택
  - 또는 4월 21일~4월 23일 범위 선택
- 일괄 신청 시 선택된 날짜 목록이 명확히 보이게 해주세요

권장 표시 방식:
- 신청 팝업 또는 신청 패널에서
  - 선택 날짜 목록
  - 총 선택 일수
  - 신청 유형
  - 사유(선택)
  확인 가능

[4. 신청 유형과 실제 반영 유형을 구분해서 보여주기]
중요:
자동 스케줄 엔진은 14일 안에 4일 휴무 규칙을 먼저 반영하기 때문에,
직원이 “연차”로 신청했더라도 실제 근무표에는 “휴무”로 반영될 수 있습니다.

따라서 직원이 아래를 반드시 구분해서 볼 수 있어야 합니다.
- 내가 신청한 유형
- 실제 근무표에 반영된 유형

예시 상황:
- 신청 유형: 연차
- 실제 반영: 휴무

즉,
근태 신청 화면과 신청 상세 보기에서
“신청값”과 “최종 반영값”을 분리해서 보여주세요.

권장 컬럼 또는 표시 방식:
- 신청 유형
- 최종 반영
- 상태

예:
- 신청 유형: 연차
- 최종 반영: 휴무
- 상태: 자동 반영 완료

또는 상세 모달에서
- 내가 신청한 유형
- 시스템이 최종 반영한 유형
- 반영 사유 요약
을 보여줄 수 있게 해주세요.

[5. 14일 4휴무 우선 규칙이 반영 결과에 드러나게]
자동 스케줄 엔진은
“14일 안에 4일 휴무” 규칙을 먼저 만족시키는 것이 우선입니다.

따라서 직원이 신청 내역을 볼 때,
왜 본인의 신청이 그대로 반영되지 않았는지 이해할 수 있어야 합니다.

권장 안내 예시:
- 신청 유형은 연차였으나, 해당 날짜는 기본 휴무 규칙에 따라 휴무로 반영되었습니다.
- 운영 스케줄 기준에 따라 실제 반영 유형이 변경되었습니다.
- 14일 4휴무 규칙 우선 반영 후 신청이 조정되었습니다.

중요:
- 너무 긴 설명문이 아니라
- 태그, 보조 문구, 상세 모달 설명 수준으로
직원이 이해할 수 있게 정리해주세요.

[6. 신청 목록 테이블 구조 보완]
현재 “내 신청 내역” 테이블에 아래 항목이 더 명확히 보이게 수정해주세요.

권장 컬럼:
- 날짜
- 신청 유형
- 최종 반영
- 사유
- 상태
- 신청일
- 상세 / 관리

중요:
- 기존 “유형”만 보여주는 구조보다
  “신청 유형”과 “최종 반영”을 구분해서 보여주는 방향이 더 중요
- 사유는 없을 수 있으므로 빈값 또는 “-” 처리 가능
- 상태와 최종 반영 결과가 함께 읽히도록 정리해주세요

[7. 상태값과 액션 정리]
근태 신청 화면에서는 아래 상태 구조가 자연스럽게 보이게 해주세요.

예시 상태:
- 신청 접수
- 자동 반영 완료
- 검토 필요
- 반영 불가
- 수정 재신청

또는 기존 구조를 유지하되,
상세 화면에서는 현재 가능한 액션이 명확히 보여야 합니다.

각 신청 건 상세에서 가능하면 좋은 액션:
- 수정
- 재신청
- 삭제

중요:
- 자동 반영 완료된 건도 완전히 잠가두지 말고
  수정 또는 재신청 흐름으로 이어질 수 있어야 함
- 다만 이미 근무표 최종 확정 이후라면 수정 제한 안내가 붙을 수 있음

[8. 신청하기 UX 보완]
상단 “신청하기” 버튼을 누르면 열리는 신청 UI는
단일 날짜 선택이 아니라 다중 날짜 선택에 최적화된 구조로 바꿔주세요.

신청 UI에 포함할 요소:
- 달력
- 다중 날짜 선택
- 연속 날짜 선택 가능
- 선택된 날짜 리스트
- 신청 유형 선택(휴가 / SL / 연차)
- 사유 입력(선택)
- 일괄 신청 버튼

중요:
- 직원이 직관적으로 여러 날짜를 선택할 수 있어야 함
- 선택된 날짜가 시각적으로 분명히 표시되어야 함
- 신청 전에 어떤 날짜를 선택했는지 재확인할 수 있어야 함

[9. 직원용 화면에서 보여줄 핵심 정보]
직원용 근태 신청 화면에서 사용자가 바로 이해해야 하는 것은 아래입니다.

- 내 휴가는 총 몇 일 중 몇 일이 남았는지
- 내 연차는 총 몇 일 중 몇 일이 남았는지
- 내가 어떤 날짜를 어떤 유형으로 신청했는지
- 실제로는 무엇으로 반영됐는지
- 자동 반영 완료인지 검토 필요인지
- 이미 낸 신청을 다시 수정하거나 삭제할 수 있는지

즉,
직원용 화면은 단순 제출 화면이 아니라
“신청 → 자동 반영 → 최종 반영 결과 확인 → 수정/재신청”
흐름이 연결된 화면처럼 보여야 합니다.

[10. 톤앤매너 유지]
반드시 현재까지 만든 시스템과 동일한 톤앤매너를 유지해주세요.

유지 조건:
- LOTTE HOTELS & RESORTS 관리자/직원 공용 SaaS 무드
- 차분하고 정제된 실무형 UI
- 과한 캘린더 앱 느낌 금지
- 과한 카드형 디자인 금지
- 상태 태그, 표, 버튼, 모달이 기존 컴포넌트 체계와 연결된 느낌으로 보이게

[11. 최종 요청]
현재 “근태 신청” 화면을 아래 기준으로 수정해주세요.

- 휴가 / 연차 잔여를 “총 일수 중 남은 일수” 형태로 표시
- 자동 반영 완료된 신청도 수정 / 재신청 / 삭제 가능하게 기능 추가
- 여러 날짜를 동시에 선택해서 일괄 신청할 수 있게 수정
- 신청 유형과 실제 최종 반영 유형을 구분해서 표시
- 14일 안에 4일 휴무 규칙이 먼저 반영되기 때문에
  신청은 연차였지만 실제 반영은 휴무일 수 있다는 점이 화면에서 이해되게 구성
- 신청 상세 보기에서 반영 결과와 조정 이유를 확인할 수 있게 정리

이 기준으로 근태 신청 화면의 UI와 기능 흐름을 보완해주세요.
