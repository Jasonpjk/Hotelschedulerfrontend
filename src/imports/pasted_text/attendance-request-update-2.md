현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “근태 신청” 화면을 아래 기준으로 수정해주세요.

중요:
- 이번 수정은 “근태 신청” 화면에만 적용해주세요.
- 다른 화면은 건드리지 말아주세요.
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 / 넓은 여백 / 얇은 선 / 정돈된 그리드 유지

[현재 문제]
현재 근태신청에서 여러 날짜를 일괄로 신청하면
예를 들어
- 4월 4일
- 4월 12일
이렇게 2일을 신청했을 때

신청 내역에
- 4월 4일 외 1일
처럼 축약되어 보여서,
직원이 어떤 날짜들을 신청했는지 바로 알아보기 어렵습니다.

이 방식은 실제 운영상 불편하므로 수정해주세요.

[1. 날짜 축약 표시 제거]
여러 날짜를 일괄 신청했더라도
“4월 4일 외 1일”처럼 축약하지 말아주세요.

즉,
- 외 n일
- 외 1일
- 외 2일
같은 축약형 표현 제거

중요:
- 사용자가 신청한 날짜를 한눈에 바로 확인할 수 있어야 함
- 날짜가 숨겨져 있으면 안 됨

[2. 신청 날짜 각각 독립적으로 표시]
일괄 신청한 날짜들은
각 날짜가 각각 독립적으로 보이게 수정해주세요.

예시:
잘못된 방식
- 4월 4일 외 1일

원하는 방식
- 4월 4일
- 4월 12일

또는 한 행 안에서 보여야 한다면
- 4월 4일, 4월 12일

즉,
신청한 날짜가 모두 그대로 드러나야 합니다.

[3. 가장 좋은 표시 방식]
가장 권장하는 방식은 아래 둘 중 하나입니다.

방식 A. 날짜별 개별 태그 표시
- 4월 4일
- 4월 12일
- 4월 18일

방식 B. 줄바꿈 또는 리스트 표시
- 4월 4일
- 4월 12일
- 4월 18일

중요:
- 날짜가 각각 독립된 값처럼 읽혀야 함
- 사용자가 상세를 누르지 않아도 신청한 날짜를 바로 알 수 있어야 함

[4. 신청 내역 테이블/카드에서 바로 보이게]
이 날짜 표시는
상세 버튼을 눌러야만 보이는 구조가 아니라,
근태 신청 메인 목록에서 바로 보이게 해주세요.

즉,
직원이 내 신청 내역을 봤을 때
- 내가 며칠을 신청했는지
- 정확히 어느 날짜를 신청했는지
즉시 알 수 있어야 합니다.

[5. 날짜가 많을 경우 표시 방식]
신청 날짜가 아주 많아질 경우에도
날짜를 숨기거나 “외 n일”로 축약하지 말고,
아래처럼 정리해주세요.

권장 방식:
- 날짜 태그가 여러 줄로 자연스럽게 줄바꿈
- 또는 신청 날짜 영역 안에서 세로 리스트 표시
- 또는 카드 안에서 날짜칩이 wrap되는 구조

중요:
- 많아도 각각의 날짜가 보여야 함
- 단지 공간상 정돈해서 보여줄 뿐, 축약해서 숨기지 않기

[6. 신청 내역에서 읽혀야 하는 구조]
수정 후에는 근태 신청 내역에서 아래가 명확히 보여야 합니다.
- 신청 유형 또는 신청 의사
- 신청한 날짜들 각각
- 최종 반영 결과
- 상태

즉,
“몇 일 신청했는지”보다
“어느 날짜를 신청했는지”가 더 중요하게 보이도록 정리해주세요.

[7. 최종 요청]
현재 근태 신청 화면을 아래 기준으로 수정해주세요.

- 여러 날짜를 일괄 신청했을 때 “외 n일” 형태로 축약하지 않기
- 신청한 날짜들이 각각 독립적으로 보이게 표시
- 근태 신청 메인 목록에서 상세를 누르지 않아도 날짜들이 바로 보이게 수정
- 날짜가 많을 경우에도 태그/줄바꿈/리스트 방식으로 모두 노출
- 사용자가 내가 신청한 쉬는 날이 정확히 언제인지 한눈에 알 수 있게 정리

핵심은,
일괄 신청한 날짜들을 축약하지 말고
신청한 모든 날짜가 각각 독립적으로 바로 보이게 만드는 것입니다.

핵심: “4월 4일 외 1일”처럼 축약하지 말고, 신청한 날짜들을 4월 4일 / 4월 12일처럼 각각 독립적으로 그대로 보여주세요.
