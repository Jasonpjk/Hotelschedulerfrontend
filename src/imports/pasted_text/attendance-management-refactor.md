현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “근태 관리” 화면을 아래 기준으로 다시 재구성해주세요.

중요:
- 이번 수정은 “근태 관리” 화면을 중심으로 적용해주세요.
- 단, “근태신청 마감” 기능은 직원용 “근태 신청” 화면에도 연결되어 반영되어야 합니다.
- 다른 화면은 건드리지 말아주세요.
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 / 넓은 여백 / 얇은 선 / 정돈된 그리드 유지

[1. 근태 관리의 기본 역할 재정의]
근태 관리는 관리자가 직원 신청을 하나씩 승인/반려하는 화면이 아니라,
직원들이 제출한 “쉬는 날 희망”과 “휴가 계획”을 기준으로
시스템이 자동 반영한 결과를 검토하고,
필요한 예외만 조정하는 화면처럼 보이게 해주세요.

즉,
근태 관리는 아래 성격을 가져야 합니다.
- 자동 반영 결과 확인
- 예외 상황 검토
- 일부 수동 조정
- 최종 반영 상태 관리

중요:
- 전건 수동 승인 화면처럼 보이면 안 됨
- 자동 배정 결과를 보는 화면처럼 보여야 함

[2. 직원 신청 구조를 전제로 화면 정리]
근태 관리 화면은 아래 구조를 전제로 설계해주세요.

직원 일반 신청:
- 특정 날짜에 쉬는 날 희망 제출
- 주말 / 공휴일 근무 희망은 별도 신청
- 휴가는 직원이 직접 별도 신청 가능

시스템 자동 반영:
- 일반 쉬는 날 희망에 대해 시스템이 자동 반영
- 자동 반영 가능한 최종 유형은 아래 2개만 허용
  - SL
  - 연차

중요:
- 시스템은 휴가를 자동 배정하지 않음
- 휴가는 직원이 직접 신청한 경우에만 반영
- 일반 쉬는 날 희망 결과는 SL 또는 연차 또는 미반영으로 정리

[3. 자동 반영 우선순위가 화면에 드러나게]
근태 관리 화면에서는 시스템의 자동 배정 우선순위가 자연스럽게 이해되어야 합니다.

자동 반영 우선순위:
1. SL 먼저 배정
2. 해당 월 SL이 이미 소진되었으면 연차 배정

중요:
- SL은 여성 보건휴가로 월 1회 사용 가능
- SL이 이미 사용된 경우 더 이상 SL 자동 배정하지 않음
- 그 다음부터는 연차로 자동 배정

즉,
관리자가 근태 관리 화면을 볼 때
왜 어떤 직원은 SL로 들어갔고, 어떤 직원은 연차로 들어갔는지 이해할 수 있어야 합니다.

[4. 근태 관리 화면의 메인 구조]
근태 관리 화면은 직원별 신청 묶음 + 자동 반영 결과 중심 구조로 보여야 합니다.

권장 메인 테이블/리스트 컬럼:
- 직원명
- 신청 건수
- 신청 날짜 요약
- 신청 의사
- 최종 반영 결과 요약
- 자동 반영 완료 건수
- 검토 필요 건수
- 미반영 건수
- 휴가 잔여
- 연차 잔여
- 관리 / 상세 검토

중요:
- 날짜 1건당 한 줄이 아니라 직원별 신청 묶음 중심 구조
- 직원이 여러 날짜를 신청한 경우 한 직원 기준으로 묶어서 요약
- 상세 검토 시 날짜별 내역을 별도로 확인 가능하게

[5. 상세 검토 화면에 반드시 보여야 할 정보]
직원별 “관리” 또는 “상세 검토” 버튼을 누르면
해당 직원의 날짜별 세부 내역이 보여야 합니다.

상세 영역에 포함할 항목:
- 날짜
- 신청 의사
  - 쉬는 날 희망
  - 신청 없음
  - 휴가 신청
- 최종 반영 결과
  - SL
  - 연차
  - 휴가
  - 미반영
- 반영 조정 사유
- 자동 반영 여부
- 수동 조정 여부
- 휴가 잔여
- 연차 잔여
- 상태
- 조정 액션

중요:
- 신청 의사 / 최종 반영 / 반영 조정 사유는 반드시 분리
- 관리자가 왜 이렇게 반영되었는지 바로 이해할 수 있어야 함

[6. 신청하지 않은 자동 배정도 관리 가능하게]
직원이 쉬는 날 신청을 하지 않았는데도
운영상 적정 인원 대비 근무 인원이 과다하여
시스템이 자동으로 SL 또는 연차를 배정한 경우도
근태 관리 화면에 보여야 합니다.

예시:
- 신청 의사: 신청 없음
- 최종 반영: SL
- 반영 조정 사유: 적정 인원 대비 근무 인원 과다로 SL을 배정했습니다

또는
- 신청 의사: 신청 없음
- 최종 반영: 연차
- 반영 조정 사유: 적정 인원 대비 근무 인원 과다로 연차를 배정했습니다

중요:
- 신청하지 않은 자동 배정도 관리 대상에 포함
- 관리자가 자동 배정 결과를 검토할 수 있어야 함

[7. 관리자 수동 조정은 SL → 연차 변경 중심]
근태 관리 화면에서 관리자가 나중에 손댈 수 있는 핵심 수동 조정은
“자동 배정된 SL을 연차로 변경하는 것”에 집중해주세요.

즉,
관리자의 대표 수동 액션은 아래처럼 보이게 해주세요.
- 현재 최종 반영: SL
- 관리자 수동 조정: 연차로 변경

중요:
- 이번 단계에서는 관리자 수동 조정을 복잡하게 많이 열지 말고
- 핵심 액션을 “SL을 연차로 변경”하는 흐름 중심으로 정리해주세요

[8. SL → 연차 수동 조정 UX]
관리자가 상세 검토 화면에서
SL로 자동 배정된 항목을 연차로 변경할 수 있게 해주세요.

권장 흐름:
1. 상세 검토 진입
2. 현재 최종 반영이 SL인 항목 확인
3. “연차로 변경” 또는 “연차로 수정” 버튼 클릭
4. 확인 팝업 또는 즉시 변경
5. 최종 반영 결과가 연차로 바뀜
6. 수동 조정 이력이 남음

중요:
- 관리자는 왜 이 항목을 조정하는지 직관적으로 이해할 수 있어야 함
- 연차 소진율 관리 때문에 조정하는 흐름처럼 읽히게 해주세요

[9. 직원 화면에 보여줄 사유와 연결]
관리자가 근태 관리에서
SL을 연차로 수동 조정한 경우,
근태 신청 화면의 직원에게는 아래 문구만 보이게 연결되도록 설계해주세요.

직원에게 보여줄 문구:
- 관리자가 SL을 연차로 수정했습니다.

중요:
- 직원에게는 복잡한 운영 사유를 길게 보여주지 않음
- 단순하고 명확한 안내만 보이게
- 근태 관리에서 발생한 수동 조정 결과가 근태 신청 화면에도 반영되는 구조처럼 보여야 함

[10. 반영 조정 사유 구조]
근태 관리 화면에서는 반영 조정 사유가 더 상세히 보여도 되지만,
직원 화면과 연결될 때는 간단히 정리되도록 구성해주세요.

근태 관리에서 보이는 예시:
- 적정 인원 대비 근무 인원 과다로 SL을 배정했습니다
- 적정 인원 대비 근무 인원 과다로 연차를 배정했습니다
- 관리자가 SL을 연차로 수정했습니다

중요:
- 자동 반영 사유와 관리자 수동 조정 사유를 구분해서 보이게
- 관리자 화면에서는 상세,
  직원 화면에서는 필요한 최소 문구만 보이게 연결

[11. 직원별 미반영 건수 / 예외 건수 유지]
근태 관리 화면은 여전히 예외 관리 화면이어야 하므로,
아래 정보가 유지되어야 합니다.
- 직원별 미반영 건수
- 직원별 검토 필요 건수
- 반영 안 된 직원 수
- 자동 반영 완료 건수

즉,
자동 배정 결과를 관리자 관점에서 빠르게 파악할 수 있어야 합니다.

[12. 직원별 휴가 / 연차 잔여 표시 유지]
근태 관리 담당자는 직원별로 아래를 계속 볼 수 있어야 합니다.
- 휴가 잔여
- 연차 잔여

표시 예시:
- 휴가: 15일 중 5일
- 연차: 15일 중 12일

중요:
- 연차 소진율을 보고 SL을 연차로 바꾸는 판단을 할 수 있어야 하므로
- 연차 잔여 정보는 특히 잘 보여야 합니다

[13. 근태신청 마감 기능 추가]
근태 관리 화면 상단 또는 주요 액션 영역에
“근태신청 마감” 버튼을 추가해주세요.

이 버튼의 역할:
- 해당 월 또는 현재 작업 대상 기간의 직원 근태 신청을 마감
- 마감 후에는 직원들이 더 이상 기존 근태 신청을 수정, 삭제, 재신청할 수 없게 만듦

권장 흐름:
1. 관리자 화면에서 “근태신청 마감” 버튼 클릭
2. 확인 팝업 오픈
   - 이 기간의 근태 신청을 마감하시겠습니까?
   - 마감 후 직원은 신청 내역을 수정하거나 삭제할 수 없습니다
3. 취소 / 마감 버튼
4. 마감 완료 후 상태 표시

중요:
- 이 버튼은 관리자 전용 액션처럼 보여야 함
- 단순 토글보다 명확한 확정 액션처럼 보이게

[14. 마감 후 직원 화면 잠금 연결]
근태신청 마감이 완료되면
직원용 “근태 신청” 화면에서도 아래가 반영되어야 합니다.

반드시 반영:
- 기존 신청 내역의 수정 버튼 비활성화
- 삭제 버튼 비활성화
- 재신청 불가
- 신규 신청도 차단할지 여부는 현재는 기존 수정/삭제 차단 중심으로 보이게 해도 됨
- 화면에 마감 상태 안내 문구 표시

직원 화면 안내 문구 예시:
- 근태 신청이 마감되어 더 이상 수정할 수 없습니다
- 현재 기간의 근태 신청은 마감되었습니다

중요:
- 마감 후에는 사용자가 왜 수정이 안 되는지 바로 이해할 수 있어야 함
- 버튼만 죽어 있으면 안 되고, 마감 상태가 분명히 보여야 함

[15. 마감 상태 표시]
근태 관리 화면과 근태 신청 화면 모두에서
현재 기간이 마감되었는지 여부가 분명히 보여야 합니다.

권장 표시:
- 상태 배지: 마감됨
- 상단 안내 바
- 마지막 마감 시각 표시
- 마감 처리한 관리자 표시 가능

중요:
- 관리자는 마감 여부를 명확히 알아야 하고
- 직원도 현재 수정 가능 기간인지 아닌지 바로 알아야 함

[16. 화면이 읽혀야 하는 방식]
근태 관리 화면은 사용자가 봤을 때 아래처럼 읽혀야 합니다.
- 직원은 쉬는 날 희망만 냈다
- 시스템은 우선 SL, 그 다음 연차로 자동 반영했다
- 휴가는 자동 배정되지 않는다
- 신청이 없더라도 인원 과다로 자동 배정될 수 있다
- 관리자는 필요하면 SL을 연차로 바꿀 수 있다
- 근태신청 마감을 누르면 직원은 더 이상 신청 수정이 불가능하다
- 그 결과는 직원 화면에 단순 명확하게 안내된다

[17. 최종 요청]
현재 근태 관리 화면을 아래 기준으로 다시 수정해주세요.

- 근태 관리를 자동 배정 결과 + 예외 관리 화면으로 재구성
- 일반 쉬는 날 희망에 대한 시스템 자동 반영 유형은 SL과 연차만 허용
- 휴가는 시스템 자동 배정 대상에서 제외
- 자동 배정 우선순위는 SL 먼저, 월 1회 소진 후 연차로 반영
- 신청하지 않았는데도 인원 과다로 자동 배정된 경우도 관리 화면에서 보이게
- 관리자가 상세 검토에서 SL을 연차로 수동 조정할 수 있게 구성
- 이 수동 조정 결과는 직원 근태 신청 화면에서
  “관리자가 SL을 연차로 수정했습니다.”
  라고만 안내되도록 연결
- 직원별 미반영 건수, 검토 필요 건수, 연차/휴가 잔여도 함께 보이게 유지
- 근태 관리 화면에 “근태신청 마감” 버튼 추가
- 마감 후에는 직원들이 근태 신청 내역을 수정/삭제/재신청하지 못하게 연결
- 마감 상태와 안내 문구가 관리자/직원 화면 양쪽에 명확히 보이게 정리

핵심은,
근태 관리가 “자동으로 배정된 SL/연차 결과를 검토하고, 필요하면 관리자가 SL을 연차로 조정하며, 마감 후에는 직원 수정이 막히는 화면”처럼 보이도록 만드는 것입니다.
핵심 보강: 관리자 수동 조정은 SL → 연차 변경 중심으로 보여주고, 근태신청 마감을 누르면 직원 화면의 수정/삭제/재신청이 모두 잠기게 해주세요.
