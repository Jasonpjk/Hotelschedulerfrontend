현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “직원 관리” 화면을 아래 기준으로 수정해주세요.

이번 수정은 직원 관리 팝업의 필수 입력 구조, 필터 문구, 자동 근속 계산, 프런트 복귀자 입력 방식, 그리고 상단 액션 버튼 구성까지 포함한 기능 보완입니다.

[1. 배치 그룹 선택값 → 필수값으로 변경]
현재 관리 팝업에서 “배치 그룹 선택”이 선택 사항처럼 보이는데,
이제부터는 선택이 아니라 필수 입력 항목으로 바꿔주세요.

수정 방향:
- “배치 그룹 선택”은 필수값으로 처리
- 라벨에도 필수 표시가 보이게 정리
- 저장 전 반드시 하나 이상 선택해야 하는 구조로 보이게 수정
- 안내 문구도 “선택 사항”처럼 보이지 않게 수정

예:
- 배치 그룹 선택 *
- 실제 근무표에 반영할 배치 그룹을 선택해주세요.

중요:
- 관리 팝업에서 배치 그룹을 선택하지 않으면 저장할 수 없는 흐름처럼 보이게 해주세요.
- 필수 입력 항목으로 명확하게 인식되어야 합니다.

[2. 상단 역할 필터 드롭다운 문구 수정]
직원 관리 화면 상단 필터에서
현재 “전체 역할” 드롭다운을 눌렀을 때 아래처럼 보이는 문구를 수정해주세요.

변경 전 예시:
- 인차지만
- 담당만

변경 후:
- 인차지
- 담당

즉,
역할 필터 드롭다운 옵션은 아래처럼 단순하고 명확하게 정리해주세요.
- 전체 역할
- 인차지
- 담당

[3. 관리 팝업에 입사일 / 프런트 시작일 입력 기능 추가]
현재 관리 팝업에서 직원의 날짜 정보를 직접 입력할 수 있게 해주세요.

반드시 추가할 입력 항목:
- 입사일
- 프런트 시작일

중요:
- 관리 팝업에서 날짜를 입력하면
- 전체 근속
- 현재 프런트 근속
- 누적 프런트 경력
이 자동 계산되도록 구성해주세요.

즉,
사용자가 날짜를 입력하면 근속은 수기 입력이 아니라 자동 계산 결과로 표와 팝업에 반영되어야 합니다.

[4. 자동 계산 결과가 표에 반영되게]
관리 팝업에서 날짜를 입력/수정한 뒤 저장하면
메인 테이블에 아래 값이 자동 계산되어 표시되게 해주세요.

자동 반영 대상:
- 전체 근속
- 현재 프런트 근속
- 누적 프런트 경력

예:
- 입사일 입력 → 전체 근속 자동 계산
- 프런트 시작일 입력 → 현재 프런트 근속 자동 계산
- 프런트 복귀 이력 포함 입력 → 누적 프런트 경력 자동 계산

중요:
- 계산 결과는 표에서 바로 읽히게
- 별도 수기 편집값처럼 보이지 않게
- 시스템이 계산한 값처럼 정리

[5. 프런트 복귀자 대응 기능 추가]
프런트에서 다른 직무로 이동했다가 다시 프런트로 돌아온 직원이 있으므로
“프런트 시작일”은 한 번만 입력하는 구조가 아니라
중복으로 여러 기간을 입력할 수 있어야 합니다.

원하는 방식:
- 관리 팝업 안에 프런트 이력 입력 영역 추가
- “추가” 버튼을 누르면 입력 블록이 하나 더 생김
- 각 블록에 아래 항목 입력 가능
  - 시작일
  - 종료일

즉,
프런트 근무 이력을 복수로 입력할 수 있게 해주세요.

권장 UX:
- 기본 1개 프런트 이력 입력 영역 표시
- “+ 추가” 버튼 클릭 시
  새로운 프런트 근무 기간 입력 행이 생성
- 각 행에는
  - 시작일
  - 종료일
  - 삭제 아이콘(선택)
  가 있으면 좋음

자동 계산 방식:
- 가장 최근 시작일 기준 → 현재 프런트 근속 계산
- 모든 프런트 이력 합산 → 누적 프런트 경력 계산

중요:
- “프런트 시작일” 컬럼에 굳이 “복귀자”라고 표시할 필요는 없음
- 복귀자 여부를 텍스트로 따로 강조하지 말고
- 이력 입력 구조와 자동 계산 결과로 자연스럽게 처리해주세요.

[6. “프런트 시작일”에서 복귀자 표시 제거]
현재 또는 이전 시안에서 프런트 시작일 아래에
“복귀자” 같은 보조 표시가 있다면 제거해주세요.

즉,
- 프런트 시작일 컬럼은 날짜 중심으로만 보이게
- 복귀자 여부는 별도 라벨로 노출하지 않아도 됨
- 대신 누적 프런트 경력 계산으로 자연스럽게 의미가 드러나게 정리

[7. 상단 버튼 구성 수정]
현재 직원 관리 화면 상단 우측에 있는 전체 확정 버튼 옆에
“직원 추가” 버튼을 새로 만들어주세요.

버튼 구성:
- 직원 추가
- 전체 확정

중요:
- 두 버튼 모두 검정색 버튼으로 맞춰주세요.
- 현재 시스템의 짙은 네이비/블랙 계열 액션 버튼 스타일과 통일
- 골드 버튼이 아니라 검정색 계열 주요 액션 버튼으로 수정

배치 예시:
- 우상단에 나란히 배치
- 직원 추가 버튼
- 전체 확정 버튼

[8. 직원 추가 기능]
새로 추가되는 “직원 추가” 버튼을 누르면
신규 직원 정보를 입력할 수 있는 팝업 또는 관리 팝업과 유사한 입력 화면이 열리게 해주세요.

직원 추가 시 입력 가능해야 하는 항목:
- 이름
- 직급
- 역할(인차지 / 담당)
- 입사일
- 프런트 이력 입력
  - 시작일
  - 종료일
  - 추가 버튼으로 복수 입력 가능
- 배치 그룹(필수)

저장 후 반영:
- 직원이 메인 테이블에 추가
- 전체 근속 / 현재 프런트 근속 / 누적 프런트 경력 자동 계산 표시
- 이후 전체 확정 흐름에 포함되게

[9. 관리 팝업 구성 정리]
직원별 관리 버튼을 눌렀을 때 열리는 팝업은 아래 흐름으로 정리해주세요.

관리 팝업 필수 항목:
- 직원명
- 직급
- 역할 선택(인차지 / 담당)
- 입사일 입력
- 프런트 이력 입력
  - 시작일
  - 종료일
  - 추가 버튼
- 배치 그룹 선택(필수)
- 자동 계산 결과 표시
  - 전체 근속
  - 현재 프런트 근속
  - 누적 프런트 경력
- 저장 / 취소 버튼

중요:
- “배치 그룹”은 필수
- 날짜 입력은 수정 가능
- 자동 계산 결과가 팝업 안에서도 보이고 저장 후 표에도 반영되어야 함

[10. 메인 테이블 정리 방향]
직원 관리 메인 테이블은 아래 정보가 명확히 보이게 유지해주세요.

유지/반영할 항목:
- 이름
- 직급
- 역할
- 전체 근속
- 프런트 시작일 또는 최근 프런트 시작 기준값
- 현재 프런트 근속
- 누적 프런트 경력
- 상태
- 관리 버튼

중요:
- 날짜/이력 입력 결과가 표에 자동 반영되어야 함
- 계산된 근속 값이 일관되게 보여야 함

[11. 전체 확정 흐름 유지]
기존 전체 확정 기능은 유지해주세요.

즉,
- 직원별 관리 팝업에서 역할 / 날짜 / 프런트 이력 / 배치 그룹 설정
- 신규 직원 추가 가능
- 모든 직원 설정 완료
- 전체 확정 버튼 클릭
- 근무표 생성 기준으로 일괄 반영

이 흐름은 유지하되,
이번 수정사항이 모두 반영된 형태로 정리해주세요.

[12. 톤앤매너 유지]
반드시 현재까지 만든 LOTTE HOTELS & RESORTS 관리자용 SaaS의 톤앤매너를 그대로 유지해주세요.

유지 조건:
- 프리미엄 호텔 운영 관리자용 SaaS 무드
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 기반이되
  이번 상단 액션 버튼은 검정색 계열로 통일
- 얇은 선, 넓은 여백, 정돈된 그리드
- 소비자용 인사관리 툴 느낌 금지
- 실제 운영팀이 쓰는 실무형 관리자 화면처럼 보이게

[13. 최종 요청]
현재 직원 관리 화면을 아래 기준으로 수정해주세요.

- 관리 팝업의 배치 그룹 선택은 필수값으로 변경
- “전체 역할” 필터 드롭다운 옵션 문구를 “인차지”, “담당”으로 수정
- 관리 팝업에서 입사일과 프런트 시작일 입력 가능하게 추가
- 날짜 입력 시 전체 근속 / 현재 프런트 근속 / 누적 프런트 경력 자동 계산
- 프런트 이력은 복수 입력 가능하게 “추가” 버튼 제공
- 각 이력은 시작일 / 종료일 입력 방식으로 구성
- 프런트 시작일에 “복귀자” 문구는 표시하지 않음
- 전체 확정 버튼 옆에 직원 추가 버튼 생성
- 직원 추가 버튼과 전체 확정 버튼은 모두 검정색으로 변경
- 직원 추가 팝업에서도 필요한 직원 정보를 입력할 수 있게 구성

이 기준으로 UI와 기능 흐름을 정리해주세요.

중요: 직원 관리 팝업은 단순 조회창이 아니라 역할, 날짜, 프런트 이력, 배치 그룹을 입력하고 자동 계산 결과를 반영하는 실질적인 직원 마스터 설정 화면처럼 보여야 합니다.
