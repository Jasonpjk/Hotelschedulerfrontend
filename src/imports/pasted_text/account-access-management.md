현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “호텔 설정 > 계정 및 접근 관리” 화면을 아래 기준으로 수정해주세요.

중요:
- 이번 수정은 “호텔 설정 > 계정 및 접근 관리” 화면에만 적용해주세요.
- 다른 화면은 건드리지 말아주세요.
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 / 넓은 여백 / 얇은 선 / 정돈된 그리드 유지

[1. 회원가입 승인 대기 영역에 관리 버튼 추가]
현재 “회원가입 승인 대기” 영역에는 신청자 정보만 보이는데,
각 신청 건마다 실제로 처리할 수 있는 “관리” 버튼이 필요합니다.

반드시 반영할 것:
- 회원가입 승인 대기 리스트의 각 행 오른쪽에 “관리” 버튼 추가
- 관리 버튼을 누르면 해당 신청 건을 처리할 수 있는 팝업 또는 상세 패널 오픈

즉,
단순 조회 화면이 아니라
실제로 승인 / 거절할 수 있는 계정 승인 관리 화면처럼 보여야 합니다.

[2. 관리 버튼을 누르면 승인 / 거절 액션 가능]
회원가입 승인 대기 리스트에서 “관리” 버튼을 누르면
아래 액션이 가능해야 합니다.

필수 액션:
- 승인
- 거절

팝업 또는 상세 패널에 보이면 좋은 정보:
- 이름
- 사번
- 이메일
- 연락처
- 호텔
- 부서
- 직급
- 신청일
- 현재 상태: 승인 대기

하단 버튼 예시:
- 취소
- 거절
- 승인

중요:
- 승인과 거절이 명확히 구분되어야 함
- 관리자용 승인 처리 팝업처럼 보여야 함

[3. 승인 시 활성 계정 관리에 자동 추가]
가장 중요한 요구사항:
회원가입 승인 대기 상태의 신청자를 “승인”하면
해당 인원이 자동으로 아래 “활성 계정 관리” 영역에 추가되어야 합니다.

즉, 흐름은 아래처럼 보여야 합니다.
1. 회원가입 승인 대기 리스트에서 관리 버튼 클릭
2. 승인 버튼 클릭
3. 상태가 승인 완료로 변경
4. 해당 신청자가 승인 대기 목록에서 빠짐
5. 동시에 활성 계정 관리 목록에 자동 추가됨

중요:
- 승인 후 별도로 다시 등록하는 흐름이면 안 됨
- 승인 = 활성 계정 생성 완료로 바로 이어져야 함
- 사용자가 “승인하면 활성 계정으로 넘어가는구나”를 즉시 이해할 수 있어야 함

[4. 거절 시 처리 방식]
회원가입 승인 대기 상태의 신청자를 “거절”하면
아래 흐름처럼 보이게 해주세요.

권장 흐름:
1. 관리 버튼 클릭
2. 거절 버튼 클릭
3. 확인 팝업 또는 즉시 처리
4. 해당 신청자는 승인 대기 목록에서 제거되거나
   상태가 “거절”로 바뀌어 별도 관리 가능하게 보임

권장 방식:
- MVP 기준으로는 승인 대기 목록에서 빠지고
- 최근 처리 이력 또는 수정 이력에 “거절” 내역이 남는 구조가 가장 깔끔함

[5. 승인 후 활성 계정 관리에 들어갈 기본 정보]
승인된 사용자가 “활성 계정 관리”로 이동하면
아래 정보가 자동으로 채워져 보이게 해주세요.

권장 컬럼:
- 이름
- 사번
- 이메일
- 호텔
- 부서
- 직급
- 권한
- 계정 상태
- 관리

기본값 예시:
- 권한: 일반 직원
- 계정 상태: 활성

중요:
- 승인만으로도 기본 계정이 생성된 느낌이 나야 함
- 권한은 기본적으로 일반 직원으로 들어가고
  이후 필요 시 최종 관리자가 권한 변경 가능하게 보이면 좋음

[6. 승인 / 거절 후 상태 피드백]
승인 또는 거절 후에는
사용자가 처리 결과를 즉시 알 수 있어야 합니다.

예시:
- 승인 완료
- 활성 계정에 추가되었습니다
- 거절 완료
- 회원가입 신청이 거절되었습니다

표현 방식:
- 토스트 메시지
- 상태 배지 변경
- 리스트 이동
- 상단 상태 메시지

중요:
- 버튼만 눌리고 끝나는 구조가 아니라
  결과가 화면에서 분명히 보여야 함

[7. 계정 및 접근 관리 화면 구조 정리]
이 화면은 아래 흐름이 자연스럽게 보이게 해주세요.

상단 또는 첫 번째 섹션:
- 회원가입 승인 대기
- 각 신청 건별 관리 버튼
- 승인 / 거절 처리 가능

두 번째 섹션:
- 활성 계정 관리
- 승인된 계정이 자동으로 누적
- 이후 비밀번호 초기화, 권한 변경, 비활성화 같은 관리 가능

즉,
“회원가입 승인 대기”와 “활성 계정 관리”가
완전히 따로 노는 것이 아니라
승인 액션으로 직접 연결되는 구조처럼 보여야 합니다.

[8. 디자인 방향]
반드시 아래 느낌을 유지해주세요.
- 실무형 관리자 계정 관리 화면
- 승인 대기 리스트와 활성 계정 리스트가 명확히 구분
- 승인/거절 버튼은 분명하지만 과하게 공격적으로 보이지 않게
- 관리 버튼을 통해 상세 처리하는 흐름이 자연스럽게 보이게
- 프리미엄 호텔 운영 관리자 콘솔 느낌 유지

[9. 최종 요청]
현재 호텔 설정 > 계정 및 접근 관리 화면을 아래 기준으로 수정해주세요.

- 회원가입 승인 대기 리스트 각 행에 “관리” 버튼 추가
- 관리 버튼 클릭 시 승인 / 거절 가능한 팝업 또는 상세 패널 오픈
- 승인하면 해당 신청자가 활성 계정 관리 영역에 자동으로 추가되게 수정
- 거절도 처리 가능하게 구성
- 승인/거절 후 결과가 사용자에게 명확히 보이게 상태 피드백 추가

핵심은,
“회원가입 승인 대기”에서 승인한 사람이
자동으로 “활성 계정 관리”로 넘어가는 흐름이
화면에서 분명히 보이도록 만드는 것입니다.

핵심: 회원가입 승인 대기 리스트에서 관리 버튼으로 승인/거절을 처리하고, 승인 시 해당 인원이 활성 계정 관리에 자동으로 추가되게 해주세요.
