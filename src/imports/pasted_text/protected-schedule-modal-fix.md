--------------------------------------------------
[현재 문제 1. 보호 수준이 뒤섞임]
--------------------------------------------------

현재 보호 일정 선택 모달에서 아래 문제가 발생함.

동작 예:
1) 우선 보호를 선택
2) 몇 개의 일정을 선택
3) 다시 절대 보호를 선택
4) 다른 일정을 선택
5) 저장

문제:
- 처음에 우선 보호로 선택했던 일정까지 모두 절대 보호로 저장됨

이건 잘못된 동작임.

--------------------------------------------------
[수정 목표 1. 셀마다 선택 당시의 보호 수준을 저장]
--------------------------------------------------

보호 일정은 “현재 선택된 보호 수준” 하나로 전체 선택값을 덮어쓰면 안 됨.

각 셀은 선택되는 순간의 보호 수준을 개별적으로 저장해야 함.

예:
- 우선 보호 모드에서 5월 3일 김민준 OFF 선택
  → 해당 셀은 우선 보호로 저장
- 이후 절대 보호 모드로 변경
- 5월 10일 이서연 VAC 선택
  → 해당 셀은 절대 보호로 저장

저장 결과:
- 김민준 5월 3일 = 우선 보호
- 이서연 5월 10일 = 절대 보호

중요:
- 보호 수준 버튼을 바꿨다고 기존 선택 셀의 보호 수준이 자동으로 바뀌면 안 됨
- 기존 선택 셀의 보호 수준은 사용자가 명시적으로 변경하기 전까지 유지
- 선택 목록은 보호 수준별로 따로 관리해야 함

권장 데이터 구조:
- selectedProtectedCells: Array
  - employeeId
  - employeeName
  - date
  - shiftCode
  - protectionLevel: absolute | priority
  - source: manualSelection

또는:
- absoluteProtectedCells
- priorityProtectedCells

둘 중 어떤 구조든 좋지만, 저장 시 보호 수준이 섞이면 안 됨.

--------------------------------------------------
[수정 목표 2. 선택 상태 표시]
--------------------------------------------------

선택된 셀은 보호 수준에 따라 다르게 표시해줘.

예:
- 절대 보호: 자물쇠 아이콘 또는 진한 테두리
- 우선 보호: 방패 아이콘 또는 연한 골드 테두리

셀을 선택한 뒤 보호 수준이 무엇인지 바로 알아볼 수 있어야 함.

하단 저장 버튼 문구도 보호 수준별 개수를 보여줘.

예:
- 보호 일정으로 저장 (절대 3건 / 우선 5건)

현재처럼 전체 선택 수만 보여주면 보호 수준이 헷갈릴 수 있음.

--------------------------------------------------
[현재 문제 2. 월말일까지 볼 수 없음]
--------------------------------------------------

현재 보호 일정 선택 모달 안의 근무표가 월말일까지 제대로 보이지 않음.
가로 스크롤 또는 날짜 영역 스크롤이 부족해서 월말 날짜까지 선택하기 어려움.

--------------------------------------------------
[수정 목표 3. 월말일까지 스크롤 가능]
--------------------------------------------------

근무표 선택 모달에서 해당 월의 마지막 날짜까지 모두 볼 수 있도록 스크롤을 추가해줘.

수정 요구:
- 날짜 열이 월말일까지 표시되어야 함
- 가로 스크롤바를 명확히 제공
- 직원명 고정 영역은 유지
- 날짜 영역만 가로 스크롤 가능하게 처리
- 세로 스크롤과 가로 스크롤이 서로 겹치거나 잘리지 않게 처리
- 모달 하단 버튼 영역 때문에 마지막 행이나 스크롤바가 가려지지 않게 처리

중요:
- 5월이면 5월 31일까지 선택 가능해야 함
- 2월이면 28일 또는 29일까지 선택 가능해야 함
- 선택 가능한 날짜는 변경 기준일 이후 날짜만 유지
- 변경 기준일 이전 날짜는 비활성 또는 잠금 표시

--------------------------------------------------
[현재 문제 3. 기존 절대 보호를 다시 우선 보호로 선택하면 기준이 없음]
--------------------------------------------------

한 번 절대 보호로 저장한 일정을 다시 열어서 우선 보호로 선택할 경우
어떻게 처리할지 확인 과정이 필요함.

특히 절대 보호는 강한 보호 수준이므로,
사용자가 우선 보호로 낮추려고 하면 반드시 확인해야 함.

--------------------------------------------------
[수정 목표 4. 기존 보호 수준 변경 시 확인창 표시]
--------------------------------------------------

이미 보호 일정으로 저장된 셀을 다시 선택하거나 보호 수준을 바꾸려는 경우
확인 모달을 띄워줘.

상황 1. 절대 보호 → 우선 보호로 변경하려는 경우

확인 모달 문구:
제목: 보호 수준 변경
내용:
“이 일정은 현재 절대 보호로 설정되어 있습니다.
우선 보호로 변경하면 스케줄 재생성 시 관리자 확인 후 변경될 수 있습니다.
보호 수준을 우선 보호로 변경하시겠습니까?”

버튼:
- 우선 보호로 변경
- 절대 보호 유지
- 취소

상황 2. 우선 보호 → 절대 보호로 변경하려는 경우

확인 모달 문구:
제목: 보호 수준 변경
내용:
“이 일정은 현재 우선 보호로 설정되어 있습니다.
절대 보호로 변경하면 스케줄 재생성 시 자동 변경 대상에서 제외됩니다.
절대 보호로 변경하시겠습니까?”

버튼:
- 절대 보호로 변경
- 우선 보호 유지
- 취소

상황 3. 이미 같은 보호 수준으로 선택된 셀을 다시 클릭하는 경우

동작:
- 선택 해제 여부를 물어보거나
- 바로 선택 해제

권장:
- 같은 보호 수준으로 다시 클릭하면 선택 해제
- 단, 이미 저장된 보호 일정이면 확인창 표시

확인 모달 문구:
제목: 보호 일정 해제
내용:
“이 일정은 보호 일정으로 저장되어 있습니다.
보호 설정을 해제하시겠습니까?”

버튼:
- 보호 해제
- 유지
- 취소

--------------------------------------------------
[저장 동작]
--------------------------------------------------

“보호 일정으로 저장” 버튼을 누르면
현재 선택된 보호 일정이 스케줄 재생성 화면의 보호 일정 카드에 반영되어야 함.

저장 후:
- 절대 보호 건수 갱신
- 우선 보호 건수 갱신
- 보호 일정 상세 목록 갱신
- 스케줄 재생성 시 해당 보호 수준 반영

예:
- 절대 보호 4건
- 우선 보호 7건

저장 완료 toast:
- 제목: 보호 일정 저장 완료
- 설명: 절대 보호 4건, 우선 보호 7건이 저장되었습니다.

--------------------------------------------------
[보호 수준 우선순위]
--------------------------------------------------

보호 수준은 아래 우선순위를 따름.

1) 절대 보호
2) 우선 보호
3) 일반 재배치 가능

자동 절대 보호 대상은 사용자가 임의로 우선 보호로 낮출 수 없거나,
낮출 수 있더라도 강한 확인창이 필요함.

자동 절대 보호 대상 예:
- 승인된 교육
- 승인된 병가
- 직원이 신청했고 승인 안내까지 나간 휴일
- 직원에게 승인 안내가 나간 연차
- 직원에게 승인 안내가 나간 SL
- 직원에게 승인 안내가 나간 HO
- 신혼여행
- 장기 병가
- 장기 교육
- 퇴사 이후 기간

이런 자동 절대 보호 일정은 기본적으로 절대 보호 유지가 원칙임.

선택 보호 수준:
