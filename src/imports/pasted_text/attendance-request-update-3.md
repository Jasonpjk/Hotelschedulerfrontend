중요:
- 이번 수정은 “근태 신청” 화면에만 적용해주세요.
- 다른 화면은 건드리지 말아주세요.
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 / 넓은 여백 / 얇은 선 / 정돈된 그리드 유지

[1. 주말 / 공휴 근무 희망 입력 위치 수정]
현재 주말 / 공휴일 근무 희망을 “쉬는 날 신규 신청” 안에서 같이 처리하는 구조를 수정해주세요.

수정 방향:
- 주말 근무 희망
- 공휴 근무 희망
은 쉬는 날 신규 신청 폼 안에서 입력하지 않게 해주세요.

즉,
쉬는 날 신청과
주말/공휴 근무 희망 표현은
서로 다른 기능으로 분리해주세요.

권장 방식:
- 근태 신청 화면 안에 별도 영역 또는 별도 카드 또는 별도 팝업 버튼 구성
- 예:
  - 쉬는 날 신규 신청
  - 주말/공휴 근무 희망 신청

중요:
- 사용자가 “쉬는 날 신청”을 하면서 동시에 주말/공휴 근무 희망을 고르는 구조로 보이면 안 됨
- 완전히 별도 입력 흐름처럼 보여야 합니다

[2. 주말 / 공휴 근무 희망도 저장이 아니라 신청으로 끝나야 함]
현재 주말 / 공휴 근무 희망 관련 입력이 단순 저장 느낌으로 끝나면 안 됩니다.

수정 방향:
- 주말 근무 희망 선택
- 공휴 근무 희망 선택
후에
“저장”이 아니라 “신청” 또는 “신청하기” 버튼으로 마무리되게 해주세요.

즉,
이 기능도 단순 개인 설정 저장이 아니라
근태 신청의 한 종류로 인식되게 해야 합니다.

권장 흐름:
1. 별도 버튼 클릭
2. 주말 근무 희망 / 공휴 근무 희망 선택
3. 적용 기간 또는 대상 월 확인
4. 신청하기 버튼 클릭
5. 신청 완료 상태 표시

중요:
- 사용자가 이걸 실제 신청으로 인식해야 함
- 저장만 하면 단순 환경설정처럼 보여서 안 됨

[3. 주말 / 공휴 근무 희망 신청 UI]
주말 / 공휴 근무 희망은 아래처럼 보이게 정리해주세요.

권장 구성 예시:
- 별도 섹션명: 주말/공휴 근무 희망
- 선택 항목:
  - 주말 근무 희망
  - 공휴 근무 희망
- 하단 버튼:
  - 취소
  - 신청하기

가능하면 보이면 좋은 보조 문구:
- 주말 또는 공휴일에 근무를 희망하는 경우 신청해주세요.
- 이 신청 내역은 자동 스케줄 및 공정성 계산에 반영됩니다.

[4. 신청 내역에도 주말 / 공휴 근무 희망이 신청으로 보여야 함]
주말 / 공휴 근무 희망도
근태 신청 내역 안에서 별도의 신청 건처럼 보여야 합니다.

예시:
- 신청 유형: 주말 근무 희망
- 신청 유형: 공휴 근무 희망

즉,
설정값처럼 숨지 말고
신청 이력처럼 남아야 합니다.

[5. 일괄 신청 내역 표시 방식 수정]
현재 여러 날짜를 일괄 신청하면
날짜가 각각 보이긴 하지만 같은 줄 안에 함께 들어가 있어서,
하나의 묶음처럼만 보여서 직관성이 떨어집니다.

이 부분을 수정해주세요.

원하는 방식:
- 일괄 신청했더라도
- 각 날짜가 완전히 독립적인 신청 내역처럼 보여야 합니다

즉,
예를 들어
4월 4일, 4월 12일을 일괄 신청했으면
한 줄에 “4월 4일, 4월 12일” 같이 묶여 보이지 말고

아래처럼 각각 별도 행으로 보여야 합니다.
- 4월 4일 / 쉬는 날 희망 / 상태 ...
- 4월 12일 / 쉬는 날 희망 / 상태 ...

중요:
- 직원이 봤을 때 “이 날짜도 하나의 신청 건이구나”라고 인식되어야 함
- 일괄 신청은 입력 편의 기능일 뿐, 내역에서는 각각 독립적인 신청 건처럼 보여야 합니다

[6. 일괄 신청 후 내역 분리 규칙]
일괄 신청으로 여러 날짜를 한 번에 제출하더라도,
저장/신청 후에는 시스템이 내부적으로 날짜별 개별 신청으로 분리한 것처럼 보여주세요.

즉,
입력 단계:
- 여러 날짜를 한 번에 선택 가능

신청 완료 후 내역:
- 날짜별로 각각 분리된 신청 건 생성
- 각 날짜별로 별도 상태, 별도 상세 보기 가능
- 각 날짜별로 최종 반영 결과 확인 가능

중요:
- 하나의 줄에 날짜만 여러 개 나열되면 안 됨
- 완전히 개별 신청으로 인식되어야 함

[7. 상세 / 수정 흐름도 날짜별 개별 신청 기준]
일괄 신청 후 분리된 각 날짜 건은
상세 보기와 수정도 각각 따로 가능하게 보이도록 정리해주세요.

예:
- 4월 4일 신청 건 상세
- 4월 12일 신청 건 상세

즉,
일괄 신청은 제출 방식만 묶여 있을 뿐,
내역/상세/수정은 날짜별 개별 단위로 보여야 합니다.

[8. 최종 요청]
현재 근태 신청 화면을 아래 기준으로 수정해주세요.

- 주말 / 공휴 근무 희망은 쉬는 날 신규 신청 안에서 같이 처리하지 않게 분리
- 주말 / 공휴 근무 희망은 별도 선택 영역 또는 별도 신청 흐름으로 구성
- 해당 기능은 저장이 아니라 “신청하기”로 마무리되게 수정
- 주말 / 공휴 근무 희망도 신청 내역에 신청 건처럼 남게 정리
- 여러 날짜를 일괄 신청하면 날짜들이 같은 줄에 묶여 보이지 않게 수정
- 일괄 신청 후에는 각 날짜가 완전히 독립적인 신청 내역으로 분리되어 보이게 정리
- 각 날짜 건은 별도 상태 / 상세 / 수정 흐름이 가능해 보이게 구성

핵심은,
주말/공휴 근무 희망은 별도 신청 기능으로 분리하고,
일괄 신청한 날짜들은 내역에서 각각 완전히 독립적인 신청 건처럼 보여야 한다는 점입니다.

핵심: 주말/공휴 근무 희망은 쉬는 날 신청과 분리된 별도 “신청” 기능으로 바꾸고, 일괄 신청한 날짜들은 내역에서 한 줄 묶음이 아니라 날짜별 개별 신청 건으로 나눠서 보여주세요.
