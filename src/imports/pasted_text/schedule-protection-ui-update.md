현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS의
“스케줄 재생성 > 근무표에서 보호 일정 선택” 모달에서
보호 해제 예정 셀의 가독성을 개선해줘.

이번 작업은 보호 일정 선택/해제 기능 자체를 바꾸는 것이 아니라,
사용자가 기존 보호 일정을 해제했을 때 그 셀이 현재 어떤 상태인지 명확히 보이도록 UI를 개선하는 작업이야.

--------------------------------------------------
[현재 문제]
--------------------------------------------------

현재 보호 일정 선택 모달에서 기존 보호 일정을 해제하면
해당 셀이 해제 예정 상태로 바뀌긴 하지만,
근무표 안에서 표시가 너무 약해서 알아보기 어렵습니다.

문제:
- 해제 예정 셀이 기존 보호 셀인지 일반 셀인지 구분이 어려움
- 셀 안의 근무 코드가 흐려지면서 가독성이 떨어짐
- 좌측 목록에는 “해제 예정”이라고 보이지만 우측 근무표에서는 한눈에 안 보임
- 여러 셀을 해제했을 때 어떤 셀이 해제 예정인지 확인하기 어려움

--------------------------------------------------
[수정 목표]
--------------------------------------------------

보호 해제 예정 셀은 일반 셀, 절대 보호 셀, 우선 보호 셀과 명확히 구분되어야 함.

해제 예정 상태는 아래처럼 보여줘.

- 셀 안 근무 코드는 계속 선명하게 유지
- 셀 테두리 또는 배경으로 “해제 예정” 상태 표시
- 작은 배지 또는 아이콘으로 “해제” 표시
- hover 시 해제 예정 설명 표시
- 좌측 목록과 우측 셀이 서로 연동되어 보이게 처리

--------------------------------------------------
[해제 예정 셀 스타일]
--------------------------------------------------

해제 예정 셀은 아래 스타일 중 조합해서 적용해줘.

권장 스타일:
- 붉은색 점선 테두리
- 셀 우측 상단 작은 X 아이콘
- 작은 텍스트 배지: “해제”
- 아주 연한 붉은 배경
- 기존 근무 코드 배지는 흐리게 하지 말고 그대로 선명하게 유지

중요:
- 근무 코드 M07, A13, N22, OFF, VAC, SL, EDU, SICK, REQ 등은 계속 읽기 쉬워야 함
- 해제 예정 표시가 근무 코드 배지를 덮으면 안 됨
- 너무 흐리게 처리하지 말 것
- 단순 opacity 낮추기만 사용하지 말 것

--------------------------------------------------
[상태별 셀 표시 기준]
--------------------------------------------------

보호 일정 선택 모달 안의 셀 상태를 아래처럼 구분해줘.

1) 절대 보호
- 자물쇠 아이콘
- 붉은 계열 또는 진한 테두리
- tooltip: 절대 보호

2) 우선 보호
- 방패 아이콘
- 골드 계열 또는 연한 테두리
- tooltip: 우선 보호

3) 신규 추가 예정
- 선택한 보호 수준에 따라 절대/우선 스타일 적용
- 신규 추가 배지 가능

4) 해제 예정
- X 아이콘 또는 “해제” 배지
- 붉은 점선 테두리
- 근무 코드는 선명하게 유지
- tooltip: 저장 시 보호 일정에서 해제됩니다.

5) 일반 셀
- 기존 월별 근무표 스타일 그대로 유지

--------------------------------------------------
[좌측 목록 표시 개선]
--------------------------------------------------

좌측 보호 일정 목록에서도 해제 예정 항목을 명확히 표시해줘.

현재처럼 항목이 흐리기만 하면 안 됨.

해제 예정 항목 표시:
- 항목 배경을 연한 붉은색으로 표시
- 상태 배지: 해제 예정
- 취소 버튼 또는 “해제 취소” 버튼 제공

예:
REQ 정예은 05-09 승인 안내 발송된 휴일 신청
상태: 해제 예정
버튼: 해제 취소

해제 취소를 누르면:
- 해당 항목이 다시 기존 보호 상태로 복구
- 우측 근무표 셀도 다시 절대 보호 또는 우선 보호 스타일로 복구
- 하단 해제 예정 건수 차감

--------------------------------------------------
[해제 취소 기능]
--------------------------------------------------

사용자가 실수로 보호 해제를 선택했을 수 있으므로
해제 예정 상태에서 다시 복구할 수 있어야 함.

동작:
1) 기존 보호 셀 클릭
2) 보호 해제 확인
3) 셀 상태가 해제 예정으로 변경
4) 사용자가 같은 셀을 다시 클릭하거나 좌측 목록에서 “해제 취소” 클릭
5) 기존 보호 상태로 복구

복구 시:
- 기존 protectionLevel 유지
- 기존 보호 사유 유지
- 해제 예정 목록에서 제거
- 카운트 갱신

--------------------------------------------------
[하단 카운트 표시 개선]
--------------------------------------------------

하단 카운트는 현재 상태를 더 명확히 보여줘.

예:
- 절대 보호 1건
- 우선 보호 5건
- 해제 예정 3건

저장 버튼 문구:
“보호 일정으로 저장 (절대 1건 / 우선 5건 / 해제 3건)”

해제 예정이 있을 경우 저장 버튼 근처에 안내 문구 추가:
“해제 예정 일정은 저장 후 재생성 대상에 포함될 수 있습니다.”

--------------------------------------------------
[tooltip 문구]
--------------------------------------------------

해제 예정 셀 hover 시 아래 문구를 보여줘.

제목:
해제 예정

내용:
“이 일정은 보호 해제 예정입니다.
저장하면 스케줄 재생성 시 변경 대상에 포함될 수 있습니다.”

버튼 또는 안내:
“다시 클릭하면 해제 취소”

--------------------------------------------------
[선택/해제 상태 우선순위]
--------------------------------------------------

한 셀에 여러 상태가 겹칠 경우 표시 우선순위를 아래처럼 정리해줘.

1) 해제 예정
2) 절대 보호
3) 우선 보호
4) 신규 선택
5) 일반 근무

즉, 기존 절대 보호였던 셀이라도 사용자가 해제 예정으로 바꿨다면
우측 근무표에서는 “해제 예정”이 가장 먼저 보이게 해줘.

다만 tooltip에는 원래 보호 수준도 함께 표시해줘.

예:
해제 예정
기존 보호 수준: 절대 보호
사유: 승인 안내 발송된 휴일 신청

--------------------------------------------------
[목록과 셀 연동]
--------------------------------------------------

좌측 목록에서 해제 예정 항목을 클릭하면
우측 근무표의 해당 셀로 스크롤 이동하고 셀을 짧게 강조해줘.

우측 근무표에서 해제 예정 셀을 클릭하면
좌측 목록의 해당 항목도 선택 상태로 표시해줘.

목록과 근무표가 따로 놀면 안 됨.

--------------------------------------------------
[저장 후 동작]
--------------------------------------------------

“보호 일정으로 저장”을 누르면 해제 예정 항목이 실제 보호 일정에서 제거되어야 함.

저장 후:
- 해제 예정 항목은 보호 목록에서 사라짐
- 일반 셀 스타일로 돌아감
- 스케줄 재생성 화면의 보호 일정 카운트 갱신
- toast 표시

toast 예:
제목: 보호 일정 저장 완료
설명: 절대 보호 1건, 우선 보호 5건, 해제 3건이 반영되었습니다.
