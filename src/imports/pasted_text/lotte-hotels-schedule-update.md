현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS에
“수요예측 재업로드 및 예외 상황 발생 시 변경 기준일·잠금·보호 대상 기반 재배치” 구조를 추가해줘.

이번 작업의 목적은
스케줄이 이미 배치된 이후 체크인/체크아웃 수요가 크게 바뀌거나
야간조 인원 이탈, 장기 병가, 신혼여행 같은 장기 부재가 발생했을 때
기존 근무표를 무리하게 전체 재생성하지 않고
변경 기준일 이후의 영향 구간만 운영정책에 맞게 재배치할 수 있도록 만드는 것이야.

중요:
- 이미 지나간 날짜는 절대 바꾸면 안 됨
- 변경 기준일 이전 날짜도 절대 바꾸면 안 됨
- 직원의 중요한 확정 부재 일정은 자동 재배치에서 보호되어야 함
- SL, 연차, HO는 절대 잠금이 아니라 “가능하면 보호해야 하는 대상”으로 분류해야 함
- 야간조는 해당 월에는 오전조/오후조/중간조와 섞이지 않는 월간 블록 원칙을 적용해야 함

--------------------------------------------------
[기능 1. 수요예측 재업로드 기반 영향 구간 재배치]
--------------------------------------------------

체크인/체크아웃 상황이 크게 바뀌었을 때
수요예측 파일을 다시 업로드하고
변경된 수요에 맞춰 근무표를 재배치할 수 있게 해줘.

단, 기존 수요예측 데이터를 단순 덮어쓰기하지 말고
새 수요예측 버전으로 저장해줘.

예:
- 기존 수요예측: Forecast v1.0
- 재업로드 수요예측: Forecast v1.1

재업로드 후에는 아래 정보를 분석해서 보여줘.

수요 변경 요약:
- 체크인 증가/감소
- 체크아웃 증가/감소
- 영향 날짜
- 영향 시간대
- 중간조 추가 필요 여부
- 기존 근무표 대비 부족 예상 인원
- 재배치 필요 여부

예시:
- 체크인 +42건
- 체크아웃 +18건
- 영향 날짜: 2026-05-02 ~ 2026-05-04
- 영향 시간대: 08:00~11:00 / 14:00~18:00
- C09, C10 추가 필요 가능성 있음

--------------------------------------------------
[기능 2. 변경 기준일 설정]
--------------------------------------------------

수요예측 재업로드 또는 예외 재배치 실행 시
반드시 “변경 기준일”을 설정하게 해줘.

변경 기준일의 의미:
- 변경 기준일 이전 날짜는 자동 재배치 대상에서 제외
- 오늘 이전 날짜는 항상 재배치 대상에서 제외
- 이미 근무가 완료된 날짜는 절대 변경 금지
- 변경 기준일 이후 날짜만 재배치 가능

예:
- 오늘: 2026-04-20
- 변경 기준일: 2026-04-21
- 2026-04-20 이전 날짜: 수정 불가
- 2026-04-21 이후 날짜: 재배치 가능 후보

UI:
- 변경 기준일 선택 date picker
- 기본값은 내일 또는 현재 날짜 이후 첫 운영일
- 변경 기준일 이전 날짜는 잠금 표시
- 재배치 미리보기에서 변경 기준일 이전 날짜가 제외되었음을 명확히 표시

--------------------------------------------------
[기능 3. 재배치 범위 선택]
--------------------------------------------------

수요예측 재업로드 후 재배치 범위를 관리자가 선택할 수 있게 해줘.

기본값은 “영향 날짜만” 또는 “영향 날짜 + 전후 1일”로 해줘.
월 전체 재생성은 기본값으로 두면 안 됨.

재배치 범위 옵션:
1) 영향 날짜만
2) 영향 날짜 + 전후 1일
3) 해당 주간
4) 7~14일 범위
5) 월 전체 재생성

중요:
- 월 전체 재생성은 최후의 수단이어야 함
- 월 전체 재생성을 선택하더라도 오늘 이전 날짜와 변경 기준일 이전 날짜는 절대 변경하면 안 됨
- 영향 범위를 넓히더라도 절대 잠금과 보호 대상 규칙은 유지해야 함

--------------------------------------------------
[기능 4. 잠금 단계 구조]
--------------------------------------------------

자동 재배치 시 모든 일정을 같은 기준으로 다루면 안 됨.
잠금 수준을 아래 3단계로 구분해줘.

1) 절대 잠금
2) 보호 대상
3) 일반 재배치 가능

--------------------------------------------------
[1단계. 절대 잠금]
--------------------------------------------------

절대 잠금은 자동 재배치가 어떤 경우에도 건드리면 안 되는 항목이야.

절대 잠금 대상:
- 오늘 이전 날짜
- 변경 기준일 이전 날짜
- 이미 확정 후 근무가 완료된 날짜
- 직원이 장기간 부재로 확정된 기간
  예: 신혼여행, 장기 병가, 출산 관련 휴가, 장기 교육, 퇴사 전후 정리 기간
- 관리자가 직접 “절대 잠금” 처리한 날짜/직원/기간
- 퇴사자의 퇴사일 이후 기간
- 장기 병가 승인 기간
- 장기 교육 승인 기간

절대 잠금 규칙:
- 자동 재배치 엔진은 절대 잠금 셀을 변경하면 안 됨
- 절대 잠금 구간에 근무를 배정하면 안 됨
- 절대 잠금 구간은 재배치 후보에서 제외
- 절대 잠금을 변경하려면 일반 재배치가 아니라 별도 관리자 잠금 해제 절차가 필요

--------------------------------------------------
[2단계. 보호 대상]
--------------------------------------------------

보호 대상은 가능하면 유지해야 하는 일정이야.
다만 정말 불가피한 경우 관리자의 명시적 승인 후 변경할 수 있게 해줘.

보호 대상:
- 승인된 연차
- SL 여성보건휴가
- HO 공휴일 휴무
- 승인된 일반 휴가
- 직원이 이미 받은 확정 공지 중 아직 근무 전인 날짜
- 직원 신청이 반영된 쉬는 날 희망
- 직원에게 공지된 근무 중 잠금 강도가 “보호”인 날짜

중요:
- SL, 연차, HO는 절대 잠금이 아니라 보호 대상임
- 자동 재배치 시 우선 보존해야 함
- 불가피하게 변경이 필요하면 “보호 대상 변경 필요”로 표시하고 관리자의 확인을 받아야 함
- 시스템이 임의로 SL, 연차, HO를 바꾸면 안 됨

보호 대상 변경 필요 예시:
- 김민준 5/3 HO를 변경해야 최소 인원 충족 가능
- 보호 대상 변경 필요
- 관리자 승인 후 변경 가능

--------------------------------------------------
[3단계. 일반 재배치 가능]
--------------------------------------------------

일반 재배치 가능 대상은 자동 재배치가 운영정책 범위 안에서 조정할 수 있는 항목이야.

일반 재배치 가능 대상:
- 아직 지나지 않은 일반 근무 코드
- M07 오전조
- A13 오후조
- N22 야간조
- C08/C09/C10/C11 중간조
- 일반 OFF
- 아직 직원에게 공지되지 않은 근무

단, 아래 운영정책은 반드시 지켜야 함:
- 최소 조별 인원
- 각 조별 인차지 1명 이상
- A13 다음날 M07 지양
- 5일/6일 이상 연속근무 경고
- 전월 근무강도 회복 반영
- 야간조 전환 OFF 정책
- 휴무 기준
- 공정성 기준

--------------------------------------------------
[기능 5. 개인 기간 잠금]
--------------------------------------------------

신혼여행, 장기 병가, 장기 교육, 퇴사 전후 정리 기간처럼
특정 직원이 일정 기간 동안 절대 근무할 수 없는 경우
직원별 기간 잠금 기능을 추가해줘.

직원별 기간 잠금 예:
- 직원명: 최유진
- 기간: 2026-05-01 ~ 2026-05-14
- 사유: 신혼여행
- 잠금 수준: 절대 잠금
- 자동 재배치 제외

직원별 기간 잠금 유형:
- 신혼여행
- 장기 병가
- 장기 교육
- 출산/육아 관련 장기 부재
- 퇴사 예정 기간
- 기타 관리자 지정

UI:
- 직원 상세 또는 근태 관리 상세에서 기간 잠금 설정 가능
- 근무표 셀에 잠금 아이콘 표시
- hover 시 잠금 사유와 기간 표시

표시 예:
- 절대 잠금
- 사유: 신혼여행
- 기간: 2026-05-01 ~ 2026-05-14
- 자동 재배치 제외

--------------------------------------------------
[기능 6. 셀 단위 잠금과 기간 단위 잠금]
--------------------------------------------------

잠금은 셀 단위와 기간 단위를 모두 지원해야 함.

1) 셀 단위 잠금
예:
- 김민준 / 2026-05-03 / HO / 보호 대상

2) 기간 단위 잠금
예:
- 최유진 / 2026-05-01 ~ 2026-05-14 / 신혼여행 / 절대 잠금

기간 단위 잠금을 걸면 해당 기간의 모든 날짜 셀이 자동으로 잠금 처리되어야 함.

--------------------------------------------------
[기능 7. 야간조 월간 블록 원칙]
--------------------------------------------------

야간조는 월 단위로 강하게 관리해야 함.

원칙:
- 야간조를 시작한 월에는 해당 직원이 오전조/오후조/중간조와 섞이면 안 됨
- 야간조 대상 직원은 해당 월에 N22 중심으로 배치
- OFF/휴식일은 가능
- M07 오전조 배정 금지
- A13 오후조 배정 금지
- C08/C09/C10/C11 중간조 배정 금지
- 단, 야간조가 종료되고 다음 달 전환 정책에 따라 일반조로 돌아가는 것은 별도 정책으로 처리

야간조 월간 블록 표시:
- 해당 직원에게 “야간조 블록” 배지 표시
- 근무표 셀 또는 직원명 옆에 야간조 블록 상태 표시 가능
- 자동 재배치 시 야간조 블록 원칙을 우선 확인

--------------------------------------------------
[기능 8. 야간조 예외 상황과 재설정 모드]
--------------------------------------------------

야간조 직원이 퇴사하거나 장기 병가, 장기 부재가 발생하면
기존 야간조 계획을 유지할 수 없으므로 “야간조 재설정 모드”가 필요함.

야간조 재설정이 필요한 상황:
- 야간조 대상자 퇴사
- 야간조 대상자 장기 병가
- 야간조 대상자 신혼여행 등 장기 부재
- 야간조 대상자가 해당 월 야간조 수행 불가
- 최소 야간 인원 유지 불가

야간조 재설정 흐름:
1) 야간조 이탈 감지
2) 야간조 재설정 필요 경고 표시
3) 변경 기준일 선택
4) 변경 기준일 이전 날짜 잠금
5) 이미 지나간 날짜 잠금
6) 장기 부재 직원의 해당 기간 절대 잠금
7) 변경 기준일 이후 남은 기간만 야간조 대상자 재선정
8) 연간 야간조 계획과 후보군 참고
9) 야간조 계획 재배치 시뮬레이션
10) 새 버전 저장

예:
- 기존 5월 야간조 대상자: 김민준, 박지호, 최유진
- 박지호 5월 12일부터 장기 병가
- 5월 1일~11일은 유지
- 5월 12일~31일만 야간조 재설정
- 박지호 5월 12일~31일은 SICK 절대 잠금
- 새로운 야간조 후보 투입
- 새 버전 저장

중요:
- 야간조 재설정도 변경 기준일 이전 날짜와 절대 잠금은 건드리면 안 됨
- 이미 지나간 야간조는 절대 변경 금지
- 야간조 재설정 결과도 기존 버전을 덮어쓰지 말고 새 버전으로 저장

--------------------------------------------------
[기능 9. 수요 변화 대응 시 중간조 우선 활용]
--------------------------------------------------

체크인/체크아웃 수요가 크게 바뀌었다고 해서
오전조/오후조/야간조 기본 배치를 무리하게 흔들면 안 됨.

수요 변화 대응 우선순위:
1) 오전조/오후조/야간조 기본 인원과 인차지 유지
2) 체크아웃 증가 시 C08, C09 추가 검토
3) 체크인 증가 시 C10, C11 추가 검토
4) 그래도 부족하면 M07/A13 인원 조정
5) 그래도 부족하면 일반 OFF 대체 투입 검토
6) 보호 대상 변경은 최후의 수단으로 관리자 승인 필요

중간조는 직원의 고정 그룹이 아니라
수요예측에 따라 자동 배정되는 근무 코드로 처리해야 함.

--------------------------------------------------
[기능 10. 재배치 시뮬레이션 및 미리보기]
--------------------------------------------------

재배치를 바로 저장하지 말고
먼저 시뮬레이션과 변경 전/후 미리보기를 보여줘.

미리보기 항목:
- 수요 변경 요약
- 변경 기준일
- 재배치 대상 기간
- 절대 잠금 제외 목록
- 보호 대상 보존 목록
- 보호 대상 변경 필요 목록
- 변경 예정 셀
- 추가 중간조
- 대체 투입 직원
- 남은 운영 경고
- 공정성 영향
- 새 버전명

예시:
- 변경 불가 영역:
  - 오늘 이전 날짜 8일
  - 변경 기준일 이전 날짜 3일
  - 절대 잠금 14일
  - 야간조 블록 보호 5건

- 보호 대상:
  - 연차 4건
  - SL 1건
  - HO 2건

- 변경 예정:
  - 일반 근무 6건
  - OFF 조정 2건
  - 중간조 추가 3건

- 보호 대상 변경 필요:
  - 없음

버튼:
- 새 버전으로 저장
- 재배치 범위 조정
- 관리자 수동 조정
- 취소

--------------------------------------------------
[기능 11. 새 버전 저장]
--------------------------------------------------

수요예측 재업로드나 야간조 재설정, 장기 부재 반영으로 근무표가 바뀌면
기존 버전을 덮어쓰지 말고 새 버전으로 저장해줘.

예:
- 기존 버전: v4.0 확정 근무표
- 수요변경 반영 후: v4.1 수요변경 반영 재배치
- 야간조 재설정 후: v4.2 야간조 재설정 반영
- 장기 부재 반영 후: v4.3 장기부재 반영 재배치

새 버전 설명에 포함할 내용:
- 생성 사유
- 기준 수요예측 버전
- 변경 기준일
- 재배치 기간
- 변경 셀 수
- 보호 대상 변경 여부
- 절대 잠금 제외 내역
- 남은 운영 경고
- 생성자
- 생성 시각

--------------------------------------------------
[기능 12. 잠금 표시 UI]
--------------------------------------------------

근무표 셀 또는 직원명 영역에 잠금 상태를 표시해줘.

권장 아이콘:
- 절대 잠금: 자물쇠 아이콘
- 보호 대상: 방패 아이콘
- 공지 잠금: 알림 아이콘
- 야간조 블록: 달 아이콘 또는 N 배지

hover 또는 tooltip 예:
- 절대 잠금
- 사유: 신혼여행
- 기간: 2026-05-01 ~ 2026-05-14
- 자동 재배치 제외

보호 대상 tooltip 예:
- 보호 대상
- 사유: 승인된 연차
- 자동 재배치 시 우선 보존되며 변경 시 관리자 승인이 필요합니다.

--------------------------------------------------
[기능 13. 재배치 엔진 우선순위]
--------------------------------------------------

자동 재배치 엔진은 아래 순서로 판단해야 함.

1) 절대 잠금 제외
2) 오늘 이전 날짜 제외
3) 변경 기준일 이전 날짜 제외
4) 이미 근무 완료된 날짜 제외
5) 야간조 월간 블록 원칙 확인
6) 승인된 연차/SL/HO/휴가 등 보호 대상 우선 보존
7) 일반 근무와 일반 OFF에서 먼저 대체 인력 탐색
8) 중간조 C08~C11로 수요 변화 우선 흡수
9) 그래도 불가능하면 보호 대상 변경 필요로 관리자에게 표시
10) 절대 잠금 변경이 필요해지는 경우 자동 재배치 불가 처리

--------------------------------------------------
[실패 또는 불가 상황 처리]
--------------------------------------------------

재배치가 불가능하면 억지로 이상한 스케줄을 만들지 말고
불가 사유와 선택지를 보여줘.

불가 사유 예:
- 대체 가능한 직원 부족
- 절대 잠금 구간 충돌
- 야간조 블록 원칙 위반
- 최소 인원 충족 불가
- 인차지 배치 불가
- 보호 대상 변경 없이는 재배치 불가

선택지:
- 재배치 범위 확대
- 보호 대상 변경 승인 검토
- 관리자 수동 조정
- 현재 경고 포함 새 버전 저장
- 취소

단:
- 절대 잠금은 어떤 선택에서도 자동 변경 불가
- 변경 기준일 이전 날짜와 오늘 이전 날짜는 자동 변경 불가

--------------------------------------------------
[대시보드 및 운영 경고 연동]
--------------------------------------------------

수요예측 재업로드와 재배치 결과는 대시보드와 운영 경고에 반영되어야 함.

연동 대상:
- 대시보드 수요 예측 상태
- 대시보드 운영 경고 요약
- 공정성 리포트
- 근무표 운영 경고 패널
- 근무표 버전 상태
- 마지막 반영 시각

운영 경고 예:
- 보호 대상 변경 필요
- 야간조 재설정 필요
- 수요 증가 대비 중간조 부족
- 절대 잠금으로 인해 자동 재배치 불가
- 변경 기준일 이전 날짜 제외됨
