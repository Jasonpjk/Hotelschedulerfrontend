현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS에서 전체화면보기와 연간 야간조 계획 기능이 다시 깨졌어.

이번 작업은 새 기능 추가가 아니라
“기존에 정상 동작하던 기능 복원 + 전체화면보기 연결 정상화 + 회귀 방지”가 목적이야.

절대 새로 구버전 화면을 만들거나,
기존 최신 화면을 전체화면보기의 예전 화면에 맞춰 롤백하지 마.

--------------------------------------------------
[현재 문제 1. 전체화면보기에서 최신 메인 화면이 반영되지 않음]
--------------------------------------------------

현재 메인 근무표 화면에는 아래 기능들이 있어야 함.
- 월별 근무표 / 연간 야간조 계획 탭
- 알림 발송
- 발송 이력
- 변경 이력
- 전월 근무강도 회복 결과
- 새 버전 생성
- 다운로드
- 저장
- 확정
- EDU 교육 범례
- SICK 병가 범례
- 기존 근무/휴무/조 그룹 범례 전체

그런데 오른쪽 위 “전체화면 보기 열기”를 누르면
최신 메인 화면과 동일한 화면이 아니라
예전 버전 또는 별도 화면이 열리고 있음.

수정 목표:
- 전체화면보기는 최신 메인 근무표 화면과 완전히 같은 상태와 같은 컴포넌트를 사용해야 함
- 전체화면보기 전용 old component, old toolbar, old legend, old mock data를 쓰면 안 됨
- 전체화면보기는 단지 레이아웃만 넓어지는 구조여야 함

반드시 동일해야 하는 항목:
- 선택 호텔
- 선택 월
- 선택 버전
- 활성 탭
- 작업 중 / 확정 상태
- 상단 버튼 구성
- 범례 구성
- 경고 개수
- 근무표 데이터
- EDU / SICK 범례
- 알림 발송 / 발송 이력
- 전월 근무강도 회복 결과
- 변경 이력
- 저장 / 확정
- AI 조정 영역

수정 방향:
- 전체화면보기용 별도 Schedule 컴포넌트가 있으면 사용하지 말고 최신 ScheduleView 또는 SchedulePageContainer를 재사용해줘
- fullscreen 여부는 mode 또는 layout prop으로만 구분해줘
- 예:
  - <ScheduleView mode="normal" />
  - <ScheduleView mode="fullscreen" />
- mode에 따라 기능이 빠지면 안 됨
- 폭, 높이, 여백, 스크롤 처리만 달라져야 함

절대 하지 말 것:
- 전체화면보기 예전 UI를 메인 화면에 덮어씌우지 말 것
- 메인 화면을 예전 버전으로 롤백하지 말 것
- 전체화면보기 수정 과정에서 버튼/범례/탭을 삭제하지 말 것
- “전체화면보기는 아직 미구현”처럼 처리하지 말 것

--------------------------------------------------
[현재 문제 2. 연간 야간조 계획 기능이 사라지고 준비 중 화면으로 바뀜]
--------------------------------------------------

현재 “연간 야간조 계획” 탭을 누르면
“연간 야간조 계획 기능은 준비 중입니다” 같은 placeholder 화면만 보임.

하지만 이 기능은 이미 이전에 구현되어 있었어.
따라서 준비 중 화면으로 대체하면 안 되고,
기존 연간 야간조 계획 기능을 복원해야 해.

복원해야 하는 기존 기능:
- 연도 선택
- 상태 표시: 작업 중 / 확정
- 변경 이력 버튼
- 저장 버튼
- 확정 버튼
- 1월~12월 월별 카드 또는 월별 계획 영역
- 각 월별 목표 인원
- 계획 인원
- 우선 대상자 수
- 예비 대상자 수
- 제외 대상자 수
- 부족 표시
- 월 선택 시 대상자 선택 팝업 또는 패널
- 직원 검색
- 직원 선택
- 우선 대상자 / 예비 대상자 / 제외 대상자 관리
- 휴가 충돌 확인
- 저장 시 우측 하단 toast
- 변경 이력 UI

중요:
- “준비 중입니다” placeholder를 제거해줘
- 기존 구현된 연간 야간조 계획 화면을 다시 연결해줘
- 만약 컴포넌트가 남아 있는데 렌더링만 안 되는 상태라면 해당 컴포넌트를 다시 연결해줘
- 만약 activeTab 조건에서 잘못 빠졌다면 조건을 복구해줘
- 만약 placeholder로 교체되어 있다면 기존 NightPlan / AnnualNightShiftPlan 관련 컴포넌트를 찾아 다시 사용해줘

--------------------------------------------------
[연간 야간조 계획 화면에서 유지되어야 하는 UI]
--------------------------------------------------

연간 야간조 계획 탭 상단:
- 연도 선택 드롭다운
- 상태 표시: 작업 중 / 확정
- 변경 이력
- 저장
- 확정

중복 호텔 드롭다운은 두지 말 것.
호텔 기준은 상단 글로벌 호텔 선택값만 사용.

월별 카드:
- 1월~12월 표시
- 목표 인원
- 계획 인원
- 우선 대상자 수
- 예비 대상자 수
- 제외 대상자 수
- 부족 여부

월 선택 팝업:
- 직원 검색
- 직원 리스트
- 이름 + 직급만 표시
- 사번, 역할, 조 그룹 등 불필요한 메타 정보는 표시하지 말 것
- 저장 / 취소 버튼 유지

변경 이력:
- 월별 근무표 변경 이력 디자인과 동일한 패턴 사용
- 연간 야간조 계획 전용 이력창이 바로 열려야 함
- 월별 근무표 탭으로 이동했을 때 뒤늦게 뜨면 안 됨

저장 toast:
- 월별 근무표 저장 완료 toast와 동일한 우측 하단 toast 사용
- 제목: 저장 완료
- 설명: 연간 야간조 계획이 저장되었습니다.

[원인 점검 요청]
--------------------------------------------------

수정 전에 아래를 먼저 확인하고 고쳐줘.

1) 전체화면보기 버튼이 어떤 컴포넌트를 여는지 확인
- old fullscreen component를 쓰고 있으면 최신 ScheduleView로 교체

2) 연간 야간조 계획 탭이 어떤 컴포넌트를 렌더링하는지 확인
- placeholder 화면을 렌더링하고 있으면 제거
- 기존 AnnualNightShiftPlan / NightShiftPlan 관련 컴포넌트로 복구

3) activeTab 상태가 정상인지 확인
- activeTab === "monthly"이면 월별 근무표
- activeTab === "annualNightPlan"이면 연간 야간조 계획
- 탭 이동 시 다른 모달이나 이력이 뒤늦게 열리지 않도록 side effect 제거

4) toolbar / legend / tabs가 hardcoded old array로 덮어써지는지 확인
- 있다면 공통 최신 배열로 통합

5) 전체화면보기와 메인 화면이 같은 store/context를 구독하는지 확인
- 호텔, 월, 버전, 활성 탭, 경고, 범례, 버튼, 데이터가 분리되어 있으면 통합

[디자인 통일성 원칙]

특히:

- 전체화면보기는 메인 화면과 동일한 컴포넌트 사용

--------------------------------------------------
[최종 목표]
--------------------------------------------------

이번 작업의 최종 목표는 아래야.

1) 메인 근무표 화면은 최신 기능이 유지된 상태로 복구
2) 전체화면보기는 메인 화면의 최신 기능과 동일하게 표시
3) 연간 야간조 계획은 준비 중 placeholder가 아니라 기존 실제 기능으로 복원
4) 수정 과정에서 기존 기능이 다시 사라지지 않도록 회귀 점검
5) 같은 기능은 기존 디자인 패턴과 통일

전체화면보기의 예전 화면을 기준으로 삼지 말고,
반드시 최신 메인 근무표 화면을 기준으로 전체화면보기를 맞춰줘.
연간 야간조 계획도 placeholder가 아니라 기존 실제 기능을 복원해줘.
