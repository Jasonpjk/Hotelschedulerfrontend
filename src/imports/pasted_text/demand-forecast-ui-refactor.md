현재 호텔 프런트 근무표 SaaS의 “수요 예측” 화면 구조를 아래 기준으로 다시 정리해주세요.

이번 수정은 단순 스타일 변경이 아니라
탭 구조, 데이터 입력 흐름, 불필요 정보 제거, 화면 통합을 포함한 UX 재구성입니다.

[1. 탭 순서 변경]
현재 탭:
- 예측 결과
- 예측 근거
- 이벤트 보정
- 최종 반영값

이 순서를 아래처럼 변경해주세요.
새 탭 순서:
1) 예측 근거
2) 이벤트 보정
3) 예측 결과

중요:
- “예측 근거”가 가장 먼저 와야 함
- “이벤트 보정”은 그 다음
- “예측 결과”는 마지막에 배치
- “최종 반영값” 탭은 완전히 삭제

[2. 예측 근거 탭 기능 재구성]
예측 근거 탭은 단순 설명 탭이 아니라
실제 데이터를 넣고 예측을 시작하는 시작 화면 역할을 해야 합니다.

반드시 들어가야 할 기능:
- 엑셀 업로드 영역
- 업로드된 파일 상태 표시
- 적용 버튼

원하는 흐름:
1. 사용자가 예측 근거 탭에 들어옴
2. 엑셀 파일을 업로드함
3. 적용 버튼을 누름
4. 그 결과가 바탕이 되어 아래의 예측 결과가 생성됨

즉,
“예측 근거 → 데이터 업로드 및 적용 → 예측 결과 생성”
이 흐름이 화면상으로 명확하게 보여야 합니다.

예측 근거 탭 안에 들어갈 요소 예시:
- 올해 현재 예약 데이터 업로드
- 전년도 실적 데이터 업로드
- 필요 시 공휴일/이벤트 기준 데이터 업로드
- 적용 버튼
- 업로드 완료 상태 / 반영 일시 표시

디자인 방향:
- 파일 업로드 UI는 관리자용 SaaS답게 정돈되고 실무적으로 보이게
- 과한 일러스트나 소비자용 업로드 박스 느낌 금지
- 표, 상태 텍스트, 파일명 표시 등 업무형 인터페이스로 구성

[3. 이벤트 보정 탭 위치 유지 및 역할 명확화]
이벤트 보정 탭은 예측 근거 다음 단계로 유지해주세요.

역할:
- 예측 근거로 생성된 기본 예측값에 대해
- 지역 행사, 단체 예약, 연휴 효과, BTS 공연 같은 특수 이벤트를
- 수동으로 보정하는 단계

즉,
예측 근거에서 데이터 적용 후
이벤트 보정에서 추가 조정하고
그 최종 반영된 값이 예측 결과에 나타나는 흐름으로 정리해주세요.

[4. 예측 결과 탭 재구성]
예측 결과 탭은 현재의 예측 결과 + 최종 반영값 기능을 합친 형태로 바꿔주세요.

즉,
기존 “최종 반영값” 탭은 삭제하고
그 안에 있던 핵심 기능만 예측 결과 탭 하단으로 이동해주세요.

예측 결과 탭에 유지할 것:
- 날짜별 예측 결과 테이블
- 예상 체크인 / 예상 체크아웃
- 이벤트 보정 반영 여부
- 피크 여부
- 인력 위험 표시

예측 결과 탭에서 삭제할 것:
- 신뢰도 컬럼
- 신뢰도 관련 태그/배지/텍스트

중요:
- 예측 결과와 예측 근거에 있는 “신뢰도” 관련 정보는 모두 삭제
- 화면 어디에서도 신뢰도 점수, 신뢰 높음/중간/낮음 등의 표현이 나오지 않게 수정

[5. 오른쪽 패널 제거]
현재 예측 결과 화면 오른쪽에 있는 “근무표 연결 요약” 영역의 내용은 전부 삭제해주세요.

삭제 대상:
- 근무표 연결 요약
- 피크 체크인일 요약
- 피크 체크아웃일 요약
- 중간조 보완 필요일
- 기타 우측 보조 요약 패널 전체

중요:
- 예측 결과 화면은 좌우 분할 구조가 아니라
- 메인 콘텐츠 중심의 넓은 단일 작업 영역으로 바꿔주세요
- 불필요한 오른쪽 보조 패널 없이
- 표와 결과 중심으로 더 넓고 실무적으로 보이게 수정

[6. 최종 반영값 탭 삭제 + 기능 이동]
현재 “최종 반영값” 탭은 완전히 삭제해주세요.

대신 기존 최종 반영값 탭 하단에 있던 아래 기능만
예측 결과 탭 하단으로 이동해주세요.

남길 기능:
- “이 예측값을 4월 근무표 생성에 반영합니다” 안내 영역
- AI 권장안 조정 버튼
- 근무표 생성 반영 확정 버튼

즉,
예측 결과를 검토한 뒤
화면 하단에서 바로
- AI 권장안 조정
- 근무표 생성 반영 확정
이 가능해야 합니다.

이 기능은 별도 탭으로 분리하지 말고
예측 결과 화면의 마지막 액션 영역으로 자연스럽게 붙여주세요.

[7. 최종 구조 정리]
수정 후 최종 탭 구조는 아래처럼 정리해주세요.

1) 예측 근거
- 엑셀 업로드
- 적용 버튼
- 예측 생성 시작

2) 이벤트 보정
- 특수 이벤트 수동 보정

3) 예측 결과
- 예측 결과 테이블
- 이벤트 반영 결과 확인
- 화면 하단에서 AI 권장안 조정 / 근무표 생성 반영 확정

즉,
입력 → 보정 → 결과 및 반영
흐름이 명확히 보이도록 다시 설계해주세요.

[8. 유지할 톤앤매너]
- LOTTE HOTELS & RESORTS 관리자 SaaS 톤 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 유지
- 관리자용 실무 시스템답게 정제된 레이아웃
- 넓은 여백, 얇은 선, 정돈된 그리드 유지
- 과한 카드형 정보 남발 금지
- 소비자용 대시보드 느낌 금지

[9. 중요한 수정 의도]
이번 수정의 목적은 다음과 같습니다.

- 사용 흐름을 더 자연스럽게 만들기
- 예측 근거에서 데이터를 넣고 바로 결과로 이어지는 구조 만들기
- 불필요한 요약 패널 제거
- 신뢰도 같은 불필요한 지표 제거
- 최종반영값 탭을 없애고 예측결과에서 바로 액션하도록 단순화

최종 요청:
현재 수요 예측 화면을
“예측 근거 → 이벤트 보정 → 예측 결과”
3단계 구조로 다시 정리하고

- 예측 근거 탭에서 엑셀 업로드 + 적용
- 예측 결과 화면에서 우측 요약 패널 제거
- 신뢰도 삭제
- 최종반영값 탭 삭제
- 하단 반영 액션 영역은 예측 결과로 이동

이 기준으로 UI를 수정해주세요.
