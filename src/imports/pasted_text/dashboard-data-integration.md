현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS에서
대시보드에 표시되는 데이터가 근무표, 직원 관리, 근태 신청, 근태 관리, 수요 예측, 호텔 설정 데이터와 실제로 연결되도록 수정해줘.

이번 작업의 목적은 대시보드가 단순 mock 화면이나 고정 숫자 화면이 아니라
각 메뉴의 실제 상태를 읽어서 보여주는 운영 허브가 되도록 만드는 것이야.

--------------------------------------------------
[현재 문제]
--------------------------------------------------

현재 대시보드에 아래 항목들이 표시되고 있지만
근무표, 직원 관리, 근태 신청, 수요 예측 등 실제 메뉴의 데이터와 연결되어 있는지 불명확함.

대시보드 표시 항목 예:
- 선택 호텔
- 대상 월
- 근무표 버전
- 근무표 상태
- 직원 기준
- 수요 예측
- 마지막 반영
- 오늘 체크인
- 오늘 체크아웃
- 리스크 일수
- 근태 검토
- 미반영 직원
- 승인 대기
- 운영 진행 단계
- 운영 경고 요약
- 공정성 리포트

이 값들이 각 메뉴의 실제 상태와 다르게 보이면 안 됨.
대시보드는 모든 운영 데이터를 요약해서 보여주는 화면이어야 함.

--------------------------------------------------
[수정 목표]
--------------------------------------------------

대시보드의 모든 주요 값이 실제 화면/상태와 연결되도록 해줘.

연결 대상:
1) 근무표
2) 직원 관리
3) 근태 신청
4) 근태 관리
5) 수요 예측
6) 호텔 설정
7) 연간 야간조 계획
8) 공정성 리포트
9) 운영 경고

대시보드는 별도 mock 데이터를 사용하지 말고
각 메뉴에서 사용하는 공통 상태, store, context, service, data source를 읽어오도록 정리해줘.

--------------------------------------------------
[핵심 원칙]
--------------------------------------------------

- 대시보드 전용 하드코딩 숫자 금지
- 대시보드 전용 mock 데이터 금지
- 각 메뉴와 다른 별도 local state 사용 금지
- 선택 호텔, 대상 월, 버전, 상태는 전역 상태를 기준으로 통일
- 각 메뉴에서 값이 바뀌면 대시보드도 같은 값으로 즉시 반영
- 대시보드에서 “전체보기”, “검토”, “근무표에서 보기” 등을 누르면 실제 해당 메뉴로 이동

--------------------------------------------------
[전역 기준값 연결]
--------------------------------------------------

대시보드는 아래 전역 상태를 기준으로 모든 데이터를 조회해야 함.

1) 선택 호텔
- 상단 글로벌 호텔 선택값 사용
- 대시보드 내부에서 별도 호텔 상태를 만들지 말 것
- 근무표, 직원 관리, 근태 신청, 근태 관리, 수요 예측 모두 같은 selectedHotel 기준 사용

2) 대상 월
- 현재 운영 대상 월 사용
- 근무표의 선택 월과 대시보드 대상 월이 다르면 안 됨
- 대시보드에서 대상 월을 바꾸는 구조가 있다면 근무표와 같은 상태를 공유

3) 근무표 버전
- 근무표 화면의 현재 선택 버전과 동일해야 함
- 예: v3.1, v4.0
- 근무표에서 버전 변경 시 대시보드도 같은 버전으로 반영

4) 근무표 상태
- 근무표의 작업 중 / 확정 상태와 동일해야 함
- 근무표에서 확정하면 대시보드도 “확정”으로 바뀌어야 함
- 근무표에서 다시 편집 상태가 되면 대시보드도 반영

--------------------------------------------------
[대시보드 상단 상태값 실제 연결]
--------------------------------------------------

대시보드 상단 상태값을 아래 기준으로 연결해줘.

1) 선택 호텔
- globalHotelContext.selectedHotel

2) 대상 월
- scheduleState.selectedMonth 또는 currentOperationMonth

3) 근무표 버전
- scheduleVersionState.currentVersion

4) 근무표 상태
- scheduleStatus
- 값: 작업 중 / 확정

5) 직원 기준
- 직원 관리 화면의 전체 확정 상태와 연결
- 직원 정보가 확정되어 있으면 “확정 완료”
- 변경사항이 있거나 미확정이면 “확정 필요”

6) 수요 예측
- 수요 예측 화면의 예측 결과 반영 확정 여부와 연결
- 예측 결과가 반영 확정되었으면 “반영 완료”
- 아직 검토 전이면 “검토 필요”

7) 마지막 반영
- 근무표, 직원 기준, 수요 예측, 근태 반영 중 가장 최근 업데이트 시각 표시
- 예: 2026.03.26 14:32

--------------------------------------------------
[대시보드 요약 카드 실제 연결]
--------------------------------------------------

대시보드 요약 카드 값을 실제 데이터와 연결해줘.

1) 오늘 체크인
- 수요 예측 또는 예약/예측 데이터의 해당 날짜 체크인 수
- selectedHotel + selectedDate 기준
- 없으면 0 또는 “데이터 없음” 표시

2) 오늘 체크아웃
- 수요 예측 또는 예약/예측 데이터의 해당 날짜 체크아웃 수
- selectedHotel + selectedDate 기준

3) 리스크 일수
- 근무표 운영 경고 중 위험 또는 즉시 수정 필요 날짜 수
- 예: 최소 인원 부족, 인차지 부족, 야간조 전환 문제 등
- warningService 또는 scheduleWarnings 기준

4) 근태 검토
- 근태 신청/근태 관리에서 관리자가 검토해야 하는 건수
- 승인 대기, 자동 반영 실패, 충돌 발생 건 포함

5) 미반영 직원
- 근태 신청이 있었지만 근무표에 반영되지 않은 직원 수
- 중복 직원은 1명으로 계산
- 신청 건수와 직원 수를 혼동하지 말 것

6) 승인 대기
- 근태 신청 중 status가 승인 대기 또는 신청 접수 상태인 건수
- 교육, 병가, 쉬는 날 희망, 주말/공휴 근무 희망 포함

--------------------------------------------------
[운영 진행 단계 실제 연결]
--------------------------------------------------

대시보드의 운영 진행 단계는 실제 상태와 연결되어야 함.

예시 단계:
1) 직원 관리 기준 확정
2) 수요 예측 결과 확정
3) 근태 신청 자동 반영 검토
4) 근무표 최종 확정

각 단계 연결 기준:
- 직원 관리 기준 확정:
  직원 관리 화면의 전체 확정 상태
- 수요 예측 결과 확정:
  수요 예측 화면의 예측 결과 반영 확정 상태
- 근태 신청 자동 반영 검토:
  근태 관리 화면의 미검토/충돌/미반영 건수
- 근무표 최종 확정:
  근무표 화면의 현재 버전 확정 상태

상태값:
- 완료
- 검토 필요
- 진행 중
- 미완료

각 단계의 버튼은 실제 메뉴로 이동해야 함.
예:
- 직원 기준 확정 → 직원 관리 화면
- 수요 예측 결과 확정 → 수요 예측 화면
- 근태 신청 자동 반영 검토 → 근태 관리 화면
- 근무표 최종 확정 → 근무표 화면

[로딩 / 빈 상태 / 오류 상태]
