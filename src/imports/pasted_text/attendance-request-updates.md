현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS의 “근태 신청” 화면에서 아래 기능을 수정 및 추가해줘.

[현재 문제]
1) 이전에 요청한 교육 / 병가 신청 기능이 근태 신청 화면에 실제로 구현되지 않았음
- 현재 보이는 신청 버튼은
  - + 쉬는 날 희망 신청
  - + 주말/공휴 근무 희망 신청
  만 있음
- 교육 신청, 병가 신청으로 들어가는 버튼이나 모달이 없음

2) 주말/공휴 근무 희망 신청은 “신청”만 있고 “취소”가 없음
- 이미 신청된 상태에서는 disabled 표시만 되고
  사용자가 본인 신청을 취소할 수 없음
- 이건 UX가 막혀 있음

[수정 목표]
- 근태 신청 화면에서 직원이 직접
  - 교육 신청
  - 병가 신청
  - 주말 근무 희망 신청 취소
  - 공휴일 근무 희망 신청 취소
  를 할 수 있게 해줘

--------------------------------------------------
[수정 1. 근태 신청 화면에 교육 / 병가 신청 버튼 추가]
--------------------------------------------------

[버튼 영역 수정]
현재 “일반 근태 신청” 탭 상단 버튼 영역에 아래 버튼을 추가해줘.

기존:
- + 쉬는 날 희망 신청
- + 주말/공휴 근무 희망 신청

추가:
- + 교육 신청
- + 병가 신청

배치 원칙:
- 기존 버튼 디자인 톤과 동일
- 너무 한 줄이 답답하면 2줄 배치 가능
- 버튼 우선순위는 아래처럼 정리 권장
  1열: + 쉬는 날 희망 신청 / + 주말·공휴 근무 희망 신청
  2열: + 교육 신청 / + 병가 신청

[버튼 클릭 동작]
- + 교육 신청 클릭 시 교육 신청 모달 열기
- + 병가 신청 클릭 시 병가 신청 모달 열기
- 임시 alert, 준비중 메시지, dead-end 동작 금지
- 실제 입력 가능한 모달 또는 패널이 열려야 함

--------------------------------------------------
[수정 2. 교육 신청 모달 추가]
--------------------------------------------------

[모달 제목]
- 교육 신청

[모달 설명]
- 사내/사외 교육 일정을 신청해주세요.

[입력 항목]
- 적용 기간
  - 단일 날짜 또는 기간 선택 가능
- 종일 여부 / 시간 입력
  - 종일
  - 시간 지정
- 교육 구분
  - 사내 교육
  - 사외 교육
  - 법정 교육
  - 기타
- 교육명
- 신청 사유 또는 메모
- 첨부파일 영역
  - 아직 완전 구현이 아니더라도 업로드 UI 자리 마련
  - “첨부 예정” 또는 파일 선택 placeholder 가능

[제출 버튼]
- 신청하기
- 취소

[제출 후]
- 신청 성공 시 우측 하단 toast
  - 제목: 신청 완료
  - 설명: 교육 신청이 접수되었습니다.
- 목록에 새 신청 건이 추가되어 보여야 함
- 신청 유형은 “교육”으로 표시

--------------------------------------------------
[수정 3. 병가 신청 모달 추가]
--------------------------------------------------

[모달 제목]
- 병가 신청

[모달 설명]
- 병가 일정을 신청해주세요.

[입력 항목]
- 적용 기간
  - 단일 날짜 또는 기간 선택 가능
- 종일 / 반일 선택
- 신청 사유 또는 메모
- 증빙 첨부 영역
  - 선택 또는 placeholder 형태로 먼저 구현 가능

[제출 버튼]
- 신청하기
- 취소

[제출 후]
- 신청 성공 시 우측 하단 toast
  - 제목: 신청 완료
  - 설명: 병가 신청이 접수되었습니다.
- 목록에 새 신청 건이 추가되어 보여야 함
- 신청 유형은 “병가”로 표시

--------------------------------------------------
[수정 4. 주말/공휴 근무 희망 신청 취소 기능 추가]
--------------------------------------------------

[현재 문제]
- 현재 주말/공휴 근무 희망 신청 모달에서는
  이미 신청된 항목이 “이미 신청됨”으로 비활성화만 되어 있음
- 사용자가 직접 취소할 수 없음

[수정 요구]
- 이미 신청된 주말 근무 희망 / 공휴일 근무 희망은
  모달 안에서 직접 취소할 수 있게 해줘

[모달 동작 방식]
현재 “주말/공휴 근무 희망 신청” 모달 안에서 각 항목을 아래 상태로 구분해줘.

상태 1) 신청 가능
- 체크 가능
- 버튼: 신청하기

상태 2) 이미 신청됨
- 회색 비활성처럼 막아두지 말고
- 선택된 상태 표시 + 취소 가능하게 변경
- 버튼 또는 액션 문구 예:
  - 신청 취소
  - 또는 토글 해제 가능

권장 방식:
- 주말 근무 희망 / 공휴일 근무 희망 각각을 토글 카드처럼 만들고
- 신청된 항목은 “신청됨” 배지와 함께 해제 가능하게 처리

예시:
- 주말 근무 희망 [신청됨]
  → 클릭 시 신청 취소 가능
- 공휴일 근무 희망 [신청됨]
  → 클릭 시 신청 취소 가능

[취소 확인]
- 너무 무거운 confirm 모달까지는 필요 없고
- 취소 시 바로 처리 후 우측 하단 toast 노출

토스트 예시:
- 제목: 신청 취소 완료
- 설명: 주말 근무 희망 신청이 취소되었습니다.
또는
- 제목: 신청 취소 완료
- 설명: 공휴일 근무 희망 신청이 취소되었습니다.

--------------------------------------------------
[수정 5. 신청 내역 목록 반영]
--------------------------------------------------

근태 신청 화면의 “나의 신청 내역” 목록에 아래 유형이 모두 보여야 함.

신청 유형 확장:
- 쉬는 날 희망
- 주말 근무 희망
- 공휴일 근무 희망
- 교육
- 병가

목록 컬럼에서 자연스럽게 보이도록 수정:
- 날짜
- 신청 의사
- 최종 반영
- 신청 사유
- 반영 조정 사유
- 상태
- 신청일
- 작업

[표시 규칙]
- 교육 신청은 “교육”
- 병가 신청은 “병가”
- 주말 근무 희망은 “주말 근무 희망”
- 공휴일 근무 희망은 “공휴일 근무 희망”
으로 명확히 표시

[작업 컬럼]
- 아직 승인 전 / 접수 상태이면 직원이 취소 가능
- 상세 버튼만 두지 말고 필요 시 취소 버튼도 함께 표시
- 단, 이미 자동 반영 완료 / 승인 완료 등 취소 불가 상태면 비활성 또는 상세만 유지

--------------------------------------------------
[수정 6. 필터 반영]
--------------------------------------------------

우측 필터:
- 전체
- 내 신청
- 시스템 배정
기존 구조는 유지하되,
내 신청 목록에 교육 / 병가 / 주말근무희망 / 공휴일근무희망도 정상 포함되게 해줘.

추가 가능하면 신청 유형 필터도 고려:
- 전체
- 쉬는 날 희망
- 주말/공휴 근무 희망
- 교육
- 병가

--------------------------------------------------
[수정 7. 상태값 정리]
--------------------------------------------------

직원 화면에서 보이는 상태값을 아래처럼 정리해줘.

예시 상태:
- 신청 접수
- 승인 대기
- 승인 완료
- 자동 반영 완료
- 반려
- 신청 취소

중요:
- 직원이 직접 취소한 경우 목록에 기록이 남아야 함
- 완전히 사라지지 말고 “신청 취소” 상태로 남는 것이 좋음

--------------------------------------------------
[수정 8. 근무표 / 근태관리 연동 전제]
--------------------------------------------------

이번 단계에서는 직원 신청 UI가 우선이지만,
구조는 이후 근태 관리 / 근무표와 자연스럽게 연결되도록 해줘.

연동 전제:
- 교육 신청 승인 시 근무표 코드 EDU 반영 가능
- 병가 신청 승인 시 근무표 코드 SICK 반영 가능
- 주말/공휴 근무 희망은 자동 스케줄 생성과 공정성 계산에 반영 가능
- 직원이 취소하면 이후 자동 스케줄 계산에서도 제외되어야 함

--------------------------------------------------
[토스트 규칙]
--------------------------------------------------

근태 신청 화면도 다른 화면과 동일하게
우측 하단 toast 시스템으로 통일해줘.

예시:
1) 교육 신청 완료
- 제목: 신청 완료
- 설명: 교육 신청이 접수되었습니다.

2) 병가 신청 완료
- 제목: 신청 완료
- 설명: 병가 신청이 접수되었습니다.

3) 주말 근무 희망 신청 완료
- 제목: 신청 완료
- 설명: 주말 근무 희망이 접수되었습니다.

4) 공휴일 근무 희망 신청 완료
- 제목: 신청 완료
- 설명: 공휴일 근무 희망이 접수되었습니다.

5) 신청 취소 완료
- 제목: 신청 취소 완료
- 설명: 선택한 신청이 취소되었습니다.

6) 변경사항 없음 또는 중복 신청
- 제목: 처리할 내용이 없습니다
- 설명: 이미 같은 신청이 반영되어 있습니다.

--------------------------------------------------
[중요]
--------------------------------------------------

- 교육 / 병가 신청은 실제 버튼과 모달로 구현되어야 함
- 주말/공휴 근무 희망 신청은 “신청만 가능”이 아니라 “취소도 가능”해야 함
- 이미 신청됨 상태를 단순 비활성으로 막아두지 말고 취소 가능 상태로 바꿔줘
- 임시 알림이나 준비중 메시지 금지
- 실제 입력 가능한 UI와 실제 목록 반영까지 보여줘
- LOTTE HOTELS & RESORTS 관리자 SaaS 톤앤매너 유지
- 월별 근무표와 동일한 우측 하단 toast 시스템 사용

이번 수정은 디자인보다 기능 완성이 우선이야.
특히 아래 3가지는 반드시 해결해줘.
1) 교육 신청 구현
2) 병가 신청 구현
3) 주말/공휴 근무 희망 신청 취소 구현
