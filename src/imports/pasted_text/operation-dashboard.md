현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 기존 톤앤매너를 유지한 상태로
“대시보드” 화면을 운영 관제형 홈 화면으로 재설계해주세요.

중요:
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 유지
- 넓은 여백, 얇은 선, 정돈된 그리드, 실무형 UI 유지
- 일반 BI 대시보드처럼 차트 중심으로 가지 말 것
- 소비자용 예약 사이트 느낌 금지
- 대시보드는 “예쁜 숫자판”이 아니라 “운영 판단과 즉시 액션을 위한 관제형 홈 화면”처럼 보여야 함

[1. 대시보드의 핵심 목적]
이 시스템의 대시보드는 아래 질문에 바로 답해야 합니다.

- 이번 달 근무표가 작업 중인지 확정됐는지
- 오늘/이번 주에 인력 리스크가 있는지
- 근태 신청 중 자동 반영 안 된 건이 얼마나 있는지
- 수요 예측이 최신 상태인지
- 직원 관리 / 호텔 설정 / 근태 반영 / 근무표 확정 중 무엇이 아직 미완료인지

즉,
대시보드는 단순 요약 화면이 아니라
“현재 운영 상태를 빠르게 파악하고 바로 다음 액션으로 이동하게 해주는 첫 화면”이어야 합니다.

[2. 관리자용 대시보드로 설계]
이번 대시보드는 일반 직원용이 아니라
운영 관리자 / 최종 관리자용 대시보드로 설계해주세요.

즉, 아래 성격이 느껴져야 합니다.
- 호텔 프런트 운영 전체를 보는 화면
- 근무표 / 수요 예측 / 근태 / 직원 관리가 한곳에서 연결되는 화면
- 운영 리스크와 진행 상태를 확인하는 관리자 홈

[3. 상단 상태 바 구성]
대시보드 상단에는 현재 운영 기준 상태를 한 줄로 보여주는 얇고 정돈된 상태 바가 필요합니다.

반드시 포함할 항목:
- 선택 호텔
- 대상 월
- 현재 근무표 버전
- 근무표 상태: 작업 중 / 확정
- 수요 예측 상태: 최신 / 검토 필요
- 직원 관리 기준 상태: 전체 확정 완료 / 변경사항 있음
- 마지막 반영 시각

중요:
- 상단 상태 바는 길고 무거운 설명이 아니라
  한눈에 현재 운영 기준 상태가 읽히게 정리
- 헤더 아래에 차분하고 정돈된 형태로 배치

[4. KPI 카드 영역]
상단 상태 바 아래에는
운영 판단용 KPI 카드 6개 내외를 배치해주세요.

권장 KPI:
1. 오늘 체크인 예상
2. 오늘 체크아웃 예상
3. 이번 주 인력 리스크 일수
4. 근태 검토 필요 건수
5. 반영 안 된 직원 수
6. 회원가입 승인 대기 수

중요:
- 일반 매출/마케팅 지표가 아니라
  실제 운영자가 바로 판단하고 움직일 수 있는 숫자만 보여줄 것
- 카드 수를 너무 많이 늘리지 말 것
- 카드마다 관련 화면으로 이동 가능한 성격이 느껴져도 좋음

[5. 운영 진행 단계 영역]
대시보드 중간에는
이번 달 운영 준비가 어디까지 완료됐는지 보여주는 단계형 영역을 넣어주세요.

권장 단계:
1. 직원 관리 기준 확정
2. 수요 예측 결과 확정
3. 근태 신청 자동 반영 검토
4. 근무표 최종 확정

각 단계는 아래 상태 중 하나를 가질 수 있게 보이게 해주세요.
- 완료
- 진행 중
- 검토 필요
- 미시작

중요:
- 이 영역은 대시보드가 단순 정보판이 아니라
  업무 진행판 역할도 한다는 점을 보여줘야 함
- 각 단계 카드를 누르면 관련 화면으로 이동하는 느낌이 있으면 좋음

[6. 운영 경고 요약 영역]
대시보드에서 가장 중요한 영역 중 하나로
“운영 경고 요약” 블록을 넣어주세요.

예시 내용:
- 4월 12일 최소 조별 인원 2명 부족
- 4월 14일 인차지 배치 부족
- 4월 20일 공휴 운영 인원 부족 가능
- 4월 2주차 근태 신청 미반영 직원 3명
- 야간조 익월 전환 대상 2명

중요:
- 현재 근무표 상단 경고 요약을 대시보드형으로 확장한 느낌
- 관리자에게 “지금 가장 먼저 봐야 하는 위험”을 알려주는 영역
- 리스트형 또는 경고 카드형으로 정리 가능
- 지나치게 복잡한 차트보다 경고 리스트가 더 적합

[7. 근무표 상태 카드]
근무표는 이 시스템의 핵심이므로
별도 대표 카드 또는 블록으로 명확히 보여주세요.

반드시 보이면 좋은 정보:
- 현재 버전: 예) v3.3
- 상태: 작업 중 / 확정
- 최근 AI 조정 이력 수
- 최근 수정 시각
- 바로가기 버튼:
  - 보기
  - 새 버전 생성
  - 다운로드

중요:
- 사용자가 대시보드에서 근무표 현재 상태를 바로 이해할 수 있어야 함
- “지금 수정 중인지 / 확정됐는지 / 다시 손봐야 하는지”가 보여야 함

[8. 근태 신청/반영 요약 카드]
근태 신청과 근태 관리는 따로 화면이 있지만,
대시보드에서는 요약 형태로 한눈에 보여야 합니다.

권장 표시 항목:
- 자동 반영 완료 건수
- 검토 필요 건수
- 반영 불가 건수
- 반영 안 된 직원 수

가능하면 보조 정보:
- 주말 선호 반영 필요
- 공휴 근무 희망자 제외 후 계산
- 최근 수정 신청 수

중요:
- 대시보드에서 바로 근태 관리 화면으로 이동하고 싶게 만드는 요약 블록이어야 함
- 상태/예외 중심으로 간결하게

[9. 수요 예측 요약 카드]
수요 예측은 상세 표를 그대로 옮기는 것이 아니라
운영에 필요한 최소 요약 정보만 보여주세요.

권장 표시:
- 이번 달 피크 체크인일
- 이번 달 피크 체크아웃일
- 최소 인원 보강 필요일 수
- 최근 예측 갱신일

중요:
- 그래프보다 운영 캘린더 경고 느낌이 더 맞음
- 사용자가 “어느 날이 위험한지”를 빠르게 이해할 수 있어야 함

[10. 직원 관리 상태 카드]
직원 관리도 근무표 생성의 선행 조건이므로
대시보드에 별도 상태 블록으로 들어가야 합니다.

권장 내용:
- 총 직원 수
- 인차지 수
- 조 배치 미설정 인원
- 전체 확정 상태
- 최근 추가 직원 수

중요:
- 직원 관리 기준이 변경되면 근무표 생성에 바로 영향이 있다는 점이 느껴져야 함
- “직원 관리” 화면으로 바로 이동 가능한 연결성이 보이면 좋음

[11. 최근 변경 이력 영역]
대시보드 하단에는 최근 변경 이력 또는 최근 주요 액션 이력을 보여주세요.

권장 항목:
- AI 조정 이력
- 운영 정책 수정
- 직원 추가
- 회원가입 승인
- 비밀번호 초기화
- 권한 변경
- 근태 반영 결과 변경

권장 컬럼 또는 리스트 구성:
- 작업 유형
- 대상
- 작업자
- 시각

중요:
- 수정 이력 화면을 축약한 느낌
- 최근 시스템에 어떤 중요한 변화가 있었는지 한눈에 볼 수 있어야 함

[12. 화면 레이아웃 제안]
권장 레이아웃은 아래 흐름으로 정리해주세요.

1) 상단 상태 바
- 호텔 / 월 / 버전 / 주요 상태

2) KPI 카드 6개
- 오늘 CI
- 오늘 CO
- 리스크 일수
- 근태 검토 필요
- 미반영 직원 수
- 승인 대기 수

3) 중간 2열 또는 3열 주요 블록
- 운영 진행 단계
- 운영 경고 요약
- 근무표 상태 카드
- 근태 신청/반영 요약
- 수요 예측 요약
- 직원 관리 상태

4) 하단
- 최근 변경 이력

중요:
- 너무 많은 표나 중복 테이블은 피할 것
- 각 영역은 요약 + 바로가기 성격으로 정리
- 이미 각 상세 화면에 있는 테이블을 대시보드에서 길게 반복하지 말 것

[13. 디자인 방향]
반드시 아래 방향을 지켜주세요.
- 화려한 차트 중심 대시보드 금지
- 상태 카드, 경고 리스트, 진행 상태, 바로가기 액션 중심
- 정제되고 차분한 호텔 관리자 콘솔 느낌
- 과도한 컬러 남발 금지
- 리스크/경고는 분명하게 보이되 전체 톤을 해치지 않게
- 프리미엄 호텔 운영 본부용 시스템처럼 보여야 함

[14. 절대 피해야 할 방향]
- 마케팅/매출 대시보드처럼 그래프 위주로 가는 것
- 의미 없는 큰 숫자만 나열하는 것
- 각 상세 화면의 테이블을 그대로 대시보드에 반복하는 것
- 소비자용 앱처럼 캐주얼한 카드 UI
- 예약 사이트나 PMS 고객용 화면처럼 보이는 구성

[15. 최종 요청]
현재 시스템을 바탕으로
“대시보드” 화면을 아래 방향으로 새로 설계해주세요.

핵심 방향:
- 운영 관제형 홈 화면
- 현황 요약 + 즉시 액션 중심
- 근무표 / 수요 예측 / 근태 / 직원 관리 / 호텔 설정 상태가 한눈에 연결
- 오늘과 이번 달 운영 리스크를 빠르게 파악 가능
- 지금 무엇을 확인하고 어디로 이동해야 하는지 바로 판단 가능

즉,
대시보드는
“이번 달 스케줄 운영이 어디까지 준비됐는지,
오늘 무엇이 위험한지,
지금 무엇을 눌러야 하는지 알려주는 운영 허브”
처럼 보여야 합니다.

핵심: 대시보드는 예쁜 차트 화면이 아니라, 운영 상태·경고·진행률·바로가기 중심의 관리자 관제형 홈 화면으로 설계해주세요.
