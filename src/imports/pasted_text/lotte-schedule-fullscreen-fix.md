현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS의 근무표 화면에서
“전체화면보기 열기” 기능을 수정하는 과정에서 문제가 발생했어.

[현재 문제]
이전 상태에서는 근무표 메인 화면이 정상 복원되어 있었음.
정상 복원된 상태에는 아래 기능들이 있었음.

- 월별 근무표 / 연간 야간조 계획 탭
- 알림 발송 버튼
- 발송 이력 버튼
- 변경 이력 버튼
- 전월 근무강도 회복 결과 버튼
- EDU 교육 범례
- SICK 병가 범례
- 저장 / 확정 버튼
- 기존 LOTTE HOTELS & RESORTS 톤앤매너

그런데 오른쪽 위 전체화면보기 버튼을 눌렀을 때
전체화면보기에서는 예전 버전 화면이 보였음.

내가 원한 것은:
“현재 작업 중인 메인 근무표 화면과 동일한 화면이 전체화면보기에서도 그대로 나오는 것”
이었음.

하지만 수정 이후 오히려 메인 근무표 화면 자체가
전체화면보기의 예전 버전처럼 돌아가 버렸음.

즉, 잘못된 방향은 아래와 같음:
- 전체화면보기의 예전 UI를 메인 화면에 맞춘 것이 아니라
- 메인 화면이 전체화면보기의 예전 UI로 롤백됨

이건 잘못된 수정이므로 다시 바로잡아줘.

--------------------------------------------------
[최우선 수정 원칙]
--------------------------------------------------

1) 메인 근무표 화면을 예전 버전으로 되돌리면 안 됨
2) 전체화면보기 쪽 오래된 화면을 기준으로 삼으면 안 됨
3) 기준은 반드시 “정상 복원되어 있던 메인 근무표 화면”이어야 함
4) 전체화면보기는 그 메인 근무표 화면을 그대로 확장해서 보여주는 구조여야 함
5) 수정 중 기존 기능을 삭제하거나 덮어쓰지 말 것

--------------------------------------------------
[먼저 해야 할 일: 잘못 롤백된 메인 근무표 복원]
--------------------------------------------------

현재 메인 근무표 화면이 예전 버전처럼 돌아갔다면,
먼저 아래 기능들을 다시 복원해줘.

복원해야 하는 항목:
- 월별 근무표 탭
- 연간 야간조 계획 탭
- 알림 발송 버튼
- 발송 이력 버튼
- 변경 이력 버튼
- 전월 근무강도 회복 결과 버튼
- 새 버전 생성 버튼
- 다운로드 버튼
- 저장 버튼
- 확정 버튼
- EDU 교육 범례
- SICK 병가 범례
- 기존 근무 코드 범례
- 기존 휴무 코드 범례
- 기존 조 그룹 범례
- 기존 저장 완료 toast 패턴
- 기존 변경 이력 연결
- 기존 신청/병가/교육 관련 코드와 범례

중요:
- 전체화면보기의 예전 UI를 메인 화면에 덮어씌우지 말 것
- 메인 화면은 정상 복원된 최신 버전을 기준으로 유지해야 함

--------------------------------------------------
[전체화면보기 수정 방향]
--------------------------------------------------

전체화면보기는 별도 예전 컴포넌트, 별도 mock 데이터, 별도 toolbar를 사용하면 안 됨.

수정 목표:
- 전체화면보기는 메인 근무표 화면과 동일한 컴포넌트와 동일한 상태를 사용해야 함
- 단지 레이아웃만 더 넓게 보여주는 구조여야 함
- 전체화면보기 전용으로 오래된 Schedule 컴포넌트를 따로 렌더링하지 말 것

반드시 동일해야 하는 항목:
- 선택 호텔
- 선택 월
- 선택 버전
- 작업 중 / 확정 상태
- 상단 버튼 구성
- 탭 구성
- 범례 구성
- 경고 개수
- 근무표 셀 데이터
- EDU / SICK 범례
- 알림 발송 / 발송 이력 버튼
- 전월 근무강도 회복 결과 버튼
- 저장 / 확정 버튼
- 변경 이력 버튼
- AI 조정 입력 영역
- 스크롤 상태와 표 구성

[전체화면보기에서 사용하면 안 되는 것]
--------------------------------------------------

아래 구조가 있다면 제거하거나 메인 화면과 같은 컴포넌트로 교체해줘.

- 전체화면보기 클릭 시 메인 화면 상태를 초기화하는 코드

[절대 하지 말 것]
--------------------------------------------------

- 전체화면보기의 예전 화면을 메인 화면에 복사하지 말 것
- 메인 화면의 최신 기능을 삭제하지 말 것
- 탭 배열을 새로 만들면서 연간 야간조 계획을 누락하지 말 것
- 범례 배열을 새로 만들면서 EDU / SICK를 누락하지 말 것
- 버튼 배열을 새로 만들면서 알림 발송 / 발송 이력 / 전월 근무강도 회복 결과를 누락하지 말 것
- 전체화면보기 수정 중 메인 화면 UI를 구버전으로 되돌리지 말 것
- 기존 기능을 삭제하고 새로 만드는 방식 금지

[회귀 방지 원칙]
--------------------------------------------------

이번 수정 후에는 반드시 전체 기능 회귀 점검을 다시 해줘.

특히 아래 항목은 내가 지시하지 않았는데 사라지면 안 됨.
- 연간 야간조 계획 탭

- 알림 발송 버튼
- 발송 이력 버튼

- 주말/공휴 근무 희망 신청 취소
- 기존 좌측 메뉴 전체

수정 완료 후에는
“전체화면보기만 메인 화면과 동일하게 연결했고,
메인 근무표 화면은 최신 복원 상태를 유지했다”
는 기준으로 마무리해줘.

이번 작업의 핵심은
전체화면보기의 오래된 버전을 메인 화면에 덮어씌우는 것이 아니라,
메인 화면의 최신 정상 버전을 전체화면보기에서도 그대로 사용하도록 연결하는 것이야.
