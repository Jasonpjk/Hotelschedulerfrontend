현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “근태 신청” 화면을 아래 기준으로 다시 수정해주세요.

중요:
- 이번 수정은 “근태 신청” 화면에만 적용해주세요.
- 다른 화면은 건드리지 말아주세요.
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 / 넓은 여백 / 얇은 선 / 정돈된 그리드 유지

[1. 일반 근태 신청의 기본 구조 다시 정리]
일반 근태 신청에서는
직원이 직접 연차 / SL / 휴가를 선택하지 않게 해주세요.

또한 직원이 직접 고르는 값도 단순화해주세요.

기존처럼
- 쉬는 날 희망
- 근무 가능 / 근무 희망
- 상관없음
으로 두지 말고,

일반 근태 신청에서는 아래만 가능하게 해주세요.
- 쉬는 날 희망

즉,
직원은 특정 날짜에 대해
“이 날 쉬고 싶다”는 의사만 표현하면 됩니다.

중요:
- 일반 근태 신청에서는 근무 가능 / 근무 희망 / 상관없음 선택 제거
- 일반 근태 신청은 “쉬는 날 희망 제출” 중심으로 단순화
- 실제 휴무 / 연차 / 휴가 / SL 반영은 뒤에서 시스템 또는 관리자가 처리하는 구조로 보이게

[2. 일반 근태 신청 UI 수정]
달력에서 날짜를 선택한 후,
직원이 하는 기본 행동은 아래로 단순화해주세요.

원하는 흐름:
1. 달력에서 날짜 선택
2. “쉬는 날 희망” 신청
3. 사유 입력(선택)
4. 신청 제출

중요:
- 일반 근태 신청은 최대한 단순해야 함
- 직원이 복잡한 근태 유형을 고르지 않게 해주세요
- 사유는 여전히 선택 입력으로 유지

[3. 주말 / 공휴일 선호 구조 수정]
주말과 공휴일 관련해서는
직원이 “쉬고 싶다 / 상관없다”가 아니라
“근무 희망” 여부만 표현하면 됩니다.

즉, 아래처럼 단순화해주세요.

주말:
- 주말 근무 희망

공휴일:
- 공휴 근무 희망

중요:
- 주말 공휴일 선호에서 “휴무 희망”과 “상관없음” 선택은 제거
- 체크박스 또는 토글처럼 단순하게 표현 가능
- 예:
  - 주말 근무 희망
  - 공휴 근무 희망

즉,
직원은 기본적으로 쉬는 날 희망만 제출하고,
추가로 주말/공휴일에는 “나는 근무해도 된다 / 근무를 원한다”는 의사만 표시하는 구조로 정리해주세요.

[4. 주말 / 공휴 근무 희망의 의미가 드러나게]
주말 근무 희망 / 공휴 근무 희망은
일반 근태 신청과 별도의 보조 선호값처럼 보여야 합니다.

권장 설명 문구 예시:
- 주말 근무를 희망하는 경우 체크해주세요
- 공휴일에도 근무를 희망하는 경우 체크해주세요

중요:
- 이 값은 강제 배정이 아니라 선호값
- 자동 스케줄 생성과 공정성 계산에 반영되는 기준처럼 보여야 함

[5. 신청 내역에서 신청 의사와 최종 반영 결과 구분]
직원이 일반 근태 신청에서 “쉬는 날 희망”만 제출하더라도,
실제 근무표에서는 아래 중 하나로 반영될 수 있어야 합니다.
- 휴무
- 연차
- 휴가
- SL
- 미반영

따라서 신청 내역 또는 상세 보기에서는 아래를 구분해서 보여주세요.
- 신청 의사: 쉬는 날 희망
- 최종 반영 결과: 휴무 / 연차 / 휴가 / SL / 미반영

예시:
- 신청 의사: 4월 15일 쉬는 날 희망
- 최종 반영: 휴무

또는
- 신청 의사: 4월 22일 쉬는 날 희망
- 최종 반영: 미반영

중요:
- 직원은 단순히 쉬는 날 희망만 냈지만
- 실제 결과가 무엇인지 확인할 수 있어야 함

[6. 최대 인원 초과 시 사전 안내 유지]
직원이 쉬는 날 희망을 신청할 때,
이미 해당 날짜에 반영 가능한 최대 인원이 찬 경우에는
신청 단계에서 안내 문구가 떠야 합니다.

예시 문구:
- 해당 날짜는 이미 휴무 가능 인원이 가득 차 있어 반영이 어려울 수 있습니다. 관리자에게 문의해주세요.
- 이 날짜는 운영 인원 한도에 가까워 자동 반영이 불가할 수 있습니다.

중요:
- 신청 자체를 막는 것이 아니라
- 반영 불가 가능성을 사전에 알려주는 구조
- 직원이 신청 후 뒤늦게 알지 않도록 해주세요

[7. 연중 휴가 계획 신청 기능은 유지]
일반 근태 신청은 단순화하되,
연중 장기 휴가 계획 신청 기능은 그대로 유지해주세요.

즉, 별도 영역 또는 탭으로 아래 기능 유지:
- 여러 날짜 또는 연속 날짜 선택
- 총 신청 일수 입력
- 총 일수 안에서
  - 휴가 몇 일
  - 연차 몇 일
  - 휴무 몇 일
  나눠서 입력 가능

예:
- 총 7일
- 휴가 3일
- 연차 2일
- 휴무 2일

중요:
- 일반 근태 신청과 연중 휴가 계획 신청은 구분
- 일반 근태 신청은 “쉬는 날 희망”
- 장기 계획은 “총 휴가 계획” 성격으로 유지

[8. 직원용 화면에서 읽혀야 하는 구조]
수정 후 직원이 이 화면을 봤을 때 아래처럼 느껴져야 합니다.

- 일반 신청은 특정 날짜에 쉬고 싶은 날만 제출하면 된다
- 주말 / 공휴일은 근무 희망 여부만 별도로 표시하면 된다
- 연차 / 휴가 / SL은 내가 직접 일반 신청에서 고르지 않는다
- 실제 반영 결과는 나중에 확인할 수 있다
- 장기 휴가 계획은 별도로 신청할 수 있다

[9. 최종 요청]
현재 근태 신청 화면을 아래 기준으로 다시 수정해주세요.

- 일반 근태 신청에서 직원이 직접 고르는 값은 “쉬는 날 희망”만 남기기
- 근무 가능 / 근무 희망 / 상관없음 선택 제거
- 주말 / 공휴일은 “근무 희망” 여부만 표현하게 수정
- 주말 공휴 선호에서 휴무 희망 / 상관없음 제거
- 일반 근태 신청은 날짜 선택 + 쉬는 날 희망 + 사유(선택) 구조로 단순화
- 실제 휴무 / 연차 / 휴가 / SL 반영은 시스템 또는 관리자 후속 처리로 보이게
- 신청 내역에서는 신청 의사와 최종 반영 결과를 구분해서 보여주기
- 연중 장기 휴가 계획 신청 기능은 별도 영역으로 유지

핵심은,
일반 근태 신청은 최대한 단순하게 “쉬는 날 희망 제출”만 하도록 바꾸고,
주말/공휴일은 근무 희망 여부만 추가로 표현하게 정리하는 것입니다.
