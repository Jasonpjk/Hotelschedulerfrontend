현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “근무표” 화면을 아래 기준으로 수정해주세요.

중요:
- 이번 수정은 “근무표” 화면에만 적용해주세요.
- 다른 화면은 건드리지 말아주세요.
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 / 넓은 여백 / 얇은 선 / 정돈된 그리드 유지

[1. 상단 컨트롤 영역에 “초기화” 버튼 추가]
현재 근무표 화면 상단 오른쪽에
- 축소
- 확대
버튼이 있는데,
이 옆에 “초기화” 버튼을 추가해주세요.

버튼 위치:
- 축소 / 확대 옆
- 같은 컨트롤 그룹 안에 자연스럽게 배치

버튼 역할:
- 현재 확대/축소 상태를 기본값으로 되돌리는 기능
- 예: 100% 기본 배율로 복귀
- 사용자가 여러 번 확대/축소한 뒤 빠르게 원래 보기로 돌아올 수 있어야 함

중요:
- “초기화”는 페이지 전체 초기화가 아니라
  확대/축소 보기 상태 초기화로 읽혀야 합니다
- 버튼명만 보고도
  “배율 원위치”라는 의미가 자연스럽게 느껴지게 해주세요

권장 방식:
- 축소 / 확대 / 초기화 3개가 한 세트처럼 보이게 구성
- 관리자용 도구 버튼처럼 단정하게 정리
- 과하게 튀는 주요 CTA 버튼처럼 보이지 않게

[2. AI 자연어 조정 적용 후 변경 이력 자동 추가]
현재 근무표 하단에 “AI 자연어 조정” 입력창이 있는데,
여기에 명령을 입력하고 “적용하기”를 누르면
그 아래에 변경 이력이 자동으로 추가되도록 수정해주세요.

즉,
AI 자연어 조정은 단순 입력창으로 끝나는 것이 아니라
“무슨 명령이 언제 적용되었는지” 이력이 누적되어 보여야 합니다.

[3. 변경 이력 영역 추가]
AI 자연어 조정 영역 아래에
“변경 이력” 또는 “AI 조정 이력” 섹션을 새로 만들어주세요.

권장 섹션명:
- 변경 이력
- AI 조정 이력

위치:
- AI 자연어 조정 입력창 바로 아래
- 근무표 하단 작업 이력 영역처럼 보이게

[4. 변경 이력에 들어갈 정보]
사용자가 명령 입력 후 적용하면,
이력 리스트에 아래 정보가 자동으로 1건 추가되는 구조로 보여야 합니다.

필수 포함 항목:
- 적용 시각
- 입력한 자연어 명령
- 반영 결과 요약 또는 상태
- 적용 주체(AI 조정)

예시:
- 2026-03-08 14:32 / “박시○에 야간 연속 배정 금지” / 적용 완료
- 2026-03-08 14:40 / “주말 OFF를 더 균등하게 조정” / 적용 완료
- 2026-03-08 14:48 / “3월 22일 최소 커버리지 보완” / 일부 반영

중요:
- 단순히 명령문만 쌓이지 말고
- 적용 시각과 상태가 함께 보여야 함
- 사용자가 어떤 조정을 언제 했는지 추적 가능해야 함

[5. 적용하기 버튼과 이력 연동]
AI 자연어 조정 입력창에 명령을 입력하고
“적용하기” 버튼을 누르면
아래 흐름이 보이게 해주세요.

원하는 흐름:
1. 사용자가 자연어 명령 입력
2. 적용하기 클릭
3. 명령이 반영되는 피드백 표시
4. 하단 변경 이력에 자동 추가
5. 최신 이력이 맨 위 또는 맨 아래에 누적

중요:
- “적용” 후 아무 흔적이 남지 않으면 안 됨
- 변경 이력이 쌓여야 이후 사용자가 어떤 지시를 넣었는지 다시 볼 수 있어야 함

[6. 이력 표시 방식]
변경 이력은 너무 기술 로그처럼 딱딱하지 않게,
하지만 관리자용 기록 영역답게 신뢰감 있게 보이게 해주세요.

권장 방식:
- 리스트형 또는 테이블형
- 각 행마다 시각 / 명령 / 상태
- 최신순 정렬 가능
- 필요하면 상태 태그 사용
  예:
  - 적용 완료
  - 일부 반영
  - 검토 필요

중요:
- 화면이 너무 복잡해지지 않게
- 근무표 하단의 실무 로그 영역처럼 보이게

[7. 이력과 근무표 화면의 관계]
이 변경 이력은 근무표 버전 작업의 일부처럼 보여야 합니다.

즉,
사용자가 봤을 때 아래 의미가 느껴져야 합니다.
- 이 버전에서 어떤 AI 조정을 했는지
- 어떤 명령을 넣었는지
- 언제 반영했는지
- 반영 결과가 어땠는지

즉, 단순 채팅 로그가 아니라
“근무표 작업 이력”처럼 보여야 합니다.

[8. 디자인 방향]
반드시 아래 느낌을 유지해주세요.
- 근무표 하단에 자연스럽게 이어지는 보조 작업 영역
- 관리자용 SaaS답게 차분하고 정돈된 구조
- 과한 카드형 UI 금지
- 입력창 / 버튼 / 이력 리스트가 기존 화면과 같은 컴포넌트 체계 안에 있는 것처럼 보이게

[9. 최종 요청]
현재 근무표 화면을 아래 기준으로 수정해주세요.

- 축소 / 확대 옆에 “초기화” 버튼 추가
- 초기화 버튼은 확대/축소 배율을 기본값으로 되돌리는 의미로 보이게 구성
- AI 자연어 조정 입력 후 “적용하기”를 누르면
  하단에 변경 이력이 자동으로 추가되게 수정
- 변경 이력에는 적용 시각, 입력 명령, 반영 상태가 보이게 정리
- 이력은 근무표 작업 로그처럼 누적되어 보여야 함

핵심은,
근무표 화면에서
“확대/축소 제어”와 “AI 조정 이력 추적”이 더 명확해지도록 만드는 것입니다.

핵심: 축소/확대 옆에 배율 초기화 버튼을 추가하고, AI 자연어 조정 적용 시 하단에 시각·명령·상태가 포함된 변경 이력이 자동 누적되게 해주세요.
