현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS의
“스케줄 재생성 > 근무표에서 보호 일정 선택” 모달을 수정해줘.

이번 작업의 핵심은
이미 등록된 보호 일정이 모달에 먼저 표시되고,
스케줄러가 그 자리에서 기존 보호를 해제하거나 추가 보호 일정을 선택할 수 있게 만드는 것이야.

--------------------------------------------------
[현재 문제]
--------------------------------------------------

현재 보호 일정 선택 모달에 들어가면
이미 등록된 보호 일정이 근무표 선택 화면과 명확히 연결되어 보이지 않음.

또한 자동 절대 보호 목록에
“신혼여행 (인사팀 확인 완료)”
“장기 병가 (진단서 제출 완료)”
처럼 괄호 설명이 붙어 있는데,
목록에서는 너무 길고 복잡해 보임.

--------------------------------------------------
[수정 목표]
--------------------------------------------------

보호 일정 선택 모달에 들어가면 아래가 바로 보여야 함.

1) 이미 등록된 보호 일정 목록
2) 근무표 위에 이미 보호된 셀 표시
3) 기존 보호 일정 해제 기능
4) 추가 보호 일정 선택 기능
5) 보호 수준 변경 시 확인창
6) 목록 문구 단순화

--------------------------------------------------
[1. 기존 보호 일정 먼저 표시]
--------------------------------------------------

“근무표에서 보호 일정 선택” 모달을 열면
기존에 등록된 보호 일정이 상단 또는 좌측 요약 영역에 먼저 표시되어야 함.

표시 그룹:
- 자동 절대 보호
- 자동 우선 보호
- 수동 추가 보호

각 그룹에는 아래 정보를 표시해줘.

필수 표시:
- 근무 코드 또는 상태 배지
- 직원명
- 날짜 또는 기간
- 보호 사유

예:
자동 절대 보호 4건
- OFF 최유진 05-01 ~ 05-14 신혼여행
- SICK 박지호 04-29 ~ 05-31 장기 병가
- EDU 이수진 05-06 승인된 교육
- REQ 정예은 05-09 승인 안내 발송된 휴일 신청

자동 우선 보호 3건
- SL 이수진 05-03 SL
- VAC 강미래 05-14 ~ 05-15 연차
- VAC 박지현 05-19 ~ 05-20 연차

중요:
- 기존 보호 일정이 모달을 열 때마다 사라지면 안 됨
- 기존 보호 일정은 근무표 셀에도 표시되어야 함
- 기존 보호 일정과 새로 선택한 보호 일정이 같은 데이터 구조로 관리되어야 함

--------------------------------------------------
[2. 괄호 설명 제거]
--------------------------------------------------

보호 일정 목록에서 사유명 옆의 괄호 설명은 제거해줘.

기존:
- 신혼여행 (인사팀 확인 완료)
- 장기 병가 (진단서 제출 완료)

변경:
- 신혼여행
- 장기 병가

괄호 안의 세부 확인 내용은 목록에는 표시하지 말고
필요하면 상세 tooltip 또는 상세 보기에서만 보여줘.

예:
목록 표시: 신혼여행
tooltip: 인사팀 확인 완료

목록 표시: 장기 병가
tooltip: 진단서 제출 완료

목록은 짧고 명확하게 유지해줘.

--------------------------------------------------
[3. 근무표 셀에 기존 보호 상태 표시]
--------------------------------------------------

보호 일정 선택 모달 안의 근무표에는
이미 등록된 보호 일정이 시각적으로 표시되어야 함.

표시 방식:
- 절대 보호: 자물쇠 아이콘 또는 강한 테두리
- 우선 보호: 방패 아이콘 또는 은은한 골드 테두리

셀 hover 시 표시:
- 보호 수준
- 보호 사유
- 날짜
- 직원명

예:
절대 보호
사유: 신혼여행
기간: 05-01 ~ 05-14

우선 보호
사유: 연차
날짜: 05-14

--------------------------------------------------
[4. 기존 보호 해제 기능]
--------------------------------------------------

이미 보호된 셀이나 기존 보호 목록 항목에는
보호 해제 기능을 제공해줘.

방법:
- 목록에서 “해제” 버튼 제공
또는
- 근무표 셀을 다시 클릭하면 해제 확인창 표시

확인창 문구:
제목: 보호 일정 해제
내용:
“이 일정은 현재 보호 일정으로 등록되어 있습니다.
스케줄 재생성 시 변경 대상이 될 수 있도록 보호를 해제하시겠습니까?”

버튼:
- 보호 해제
- 유지
- 취소

중요:
- 해제 즉시 목록과 근무표 셀 표시가 같이 갱신되어야 함
- 저장 전에는 임시 변경 상태로 두고
- “보호 일정으로 저장”을 눌렀을 때 최종 반영

--------------------------------------------------
[5. 추가 보호 일정 선택 기능]
--------------------------------------------------

기존 보호 일정을 확인한 뒤,
스케줄러가 추가로 보호할 셀을 근무표에서 다중 선택할 수 있어야 함.

동작:
1) 보호 수준 선택
   - 절대 보호
   - 우선 보호

2) 근무표 셀 다중 선택

3) 선택된 셀은 선택 당시의 보호 수준으로 저장

4) 하단 저장 버튼에 건수 표시

예:
보호 일정으로 저장
절대 보호 5건 / 우선 보호 7건

중요:
- 우선 보호로 선택한 셀은 이후 절대 보호 모드로 바꿔도 자동으로 절대 보호가 되면 안 됨
- 각 셀은 선택 당시 보호 수준을 개별적으로 유지해야 함

--------------------------------------------------
[6. 보호 수준 변경 시 확인창]
--------------------------------------------------

이미 보호된 일정을 다른 보호 수준으로 바꾸려는 경우
반드시 확인창을 띄워줘.

절대 보호 → 우선 보호 변경 시:
제목: 보호 수준 변경
내용:
“이 일정은 현재 절대 보호로 설정되어 있습니다.
우선 보호로 변경하면 스케줄 재생성 시 관리자 확인 후 변경될 수 있습니다.
보호 수준을 우선 보호로 변경하시겠습니까?”

버튼:
- 우선 보호로 변경
- 절대 보호 유지
- 취소

우선 보호 → 절대 보호 변경 시:
제목: 보호 수준 변경
내용:
“이 일정은 현재 우선 보호로 설정되어 있습니다.
절대 보호로 변경하면 스케줄 재생성 시 자동 변경 대상에서 제외됩니다.
절대 보호로 변경하시겠습니까?”

버튼:
- 절대 보호로 변경
- 우선 보호 유지
- 취소

--------------------------------------------------
[7. 모달 안에서 할 수 있어야 하는 작업]
--------------------------------------------------

보호 일정 선택 모달 안에서 아래 작업이 모두 가능해야 함.

- 기존 보호 일정 확인
- 기존 보호 일정 해제
- 기존 보호 수준 변경
- 새 보호 일정 추가 선택
- 절대 보호 / 우선 보호 구분 선택
- 월말일까지 스크롤해서 선택
- 저장 후 스케줄 재생성 화면으로 복귀

--------------------------------------------------
[8. 스크롤 개선]
--------------------------------------------------

근무표 선택 영역은 월말일까지 볼 수 있어야 함.

수정:
- 날짜 열이 월말일까지 표시
- 가로 스크롤바 제공
- 직원명 영역은 고정
- 날짜 영역만 가로 스크롤
- 세로 스크롤과 가로 스크롤이 겹치지 않게 처리
- 하단 버튼 때문에 마지막 행이나 스크롤바가 가려지지 않게 처리

예:
5월 근무표이면 5월 31일까지 선택 가능해야 함.

--------------------------------------------------
[9. 저장 후 반영]
--------------------------------------------------

“보호 일정으로 저장” 클릭 시
스케줄 재생성 화면의 보호 일정 카드에 즉시 반영해줘.

반영 항목:
- 절대 보호 건수
- 우선 보호 건수
- 보호 일정 상세 목록
- 해제된 보호 일정 제외
- 변경된 보호 수준 반영

toast 문구:
제목: 보호 일정 저장 완료
설명: 절대 보호 4건, 우선 보호 7건이 저장되었습니다.
