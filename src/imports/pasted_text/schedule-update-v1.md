현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “근무표” 화면을 아래 기준으로 수정해주세요.

중요:
- 기존 톤앤매너는 그대로 유지해주세요.
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 / 넓은 여백 / 얇은 선 / 정돈된 그리드 유지
- 이번 수정은 “근무표 화면”에만 적용해주세요.
- 다른 메뉴나 다른 화면은 건드리지 말아주세요.

[1. 확정 버튼 색상 변경]
현재 우측 상단의 “확정” 버튼 색상을 검정색 계열로 변경해주세요.

조건:
- 기존 골드 버튼이 아니라 검정/딥 네이비 계열의 강한 주요 액션 버튼으로 수정
- “전체 확정” 느낌이 명확하게 보이도록 해주세요
- 다운로드 버튼보다 시각적으로 우선순위가 높게 보이게 정리해주세요

[2. 확정 후 스케줄 셀 수동 편집 잠금]
현재는 각 직원의 스케줄 셀을 클릭하면 수동 조정 팝업이 뜨는 구조인데,
“확정” 버튼을 누른 후에는 이 기능이 멈추도록 수정해주세요.

즉:
- 확정 전: 셀 클릭 가능 / 팝업 열림 / 수동 조정 가능
- 확정 후: 셀 클릭 불가 / 팝업 열리지 않음 / 수정 잠금 상태

중요:
- 확정 이후에는 스케줄이 잠긴 것처럼 보여야 합니다
- 마우스 오버나 커서, 셀 상태 등도 잠금 상태가 느껴지게 해주세요
- 필요하면 “확정된 버전은 직접 수정할 수 없습니다” 같은 안내 문구나 잠금 아이콘을 추가해도 좋습니다

[3. 초안 상태 문구를 확정으로 변경]
현재 화면에서 “초안”이라고 표시되는 상태값은
확정 버튼을 누른 후 “확정”으로 변경되게 해주세요.

예:
- 변경 전: 초안
- 변경 후: 확정

중요:
- 상단 상태 뱃지 / 버전 상태 / 근무표 상태 표시 등
  현재 초안으로 보이는 부분은 확정 이후 모두 “확정” 상태로 바뀌어야 합니다

[4. 버전명 옆의 (초안), (확정) 표기는 삭제]
버전 드롭다운 또는 버전명 옆에 붙는
- (초안)
- (확정)
이런 괄호 표기는 삭제해주세요.

즉:
- 버전명은 v3.1, v3.2 같은 버전명 자체만 깔끔하게 보이게
- 상태는 별도 뱃지나 화면 상태값으로만 보여주고
- 버전명 옆 괄호 표기는 없애주세요

예:
- v3.1 (초안) → v3.1
- v3.2 (확정) → v3.2

[5. 새 버전 생성 기능 흐름 수정]
현재 “새 버전 생성” 버튼을 누르면
왼쪽 버전 드롭다운에 새로운 버전이 추가되고,
그와 동시에 스케줄이 새로 생성되는 흐름이 보이게 수정해주세요.

원하는 흐름:
1. 사용자가 “새 버전 생성” 버튼 클릭
2. 새로운 버전 생성
   예: v3.2
3. 버전 드롭다운 목록에 자동 추가
4. 새 버전이 자동 선택됨
5. 새 버전 기준으로 스케줄이 새로 생성되어 화면에 표시됨

중요:
- 새 버전 생성은 기존 확정 버전을 덮어쓰는 것이 아니라
  새로운 버전이 추가되는 구조여야 합니다
- 사용자는 버전 드롭다운에서 이전 버전과 새 버전을 전환해서 볼 수 있어야 합니다
- 새 버전은 “수정 가능한 작업 버전”처럼 보이게 해주세요

[6. 버전 상태 표현 방식 정리]
버전명 자체는 단순하게 유지하고,
상태는 별도 방식으로 보여주세요.

권장 방식:
- 버전명은 v3.1 / v3.2 / v3.3 처럼만 표시
- 상태는 화면 내 별도 뱃지로 표시
  예:
  - 확정
  - 작업 중
  - 최신 버전

중요:
- 버전명 자체에 상태 괄호를 붙이지 말 것
- 화면이 더 정돈되고 실무형으로 보이게 해주세요

[7. 다운로드 버튼 기능 수정]
현재 “다운로드” 버튼을 누르면
바로 파일이 내려가는 느낌이 아니라,
엑셀 다운로드 관련 팝업이 뜨는 구조로 수정해주세요.

팝업 목적:
- 엑셀로 다운로드하기 전 옵션 확인
- 어떤 형식으로 받을지 선택
- 다운로드 실행

권장 팝업 구성:
- 팝업 제목: 근무표 다운로드
- 현재 선택 버전 표시
- 현재 대상 월 표시
- 다운로드 형식:
  - Excel (.xlsx)
- 포함 옵션 예시:
  - 직원별 근무표 포함
  - 운영 경고 요약 포함
  - 범례 포함
- 하단 버튼:
  - 취소
  - 다운로드

중요:
- 현재 요청 기준으로는 엑셀 다운로드 팝업이 뜨는 것이 핵심
- 추후 PDF 등 확장 여지가 있어도 지금은 엑셀 다운로드 중심으로 보이게 해주세요

[8. 확정 후 화면 상태가 분명하게 보이게]
확정 버튼을 누른 뒤에는
근무표 전체가 “확정된 버전”처럼 읽혀야 합니다.

반드시 보이면 좋은 요소:
- 상태값: 확정
- 편집 잠금
- 셀 클릭 비활성
- 확정 버전 안내 문구
- 새 버전 생성 유도

예시 문구:
- 이 버전은 확정되어 직접 수정할 수 없습니다
- 수정이 필요하면 새 버전을 생성해주세요

즉,
확정 후에는 사용자가 자연스럽게
“이제 이 버전은 잠겼고, 수정하려면 새 버전을 만들어야 하는구나”
라고 이해할 수 있어야 합니다.

[9. 전체 UX 흐름]
근무표 화면은 아래 흐름이 명확해야 합니다.

1. 작업 중 버전 선택
2. 셀 클릭하여 수동 조정 가능
3. 필요 시 AI 자연어 조정 가능
4. 확정 버튼 클릭
5. 상태가 확정으로 변경
6. 셀 직접 수정 잠금
7. 수정이 필요하면 새 버전 생성
8. 새 버전 생성 후 새 버전에서 다시 편집 가능
9. 다운로드 버튼으로 엑셀 다운로드 팝업 실행

즉,
“작업 버전 → 확정 → 잠금 → 새 버전 생성 → 다시 작업”
구조가 분명하게 보이도록 정리해주세요.

[10. 최종 요청]
현재 근무표 화면을 아래 기준으로 수정해주세요.

- 확정 버튼을 검정색 계열로 변경
- 확정 버튼을 누르면 각 스케줄 셀 클릭 시 뜨던 수동 조정 팝업이 더 이상 열리지 않게 잠금 처리
- 초안으로 표시되던 상태값은 확정으로 변경
- 버전명 옆의 (초안), (확정) 표기는 삭제
- 새 버전 생성 시 버전 드롭다운에 새 버전이 추가되고, 그 버전 기준으로 스케줄이 새로 생성되게 수정
- 다운로드 버튼 클릭 시 엑셀 다운로드 팝업이 뜨게 수정
- 확정된 버전은 수정 불가, 새 버전에서만 수정 가능하다는 흐름이 화면에서 명확히 보이게 정리

이 기준으로 “근무표” 화면만 다시 수정해주세요.

핵심은 확정 후 셀 편집 잠금, 상태값을 초안→확정으로 변경, 새 버전 생성 시 드롭다운에 새 버전 추가, 다운로드 시 엑셀 팝업 오픈입니다. 근무표 화면만 수정해주세요.
