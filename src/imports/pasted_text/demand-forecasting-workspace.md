현재 수요 예측 페이지의 디자인 시스템과 톤앤매너는 유지하되, 실제 예측 업무 흐름에 맞게 “데이터 업로드 단계”를 추가해서 페이지를 다시 정리해줘.

중요:
- 이 페이지는 단순 결과 조회 화면이 아니라, 실제로 데이터를 올리고 예측을 산출하는 관리자 워크스페이스여야 해
- 현재 체크인/체크아웃 관련 데이터와 전년도 데이터를 업로드할 수 있는 기능이 반드시 있어야 해
- 별도 페이지로 분리하지 말고, 현재 수요 예측 페이지 안에 자연스럽게 통합해줘
- 로그인/대시보드/근무표와 동일한 프리미엄 호텔 운영 SaaS 톤앤매너 유지
- 지금 시안은 한국어 기준으로 100% 통일

전체 흐름을 아래처럼 바꿔줘:
1. 데이터 업로드
2. 예측 산출
3. 예측 근거
4. 이벤트 보정
5. 최종 확정

상단 단계 표시도 이 순서로 다시 정리해줘.

새로 추가해야 하는 핵심 기능:
1. 올해 현재 예약 데이터 업로드
반드시 포함:
- 파일 업로드 버튼
- 템플릿 다운로드 버튼
- 업로드 완료 상태
- 마지막 업로드 일시
- 파일 형식 예시 표시 (엑셀/CSV 느낌)
- 어떤 데이터가 들어가는지 설명:
  - 날짜
  - 현재 예약 기준 예상 체크인 수
  - 현재 예약 기준 예상 체크아웃 수
  - 객실 타입 또는 세부 구분이 있으면 보조적으로 표현 가능

2. 전년도 데이터 업로드
반드시 포함:
- 파일 업로드 버튼
- 템플릿 다운로드 버튼
- 업로드 완료 상태
- 마지막 업로드 일시
- 어떤 데이터가 들어가는지 설명:
  - 전년 동일 요일 기준 예약 수
  - 전년 실제 체크인 수
  - 전년 실제 체크아웃 수
- 단순 “전년 동일 날짜”가 아니라 “전년 동일 요일/유사 주차 기준 비교”에 활용되는 데이터라는 점이 드러나게 해줘

3. 선택 업로드 또는 보조 입력
반드시 포함:
- 공휴일/연휴 정보
- 이벤트/공연/컨퍼런스/단체 예약 정보
- 수동 입력 또는 업로드 가능 구조
- 예: 추석, 설, 대체공휴일, 주말 포함 연휴, BTS 공연 같은 대형 이벤트
- 과거 데이터가 없는 이벤트도 추가 가능한 구조처럼 보여줘

페이지 구조 재구성:
상단 요약 카드 아래 또는 상단 단계 영역 아래에 “데이터 업로드” 섹션을 새로 만들어줘.

데이터 업로드 섹션 구성:
- 카드형 업로드 블록 3개 정도로 정리
  1) 올해 현재 예약 데이터
  2) 전년도 실적 데이터
  3) 공휴일/이벤트 데이터
- 각 카드에 다음 요소 포함:
  - 카드 제목
  - 업로드 버튼
  - 템플릿 다운로드 버튼
  - 업로드 상태 배지
  - 최근 업로드 일시
  - 데이터 검증 상태(정상 / 누락 / 검토 필요)
  - 간단한 설명문

보조 기능:
- 업로드 후 데이터 미리보기 표가 보이게
- 열 매핑이 자연스럽게 느껴지게
- 예를 들어:
  - 날짜
  - 체크인
  - 체크아웃
  - 예약 수
  - 실제 수치
  - 이벤트명
- 누락 행, 형식 오류, 중복 데이터가 있으면 경고 배지로 보이게

예측 결과와 연결:
- 업로드가 완료된 데이터가 있어야 예측 산출 단계가 활성화되는 구조처럼 보여줘
- 예측 결과 카드나 표에 “올해 업로드 데이터 기준”, “전년도 업로드 데이터 반영”, “이벤트 보정 반영” 같은 상태가 보이게 해줘
- 관리자 입장에서 이 예측이 어떤 입력 데이터로 만들어졌는지 한눈에 읽히게 해줘

예측 근거 탭 수정:
- 현재 표는 유지하되, 각 날짜 행에 업로드된 올해 데이터와 전년도 데이터가 근거가 된다는 점이 더 분명히 보이게
- “올해 현재 예약”, “전년 동일 요일 예약”, “전년 실제 체크인”, “전년 실제 체크아웃”이 더 명확히 읽히게
- 데이터 출처가 보이는 구조로 정리해줘

이벤트 보정 탭 수정:
- 업로드된 이벤트와 수동 입력된 이벤트가 함께 보이는 구조로 만들어줘
- 이벤트별 영향 수준, 체크인 중심/체크아웃 중심, 적용 여부, 조정 사유가 보이게
- 공휴일/명절/연휴 구조도 여기서 함께 검토할 수 있게 해줘

최종 확정 탭 수정:
- “최종 반영값”이 어떤 업로드 데이터와 보정을 기반으로 만들어졌는지 요약이 보이게
- 근무표 생성 반영 버튼은 유지
- 이 값이 월간 스케줄 생성에 사용된다는 연결이 분명해야 함

디자인 조건:
- 데이터 업로드 기능이 추가되어도 화면이 투박하거나 개발자용 툴처럼 보이지 않게
- 파일 업로드 UI는 고급스럽고 정돈된 관리자용 스타일
- 카드, 표, 상태 배지, 검증 상태, 업로드 이력은 실무형 SaaS 관리자 화면처럼 정리
- 너무 장식적이지 않게
- 너무 기술적이지 않게
- 실제 운영팀이 매달 말 사용하는 도구처럼 보여야 함

최종 목표:
- 현재 수요 예측 페이지가 실제 운영에 쓸 수 있는 “예측 워크스페이스”처럼 보이게
- 현재 체크인/체크아웃 데이터와 전년도 데이터를 업로드하고, 검토하고, 예측하고, 보정하고, 최종 반영하는 전체 흐름이 한 페이지 안에서 명확하게 드러나게
