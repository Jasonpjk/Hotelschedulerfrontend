현재까지 만든 LOTTE HOTELS & RESORTS 관리자용 SaaS의 톤앤매너를 그대로 유지하면서
새로운 “직원 관리” 기능 화면을 설계해주세요.

중요:
- 기존 대시보드 / 근무표 / 수요 예측 화면과 동일한 디자인 시스템 유지
- 프리미엄 호텔 운영 관리자용 SaaS 무드 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 유지
- 넓은 여백, 얇은 선, 정돈된 그리드, 실무형 UI 유지
- 소비자용 HR 페이지처럼 보이면 안 되고, 근무표 생성과 직접 연결되는 운영용 직원 관리 화면이어야 함

[1. 화면 목적]
이 화면은 프런트 직원의 기본 정보, 직급, 근속, 프런트 경력, 근무 가능 조, 인차지 여부를 관리하는 화면입니다.
근무표 생성 시 이 정보가 바로 반영되는 구조로 보여야 합니다.

[2. 기본 구조]
직원 관리 화면은 아래 구조로 설계해주세요.

- 상단: 페이지 제목 + 설명 + 신규 직원 추가 버튼
- 중단: 직원 목록 테이블
- 우측 또는 팝업: 직원 상세 입력 / 수정 폼
- 필요하면 상단 필터 영역 추가
- 전체적으로 복잡한 HR 시스템이 아니라 “근무 배치용 직원 마스터 관리 화면”처럼 보여야 함

[3. 직원 분류 기준]
직원은 근무표상 아래 조에 배치될 수 있어야 합니다.
- 오전조
- 오후조
- 야간조
- 중간조

각 조에서 특정 직원에게는 “인차지” 역할을 부여할 수 있어야 하고
나머지 직원은 “담당”으로 구분되면 됩니다.

즉, 역할 구조는 단순하게 아래 2개면 됩니다.
- 인차지
- 담당

중요:
- 인차지는 조별 운영 책임자 개념
- 담당은 일반 운영 인력 개념
- 직원별로 어떤 조에 배치 가능한지 체크할 수 있어야 함
- 직원별로 어떤 조에서 인차지 역할 수행 가능한지 설정 가능하면 좋음

[4. 직급 체계]
직급은 아래 값으로 고정해주세요.
- L1-A
- L1-C
- L1-D
- L2-A
- L2-C
- L3
- L4

직급은 드롭다운 또는 선택칩 형태로 입력 가능하게 해주세요.

[5. 날짜 기반 근속 계산 기능]
직원 관리에서 아래 날짜 입력 기능이 반드시 있어야 합니다.

1) 입사일
- 입사일을 입력하면 전체 근속이 자동 계산되어 보여야 함
- 예: 3년 2개월
- 별도 수기 입력이 아니라 자동 계산

2) 프런트 근무 시작일
- 프런트 근무 시작일을 입력하면 프런트 근속이 자동 계산되어 보여야 함
- 예: 1년 8개월
- 이것도 자동 계산

[6. 프런트 복귀자 처리]
중요 기능:
프런트 근무를 하다가 다른 직무로 이동했다가 다시 프런트로 돌아온 직원이 있습니다.
이 경우 아래 정보가 함께 입력 가능해야 합니다.

필수 지원 방식:
- 예전 프런트 근무 기간
- 새로 프런트로 다시 시작한 날짜

즉, 단순히 현재 프런트 시작일만 넣는 구조가 아니라
“이전 프런트 경력 + 현재 프런트 재시작일”을 함께 관리할 수 있어야 합니다.

원하는 입력 방식 예시:
- 이전 프런트 근무 시작일
- 이전 프런트 근무 종료일
- 현재 프런트 재시작일

또는
- 이전 프런트 누적 경력
- 현재 프런트 재시작일

둘 중 더 실무적으로 깔끔한 방식으로 제안해주세요.
다만 중요한 것은:
- 복귀자의 프런트 총 경력이 계산 가능해야 함
- 현재 연속 프런트 근속과 누적 프런트 경력을 구분해서 볼 수 있으면 좋음

추천 표시 방식:
- 전체 근속
- 현재 프런트 근속
- 누적 프런트 경력

[7. 직원 목록 테이블에 들어갈 항목]
직원 관리 메인 테이블에는 아래 정보가 잘 보이게 해주세요.

- 이름
- 직급
- 역할 구분(인차지 / 담당)
- 배치 가능 조
- 입사일
- 전체 근속
- 프런트 근무 시작일
- 현재 프런트 근속
- 누적 프런트 경력
- 상태(재직 등 기본 상태가 필요하면 최소한으로)

중요:
- 표는 근무표 생성에 필요한 정보 위주로 간결하게
- 불필요한 인사정보는 과하게 넣지 말 것
- 운영팀이 빠르게 읽고 수정할 수 있게 실무형으로 구성

[8. 직원 상세 입력/수정 폼]
직원 추가 또는 수정 시 아래 항목이 입력 가능해야 합니다.

기본 정보
- 직원명
- 직급
- 역할 구분: 인차지 / 담당

배치 가능 조
- 오전조
- 오후조
- 야간조
- 중간조
체크박스 또는 칩 선택 형태로 설정 가능

인차지 가능 조
- 어떤 조에서 인차지 역할 수행 가능한지 별도로 지정 가능하면 좋음
예:
- 오전조 인차지 가능
- 오후조 인차지 가능
- 야간조 인차지 가능
- 중간조 인차지 가능

근속 정보
- 입사일
- 전체 근속 자동 계산 표시

프런트 경력 정보
- 현재 프런트 근무 시작일
- 현재 프런트 근속 자동 계산
- 이전 프런트 경력 추가 영역
- 필요 시 “이전 경력 추가” 버튼으로 입력 블록 펼침

[9. 복귀자 UX]
프런트 복귀자 입력은 너무 복잡한 HR 화면처럼 보이지 않게 정리해주세요.

추천 UX:
- 기본값은 일반 직원 입력 구조
- “프런트 복귀자” 토글 또는 체크박스를 켜면 추가 입력란 노출
- 추가 입력란에
  - 이전 프런트 근무 시작일
  - 이전 프런트 근무 종료일
  - 현재 프런트 재시작일
  입력 가능
- 화면 하단에서 자동 계산 결과 표시
  - 현재 프런트 근속
  - 누적 프런트 경력

[10. 화면 톤앤매너]
디자인은 기존 화면과 완전히 연결된 느낌이어야 합니다.

반드시 유지할 것:
- LOTTE HOTELS & RESORTS 관리자 SaaS 무드
- 프리미엄 호텔 본사 운영시스템 같은 정제된 UI
- 카드 남발 금지
- 과한 컬러 금지
- 얇은 선과 여백 중심
- 폼도 고급스럽고 단정하게
- 태그, 칩, 드롭다운, 테이블이 기존 시스템과 같은 컴포넌트 체계처럼 보이게

[11. 이 화면에서 보이면 좋은 보조 요소]
가능하면 아래 요소도 포함해주세요.
- 직급 필터
- 인차지만 보기 필터
- 특정 조 배치 가능 직원만 보기 필터
- 신규 직원 추가 버튼
- 수정 / 저장 / 취소 버튼
- 근무표 반영용 직원 상태가 한눈에 보이는 태그

[12. 최종 목적]
이 직원 관리 화면은 단순 인사 관리가 아니라
근무표 생성과 운영 밸런스를 위한 직원 마스터 화면이어야 합니다.

즉, 사용자가 이 화면을 봤을 때 바로 아래 목적이 느껴져야 합니다.
- 누가 어느 조에 들어갈 수 있는지
- 누가 인차지인지
- 직급이 어떤지
- 전체 근속과 프런트 경력이 어느 정도인지
- 프런트 복귀자의 누적 경력을 어떻게 반영할지

최종 요청:
현재 SaaS의 디자인 시스템을 유지한 상태로
“직원 관리” 화면을 새로 만들되,

- 조 배치 가능 정보
- 인차지 / 담당 구분
- 직급 체계
- 입사일 기반 전체 근속 계산
- 프런트 시작일 기반 프런트 근속 계산
- 프런트 복귀자의 이전 경력 + 현재 재시작일 입력
- 누적 프런트 경력 표시

이 기능이 자연스럽게 포함된 관리자용 실무 화면으로 설계해주세요.
