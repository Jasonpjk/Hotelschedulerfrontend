
현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “근태 신청” 화면을 아래 기준으로 다시 수정해주세요.

중요:
- 이번 수정은 “근태 신청” 화면에만 적용해주세요.
- 다른 화면은 건드리지 말아주세요.
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 / 넓은 여백 / 얇은 선 / 정돈된 그리드 유지

[1. 일반 쉬는 날 신청 구조 유지]
일반 근태 신청에서 직원은 여전히
“특정 날짜에 쉬고 싶다”는 희망만 제출하게 해주세요.

즉,
일반 쉬는 날 신청에서는 직원이 직접 아래를 고르지 않게 유지해주세요.
- 연차
- SL
- 휴가

일반 신청 흐름:
1. 날짜 선택
2. 쉬는 날 희망 신청
3. 사유 입력(선택)
4. 신청하기

중요:
- 일반 신청은 날짜 희망만 제출
- 실제 최종 반영 유형은 시스템이 결정

[2. 시스템 자동 반영 유형 제한]
시스템이 최종 반영할 수 있는 자동 배정 유형은 아래 2개로만 제한해주세요.
- SL
- 연차

중요:
- 시스템이 휴가를 자동 배정하면 안 됨
- 휴가는 반드시 직원이 직접 신청하는 경우에만 반영 가능한 값으로 정리해주세요

즉,
자동 반영 로직에서 가능한 최종 결과는 아래처럼 보이게 해주세요.
- SL
- 연차
- 미반영

필요 시 기존 쉬는 날 희망이 받아들여지지 않을 수도 있음

[3. 휴가는 직원 직접 신청만 가능]
휴가는 시스템 자동 배정 대상에서 제외해주세요.

즉,
- 직원이 별도로 휴가를 신청한 경우에만 휴가 처리 가능
- 일반 쉬는 날 희망 신청 결과를 시스템이 휴가로 자동 변경하지 않음

중요:
- 휴가와 일반 쉬는 날 희망은 구분
- 일반 쉬는 날 희망의 자동 결과는 SL 또는 연차만 가능
- 휴가는 직원이 직접 신청한 별도 휴가 신청/휴가 계획에서만 처리되는 구조로 보이게 해주세요

[4. 자동 반영 우선순위 수정]
쉬는 날 희망을 시스템이 자동 반영할 때는 아래 순서를 따르게 해주세요.

자동 반영 우선순위:
1. SL 먼저 배정
2. SL이 이미 소진되었으면 연차 배정

중요:
- SL(여성 보건휴가)은 한 달에 1번만 사용 가능하다는 정책을 반영
- 해당 월에 SL이 이미 사용되었으면 더 이상 SL 자동 배정하지 않음
- 그 다음부터는 연차로 자동 배정

즉,
자동 반영 결과는 아래처럼 읽히게 해주세요.
- 이번 달 SL 미사용 → SL 자동 배정 가능
- 이번 달 SL 이미 사용 → 연차 자동 배정

[5. 쉬는 날 신청을 안 했는데도 시스템이 배정한 경우 표시]
직원이 특정 날짜에 쉬는 날 신청을 하지 않았는데,
시스템이 운영 상황상 쉬는 날로 자동 배정하는 경우가 있을 수 있습니다.

이 경우에도 근태 신청 내역 또는 최종 반영 내역에서
그 이유가 명확히 보여야 합니다.

반드시 추가할 규칙:
- 직원이 쉬는 날 신청을 하지 않았는데 시스템이 쉬는 날을 자동 배정한 경우
- 반영 조정 사유에 아래 문구가 표시되게 해주세요

표시 문구:
- 적정 인원 대비 근무 인원 과다로 SL을 배정했습니다
- 적정 인원 대비 근무 인원 과다로 연차를 배정했습니다

즉,
시스템이 자동 배정한 이유가
운영 인원 과다 조정 때문이라는 점이 명확히 보여야 합니다.

중요:
- “신청하지 않았는데 왜 쉬는 날이 생겼지?”라는 의문이 없도록
- 신청 없는 자동 배정도 내역에서 분명히 설명되어야 함

[6. 신청 의사 / 최종 반영 / 반영 조정 사유 구분]
근태 신청 내역과 상세 보기에는 아래 3가지를 반드시 구분해서 보여주세요.

1. 신청 의사
예:
- 쉬는 날 희망
- 신청 없음
- 휴가 신청

2. 최종 반영
예:
- SL
- 연차
- 휴가
- 미반영

3. 반영 조정 사유
예:
- 적정 인원 대비 근무 인원 과다로 SL을 배정했습니다
- 적정 인원 대비 근무 인원 과다로 연차를 배정했습니다
- 신청 인원 초과로 자동 반영이 불가했습니다
- 직원 직접 휴가 신청이 반영되었습니다

중요:
- 사용자가 “내가 뭘 신청했는지”
- “결과가 뭘로 들어갔는지”
- “왜 그렇게 됐는지”
를 각각 이해할 수 있어야 합니다

[7. 신청하지 않은 자동 배정 건도 내역에 보여주기]
직원이 아무 신청을 하지 않았는데
시스템이 SL 또는 연차를 자동 배정한 경우에도
그 내역이 신청 내역 또는 반영 내역 리스트에 보이게 해주세요.

예시:
- 날짜: 4월 18일
- 신청 의사: 신청 없음
- 최종 반영: SL
- 반영 조정 사유: 적정 인원 대비 근무 인원 과다로 SL을 배정했습니다

중요:
- 직원 입장에서 “내 신청만 보이는 화면”이 아니라
- “내 최종 근태 반영 결과”까지 확인 가능한 화면처럼 보여야 합니다

[8. 휴가 신청 기능은 별도 유지]
휴가는 직원이 직접 신청하는 기능으로 유지해주세요.

즉,
근태 신청 화면 안에는 아래 두 흐름이 함께 존재할 수 있습니다.
1. 일반 쉬는 날 희망 신청
2. 별도 휴가 신청 또는 휴가 계획 신청

중요:
- 일반 쉬는 날 희망 결과를 시스템이 휴가로 자동 바꾸지 않기
- 휴가만큼은 직원 직접 신청이 필요하다는 점이 화면 구조에서 느껴져야 함

[9. 향후 근태관리와 연결될 여지]
지금은 근태 신청 화면만 수정하지만,
향후 근태관리에서 관리자가
자동 배정된 SL을 연차로 수동 조정할 수 있는 흐름과 연결될 수 있게
현재 근태 신청 내역에서도 최종 반영 유형이 명확히 보이게 해주세요.

즉,
지금 단계에서는 아래까지만 보여주면 됩니다.
- 최종 반영이 SL인지 연차인지 분명히 표시
- 반영 조정 사유도 함께 표시

[10. 최종 요청]
현재 근태 신청 화면을 아래 기준으로 수정해주세요.

- 일반 쉬는 날 신청은 날짜 희망만 제출하는 구조 유지
- 시스템이 자동으로 최종 반영할 수 있는 유형은 SL과 연차만 허용
- 휴가는 시스템 자동 배정 대상에서 제외하고, 직원 직접 신청만 가능하게 정리
- 자동 반영은 SL을 먼저 배정하고, 월 1회 SL이 소진되면 그 다음에 연차를 자동 배정
- 쉬는 날 신청이 없었는데도 시스템이 자동으로 쉬는 날을 배정한 경우,
  반영 조정 사유에 아래 문구가 뜨게 구성
  - 적정 인원 대비 근무 인원 과다로 SL을 배정했습니다
  - 적정 인원 대비 근무 인원 과다로 연차를 배정했습니다
- 신청 의사 / 최종 반영 / 반영 조정 사유를 명확히 구분해서 보여주기
- 신청하지 않은 자동 배정 건도 내역에서 확인 가능하게 정리

핵심은,
일반 쉬는 날 신청의 자동 결과는 SL 또는 연차만 가능하게 제한하고,
신청이 없더라도 운영상 자동 배정된 결과와 그 사유를 직원이 명확히 확인할 수 있게 만드는 것입니다.핵심: 일반 쉬는 날 신청의 시스템 자동 반영은 SL → 연차 순서만 허용하고, 신청이 없는데 자동 배정된 경우에도 “적정 인원 대비 근무 인원 과다로 SL(또는 연차)를 배정했습니다” 문구가 반드시 보이게 해주세요.
