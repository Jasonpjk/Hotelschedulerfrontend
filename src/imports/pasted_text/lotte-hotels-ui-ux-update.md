현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 기존 톤앤매너를 유지한 상태로,
아래 여러 화면의 상태 처리, 확정 흐름, 다운로드/이력 팝업, 편집 가능 조건을 함께 수정해주세요.

중요:
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 유지
- 넓은 여백, 얇은 선, 정돈된 그리드, 실무형 UI 유지
- 이번 수정은 화면별 기능 흐름과 상태 피드백을 명확히 만드는 것이 핵심
- 각 화면에서 “버튼을 눌렀을 때 실제 반영되었는지” 사용자가 즉시 알 수 있어야 함

[1. 호텔 설정 페이지 수정]
호텔 설정 페이지에서 최소 조별 인원 숫자를 수정한 후,
그 설정을 실제 운영 기준으로 반영하는 “확정” 버튼이 필요합니다.

반드시 반영할 것:
1) 최소 조별 인원 설정 영역 하단 또는 우측에 “확정” 버튼 추가
2) 사용자가 숫자를 변경한 후 확정 버튼을 눌러야 반영되는 구조로 보이게
3) 확정 후에는 아래와 같은 상태 피드백이 보여야 함
   - 확정 완료
   - 저장됨
   - 운영 기준 반영 완료
   중 하나의 방식으로 명확히 표시
4) 확정 전에는 “변경 사항 있음” 같은 안내가 보여도 좋음

추가로 필요한 기능:
5) 직원 계정 관련 영역에 “비밀번호 초기화” 버튼 추가
6) 비밀번호 초기화 버튼은 계정 관리 탭 또는 계정 리스트에서 관리자 액션으로 보이게
7) 버튼을 누르면 초기화 확인 팝업이 열리게
8) 팝업에는 아래 흐름이 보이게
   - 이 직원의 비밀번호를 초기화하시겠습니까?
   - 취소 / 초기화
9) 초기화 후에는 완료 메시지 또는 상태 피드백이 보여야 함

[2. 직원 관리 페이지 수정]
직원 관리 화면에서 “전체 확정”을 누른 뒤
정말 전체 확정이 반영되었다는 신호가 사용자에게 보여야 합니다.

반드시 반영할 것:
1) 전체 확정 버튼 클릭 후 명확한 반영 상태 표시 추가
2) 예:
   - 전체 확정 완료
   - 근무표 생성 기준 반영 완료
   - 마지막 반영 시각 표시
3) 버튼 자체 상태가 바뀌어도 좋음
   예:
   - 전체 확정 → 확정 완료
   - 버튼 비활성 + 완료 배지 표시
4) 화면 상단 또는 버튼 근처에 “반영 완료” 피드백을 분명하게 보여줄 것

중요:
- 단순히 버튼 클릭만 되고 끝나면 안 됨
- 사용자가 “지금 직원 관리 기준이 확정 반영되었구나”를 바로 이해할 수 있어야 함

[3. 근태 신청 페이지 수정]
근태 신청 화면에도 제출 흐름의 마지막 단계가 분명히 보여야 합니다.

반드시 반영할 것:
1) “최종 신청” 버튼 추가
2) 신청 작성/수정 상태와 최종 제출 상태가 구분되게
3) 최종 신청 버튼을 누른 후에는 아래 중 하나 이상의 피드백이 보여야 함
   - 최종 신청 완료
   - 신청 반영 완료
   - 제출됨
   - 버튼 상태 변경
4) 예:
   - 최종 신청 → 신청 완료
   - 버튼 색상/문구 변경
   - 완료 배지 또는 상단 알림 표시
5) 이미 최종 신청된 건은 사용자가 상태를 즉시 알 수 있어야 함

중요:
- 근태 신청도 “최종 신청”이라는 명확한 종료 액션이 필요함
- 신청이 반영되었는지 안 되었는지 버튼 상태 또는 완료 메시지로 전달되어야 함

[4. 수요 예측 페이지 수정]
수요 예측 화면에서는 “예측 결과” 탭과 “예측 근거” 탭의 액션이 명확히 구분되어야 합니다.

반드시 반영할 것:
1) 다운로드 버튼 클릭 시 다운로드 팝업 오픈
2) 이력 보기 버튼 클릭 시 이력 팝업 오픈
3) 하지만 이 두 기능은 “예측 결과” 탭에서만 활성화
4) “예측 근거” 탭에서는 다운로드 / 이력 보기 버튼이 비활성 또는 숨김 처리
5) 즉, 예측 결과만 다운로드 가능하게 구성

다운로드 팝업 권장 구성:
- 제목: 수요 예측 결과 다운로드
- 대상 월 표시
- 현재 선택 버전 또는 예측 결과 기준 표시
- 파일 형식: Excel(.xlsx)
- 취소 / 다운로드 버튼

이력 보기 팝업 권장 구성:
- 제목: 예측 결과 이력
- 수정/보정/반영 이력 리스트
- 날짜, 작업자, 변경 내용, 반영 상태 표시
- 닫기 버튼

중요:
- 예측 근거는 작업 입력/검토 단계
- 예측 결과만 최종 산출물이므로 다운로드와 이력 보기를 결과 탭에만 연결해야 함

[5. 수요 예측 화면 내 체크인/체크아웃 구분 강화]
현재 수요 예측 표에서 체크인(CI)과 체크아웃(CO) 영역이 있지만,
두 영역의 구분이 더 명확해야 합니다.

반드시 반영할 것:
1) 체크인 영역과 체크아웃 영역의 시각적 구분 강화
2) 예:
   - 그룹 헤더 배경톤 차이
   - 세로 구분선 강화
   - 상단 레이블 강조
   - 블록별 배경 구획 구분
3) 사용자가 한눈에
   - 왼쪽은 체크인
   - 오른쪽은 체크아웃
   을 바로 인지할 수 있어야 함
4) 같은 표 안에서도 두 영역이 섞여 보이지 않게 정리

중요:
- 수치 비교가 쉬워야 하므로
- 과한 장식보다 “구조적 구분”이 명확해야 함

[6. 근무표 페이지 수정 - 확정 전 락 셀 삭제]
현재 근무표 화면에서는 확정 버튼을 누르기 전에도
셀 편집이 안 되는 락 처리된 셀이 일부 존재하는데,
이 기능은 삭제해주세요.

반드시 반영할 것:
1) 확정 전에는 모든 스케줄 셀이 편집 가능하게 수정
2) 확정 전 락 처리된 셀 기능 제거
3) 확정 전에는 셀 클릭 시 수동 조정 팝업이 정상적으로 열리게
4) “락 처리된 셀” 같은 사전 잠금 인터랙션 제거

중요:
- 확정 전은 작업 중 버전이므로 자유롭게 수정 가능해야 함
- 락 기능은 확정 이후에만 적용되는 구조로 정리해주세요

[7. 근무표 페이지 수정 - 일일 근무 인원 적정 여부 표시]
근무표 하단에는 현재 “일일 근무 인원”이 표시되는데,
이 숫자가 적정 인원 대비 많은지 적은지도 같이 보여야 합니다.

반드시 반영할 것:
1) 하단 일일 근무 인원 행에 “적정 대비 상태” 추가
2) 예:
   - 적정
   - 부족
   - 초과
3) 또는 숫자와 함께 차이값 표시
   예:
   - 적정 인원 8명 / 실제 6명 → 2명 부족
   - 적정 인원 8명 / 실제 9명 → 1명 초과
4) 색상 또는 태그로 구분하면 좋음
   - 적정: 중립/초록
   - 부족: 경고/빨강
   - 초과: 주의/노랑 또는 중립 경고

중요:
- 단순 인원 숫자만 보는 것이 아니라
- 운영 기준에 비해 충분한지, 부족한지, 과한지 바로 보여야 함
- 최소 조별 인원 및 운영 기준과 연결된 느낌으로 보여야 함

[8. 버튼 실행 후 반영 상태 공통 원칙]
아래 버튼들은 모두 클릭 후 “반영 완료 신호”가 사용자에게 보여야 합니다.

적용 대상:
- 호텔 설정 > 최소 조별 인원 확정
- 직원 관리 > 전체 확정
- 근태 신청 > 최종 신청
- 근무표 > 확정
- 수요 예측 > 다운로드 / 이력 보기 액션

공통 원칙:
1) 클릭 후 상태 변화가 보여야 함
2) 버튼 문구 변경, 완료 배지, 토스트, 상단 상태 메시지 중 하나 이상 사용
3) “지금 반영되었는지” 사용자가 고민하지 않게 해야 함
4) 반영 시점 또는 마지막 저장 시각이 보이면 더 좋음

[9. 최종 요청]
현재 시스템을 아래 기준으로 수정해주세요.

- 호텔 설정:
  - 최소 조별 인원 반영용 확정 버튼 추가
  - 비밀번호 초기화 버튼 추가
  - 확정/초기화 후 완료 피드백 표시

- 직원 관리:
  - 전체 확정 후 실제 반영되었다는 표시 추가

- 근태 신청:
  - 최종 신청 버튼 추가
  - 신청 후 반영 완료 상태가 분명하게 보이게 수정

- 수요 예측:
  - 다운로드 버튼 → 다운로드 팝업 오픈
  - 이력 보기 버튼 → 이력 팝업 오픈
  - 이 기능은 예측 결과 탭에서만 활성화
  - 예측 근거 탭에서는 비활성 또는 숨김
  - 체크인/체크아웃 영역 구분 강화

- 근무표:
  - 확정 전 락 셀 기능 삭제
  - 확정 전에는 모든 셀 편집 가능
  - 하단 일일 근무 인원에 적정/부족/초과 상태 표시 추가

이 기준으로 각 화면의 버튼 실행 결과와 상태 피드백이 분명하게 보이도록 UI와 흐름을 정리해주세요.

핵심: 버튼을 누른 뒤 실제 반영되었다는 상태 신호가 보여야 하고, 수요 예측 다운로드/이력은 예측 결과 탭에서만 활성화, 근무표는 확정 전 모든 셀 편집 가능해야 합니다.
