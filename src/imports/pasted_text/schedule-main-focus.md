현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “근무표” 화면을 아래 기준으로 다시 수정해주세요.

중요:
- 이번 수정은 “근무표” 화면에만 적용해주세요.
- 다른 화면은 건드리지 말아주세요.
- 기존 시스템의 프리미엄 호텔 운영 관리자용 SaaS 톤앤매너 유지
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 / 넓은 여백 / 얇은 선 / 정돈된 그리드 유지

[현재 핵심 문제]
근무표 화면의 가운데 메인 스케줄 달력/그리드가 여전히 너무 좁아 보입니다.

원인:
- 화면 안에 너무 많은 요소를 한 페이지에 한 번에 넣으려고 해서
- 메인 스케줄 표가 압축되고
- 정작 가장 중요한 달력/그리드가 보조 영역처럼 보입니다.

중요:
이 화면은 모든 내용을 한 화면 안에 억지로 다 넣지 않아도 괜찮습니다.
가장 중요한 것은 메인 스케줄 달력/그리드가 제일 넓은 영역을 차지하는 것입니다.

[1. 최우선 원칙]
근무표 화면에서 가장 넓고 가장 중심적인 영역은 반드시
“가운데 메인 스케줄 달력/그리드”가 되게 해주세요.

즉,
- 이 화면의 주인공은 스케줄 표
- 나머지 요소는 보조 도구
- 스케줄 표가 가장 먼저 눈에 들어와야 함

[2. 한 화면에 모든 내용을 억지로 넣지 않기]
현재처럼 모든 메뉴/패널/보조 정보/이력을 한 화면에 과하게 다 펼쳐놓는 구조를 줄여주세요.

수정 방향:
- 모든 보조 요소를 동시에 크게 노출하지 않아도 됨
- 일부는 접기/펼치기 가능하게
- 일부는 높이를 줄이거나 요약형으로
- 일부는 탭/아코디언/슬림 패널로 정리

중요:
- “한 페이지에 다 보여주기”보다
- “메인 달력을 충분히 크게 보여주기”가 우선입니다

[3. 메인 스케줄 달력/그리드 영역 최대 확대]
메인 달력/그리드가 화면 중앙에서 가장 넓은 가로폭과 충분한 세로높이를 가지게 해주세요.

반드시 반영:
- 달력/표의 가로폭 최대 확보
- 달력/표의 세로높이 최대 확보
- 직원명, 날짜, 근무코드가 한눈에 더 잘 읽히게
- 표가 눌려 보이지 않게
- 스크롤은 표 중심으로 자연스럽게 작동하게

중요:
- 표가 카드 안에 답답하게 갇혀 보이면 안 됨
- 현재보다 훨씬 넓은 워크스페이스처럼 보여야 함

[4. 다른 메뉴/보조 영역은 최대한 공간 축소]
메인 달력을 제외한 다른 요소들은 최대한 공간을 줄여주세요.

공간 축소 대상 예시:
- 상단 요약 카드
- 범례 영역
- 운영 경고 영역
- AI 자연어 조정 입력 영역
- 하단 변경 이력
- 기타 설명성 패널

수정 방향:
- 높이 축소
- 한 줄 요약형으로 압축
- 필요 시 접기/펼치기
- 필요 시 탭 안으로 넣기
- 여백 과다 사용 줄이기

중요:
- 보조 영역이 메인 표 공간을 침범하면 안 됨
- 보조 정보는 보여도 되지만 메인 달력보다 존재감이 크면 안 됨

[5. 상단 영역 슬림화]
상단의 상태, 버전, 버튼, 요약 정보는
현재보다 더 컴팩트하게 정리해주세요.

예시:
- 헤더 높이 줄이기
- 상태 뱃지/버전/버튼을 한 줄에 정리
- KPI 카드형 대신 얇은 상태 바 또는 소형 요약 카드로 축소
- 근무코드 범례도 너무 큰 블록이 아니라 컴팩트한 한 줄형 또는 접을 수 있는 영역으로 정리

중요:
- 상단이 커질수록 표가 아래로 밀리므로
- 상단은 관리자용 도구 바처럼 슬림하게 정리하는 것이 우선입니다

[6. AI 자연어 조정과 변경 이력 영역 축소]
AI 자연어 조정과 변경 이력은 중요하지만 메인 달력을 압도하면 안 됩니다.

수정 방향:
- AI 자연어 조정 입력창은 한 줄 중심의 슬림한 도구 영역으로 축소
- 변경 이력은 너무 큰 리스트로 펼치지 말고 요약형으로 축소
- 필요 시 “전체 보기” 또는 접기/펼치기 구조 사용
- 하단 영역이 메인 표 높이를 잡아먹지 않게 조정

중요:
- AI 조정과 이력은 보조 기능
- 근무표 메인 달력이 충분히 확보된 뒤에 배치되어야 함

[7. 좌우 패널 존재감 최소화]
좌측/우측에 붙어 있는 필터, 경고, 설명, 부가 메뉴가 있다면
존재감을 줄이거나 접을 수 있게 정리해주세요.

원칙:
- 메인 표를 가로로 압축시키는 패널 금지
- 필요한 정보는 상단 얇은 바 또는 오버레이/팝업으로 처리 가능
- 표 폭이 가장 중요

[8. 스크롤 구조 재정리]
스크롤 때문에 메인 달력이 가려지거나
사용자가 표를 보려면 불필요하게 많이 내려가야 하는 구조를 피해주세요.

권장 방향:
- 먼저 메인 표가 보이게
- 보조 영역은 접거나 요약
- 표 내부 스크롤과 페이지 스크롤 역할 분리
- 사용자가 화면 진입 즉시 달력을 볼 수 있어야 함

[9. 최종 인상]
수정 후 사용자가 이 화면을 봤을 때 아래처럼 느껴져야 합니다.

- 이 화면은 근무표를 보는 화면이다
- 메인 스케줄 달력이 가장 중요하다
- 다른 기능들은 보조 도구로 정리되어 있다
- 전체 내용을 한 페이지에 다 밀어넣지 않아도 괜찮고, 핵심 작업 영역이 충분히 넓다

[10. 최종 요청]
현재 근무표 화면을 아래 기준으로 다시 수정해주세요.

- 가운데 메인 스케줄 달력/그리드가 화면에서 가장 넓은 영역을 차지하게 수정
- 한 페이지에 모든 내용을 억지로 다 넣으려는 구조를 줄이고, 메인 달력 중심으로 재배치
- 상단 요약/범례/경고/AI 조정/변경 이력 등 보조 영역은 최대한 공간을 줄이기
- 필요 시 접기/펼치기, 요약형, 슬림 바 형태로 정리
- 메인 표가 눌리거나 가려지지 않게 레이아웃 재정비
- 사용자가 화면에 들어왔을 때 가장 먼저, 가장 크게 스케줄 달력이 보이게 해주세요

핵심은,
근무표 화면에서 “메인 달력/그리드”가 압도적으로 가장 중요한 작업 영역처럼 보이게 만드는 것입니다.
중요: 다른 메뉴와 보조 정보는 일부 접혀 있거나 축소되어도 괜찮으니, 근무표 달력/그리드가 반드시 가장 넓고 가장 크게 보이게 해주세요.
