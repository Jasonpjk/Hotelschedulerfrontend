현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “직원 관리” 화면을 아래 기준으로 수정해주세요.

이번 수정은 단순 문구 변경이 아니라
테이블 고정 방식, 컬럼명 변경, 관리(수정) 연동 방식, 그리고 근무표 생성 반영 방식까지 포함한 기능 보완입니다.

[1. 테이블 고정 방식 수정]
직원 관리 테이블에서 아래 2가지는 스크롤과 관계없이 항상 고정되게 해주세요.

1) 표 가장 상단 헤더 행 고정
- 세로 스크롤을 내려도 컬럼 헤더가 계속 보이게
- 즉, sticky header 형태로 유지

2) 가장 왼쪽 “이름” 컬럼 고정
- 가로 스크롤을 움직여도 이름 컬럼은 계속 보이게
- 즉, sticky first column 형태로 유지

중요:
- 세로/가로 스크롤이 동시에 있어도
  상단 헤더 + 왼쪽 이름 컬럼이 모두 안정적으로 고정되어야 함
- 이름 컬럼과 헤더가 겹치거나 깨져 보이지 않게 정리
- 관리자용 실무 테이블답게 안정적이고 읽기 편하게 구성

[2. 컬럼명 변경]
현재 테이블과 관리(수정) 화면에 있는
“배치 가능 조”라는 표현을 모두 아래로 변경해주세요.

변경 후:
- 배치 그룹

즉,
- 테이블 컬럼명도 “배치 그룹”
- 수정 팝업/관리 화면의 라벨도 “배치 그룹”
으로 통일해주세요.

[3. 관리(수정) 화면과 테이블 연동]
직원 관리에서 수정(관리) 화면으로 들어가
직원의 조를 배치하면,
그 배치된 조가 메인 테이블의 “배치 그룹” 컬럼에 바로 표시되게 해주세요.

의도:
- 단순히 가능한 조를 나열하는 개념보다
- 실제 이 직원이 현재 어떤 그룹으로 운영상 분류되어 있는지 보이게 하는 구조로 가고 싶습니다.

즉,
관리(수정)에서 아래와 같이 조를 지정하면
- 오전조
- 오후조
- 야간조
- 중간조

그 결과가 테이블의 “배치 그룹” 컬럼에 그대로 보이게 해주세요.

표시 방식:
- 기존 시스템의 태그/칩 스타일 유지
- 예: 오전조 / 오후조 / 야간조 / 중간조
- 여러 개가 배치될 수 있으면 다중 태그로 표시
- 단, 현재 화면 문맥상 “운영상 배치된 그룹”처럼 읽히도록 정리

[4. 역할 + 배치 그룹 설정 완료 후 전체 확정 기능 추가]
직원 관리 화면에서
모든 직원의 역할과 배치 그룹을 설정한 뒤
한 번에 확정하는 기능을 추가해주세요.

필수 기능:
- 화면 상단 또는 우상단에 “전체 확정” 버튼 추가
- 이 버튼은 직원별 역할 / 배치 그룹 설정이 끝난 후
  일괄 반영하는 최종 액션 버튼 역할

버튼 기능:
- 전체 확정 버튼 클릭 시
  현재 직원 관리 화면에서 설정한 모든 직원의
  역할 + 배치 그룹 정보가
  근무표 생성에 반영되는 기준값으로 일괄 저장되도록 표현

즉,
직원 관리 화면은 단순 조회 화면이 아니라
근무표 생성 전에 직원 마스터를 확정하는 사전 세팅 화면처럼 보여야 합니다.

[5. 전체 확정 UX]
전체 확정은 실수로 바로 반영되지 않도록
관리자용 SaaS답게 아래 흐름을 고려해주세요.

권장 흐름:
1. 역할 / 배치 그룹 수정 완료
2. 전체 확정 버튼 클릭
3. 확인 팝업 또는 확인 문구 노출
4. “이 설정을 근무표 생성 기준으로 반영합니다” 안내
5. 최종 확정 시 일괄 반영

확인 팝업 안에 들어가면 좋은 문구 예시:
- 현재 설정된 역할 및 배치 그룹을 근무표 생성 기준으로 반영합니다.
- 확정 후에는 다음 근무표 생성 시 이 기준이 우선 적용됩니다.

[6. 화면 내 상태 표현]
가능하면 전체 확정 전/후 상태가 보이게 해주세요.

예시:
- 미확정
- 일부 설정 필요
- 확정 완료

또는
- 전체 확정 전에는 상단에 안내 배너
- 확정 후에는 “근무표 생성 반영 완료” 상태 표시

중요:
사용자가 이 화면에서
“아직 근무표 생성 기준으로 확정되지 않았는지”
“이미 확정되어 반영 준비가 끝났는지”
한눈에 알 수 있어야 합니다.

[7. 테이블 구조 유지 방향]
기존 직원 관리 테이블의 실무형 구조는 유지하되
아래 방향으로 보완해주세요.

유지할 것:
- 이름
- 직급
- 역할
- 배치 그룹
- 입사일
- 전체 근속
- 프런트 시작일
- 현재 프런트 근속
- 누적 프런트 경력 등 기존 핵심 정보

수정할 것:
- “배치 가능 조” → “배치 그룹”
- 관리(수정)에서 설정된 그룹이 실제 테이블에 표시
- 헤더와 이름 컬럼 고정
- 전체 확정 액션 추가

[8. 톤앤매너 유지]
반드시 현재까지 만든 LOTTE HOTELS & RESORTS 관리자 SaaS의 톤앤매너를 그대로 유지해주세요.

유지 조건:
- 프리미엄 호텔 운영 관리자용 SaaS 무드
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트
- 얇은 선, 넓은 여백, 정돈된 그리드
- 소비자용 HR 시스템 느낌 금지
- 운영팀이 실제 쓰는 업무형 인터페이스처럼 보이게
- 표, 태그, 버튼, 상태칩 모두 기존 컴포넌트 체계와 자연스럽게 이어지게

[9. 최종 의도]
이 화면은 단순 직원 목록이 아니라
근무표 생성을 위한 “직원 운영 기준 확정 화면”처럼 보여야 합니다.

즉, 사용자는 이 화면에서
- 직원 역할 설정
- 배치 그룹 설정
- 전체 확정
을 완료하고
그 결과가 이후 근무표 생성에 반영된다는 흐름을 명확히 이해할 수 있어야 합니다.

최종 요청:
현재 직원 관리 화면을 아래 기준으로 수정해주세요.

- 표 상단 헤더 고정
- 가장 왼쪽 이름 컬럼 고정
- “배치 가능 조”를 “배치 그룹”으로 변경
- 관리(수정)에서 설정한 조가 “배치 그룹” 컬럼에 표시
- 모든 직원의 역할과 배치 그룹 설정 후
  “전체 확정” 버튼으로 일괄 확정 가능
- 이 확정 내용이 근무표 생성 기준으로 반영되는 흐름을 화면에 명확히 보여주기

이 기준으로 UI와 기능 흐름을 정리해주세요.
