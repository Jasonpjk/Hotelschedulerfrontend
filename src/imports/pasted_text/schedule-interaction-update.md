현재 LOTTE HOTELS & RESORTS 관리자용 SaaS의 “근무표” 화면에서
확정된 셀의 인터랙션 상태와 확정 이후 수정 흐름을 함께 수정해주세요.

중요:
- 이번 수정은 “근무표 화면”에만 적용해주세요.
- 다른 화면은 건드리지 말아주세요.
- 기존 톤앤매너는 그대로 유지해주세요.
- 딥 네이비 / 웜 화이트 / 절제된 골드 포인트 / 얇은 선 / 정돈된 그리드 유지

[1. 현재 문제 1: 확정된 셀 인터랙션 불일치]
지금 근무표 화면에서
확정 버튼을 누르기 전에는
이미 락 처리된 셀에 마우스를 올렸을 때
- 커서가 손가락(pointer)로 바뀌지 않고
- 클릭 가능해 보이지 않으며
- “락 처리된 셀” 같은 안내가 뜨는 상태입니다.

그런데 확정 버튼을 누른 후에는
확정된 셀에 마우스를 올렸을 때
- 커서가 손가락(pointer)처럼 보여서 클릭 가능해 보이고
- 별도 메시지는 뜨지 않습니다.

즉,
확정 전의 락 셀과
확정 후의 락 셀 인터랙션이 서로 다르게 보입니다.

이 상태는 사용자가
“지금 이 셀이 수정 가능한지 불가능한지”
헷갈리게 하므로 반드시 통일해주세요.

[2. 현재 문제 2: 확정 후 버튼이 사라짐]
지금은 확정 버튼을 누르면
상단의 “확정” 버튼이 사라지는데,
운영상 확정 이후에도 다시 수정이 필요할 수 있으므로
확정 이후에도 수정할 수 있는 흐름이 남아 있어야 합니다.

즉,
“확정하면 끝”이 아니라
“확정 후에도 다시 수정 모드로 전환 가능”
구조가 필요합니다.

[3. 확정된 셀 / 락 처리된 셀 인터랙션 통일]
확정된 셀과 락 처리된 셀은
항상 같은 인터랙션 규칙으로 보여야 합니다.

반드시 통일할 것:
1) 마우스 오버 시 커서
- 손가락(pointer) 커서로 바뀌지 않게 해주세요
- 클릭 가능한 셀처럼 보이면 안 됩니다
- 기본 커서 또는 비활성 상태 커서처럼 보여야 합니다

2) hover 상태
- hover 시 클릭 가능한 버튼 느낌이 나면 안 됩니다
- 필요하면 잠금 상태를 보여주는 정도의 미세한 변화만 허용

3) tooltip / 안내 문구
- 안내 문구를 띄울 경우 아래처럼 통일해주세요
  예:
  - 락 처리된 셀
  - 확정된 셀은 수정할 수 없습니다
- 확정 전/후가 서로 다른 메시지 체계가 되면 안 됩니다

4) 클릭 반응
- 클릭해도 팝업이 열리지 않음
- 셀 편집이 시작되지 않음
- 클릭 가능한 피드백도 보이지 않게

[4. 확정 버튼은 사라지지 않게 유지]
확정 버튼을 누른 뒤에도
상단의 확정 관련 액션 영역이 사라지지 않게 해주세요.

수정 방향:
- 확정 후 버튼이 없어지지 않게 유지
- 대신 상태에 따라 버튼 구성이 달라지는 구조로 정리

권장 방식:
확정 전:
- 확정 버튼 표시

확정 후:
- 확정 완료 상태 표시
- 수정 모드 진입 또는 재편집 관련 버튼 표시
- 필요하면 재확정 가능 흐름 유지

즉,
확정 이후에도 사용자가
“이 버전을 다시 수정할 수 있는 방법”
을 바로 이해할 수 있어야 합니다.

[5. 확정 후 수정 흐름 추가]
확정된 근무표라도
운영상 다시 수정해야 할 일이 생길 수 있으므로
확정 후 다시 수정 가능하게 해주세요.

권장 UX:
1. 사용자가 확정 버튼 클릭
2. 상태값이 “확정”으로 변경
3. 셀은 잠금 상태로 전환
4. 상단에 “수정” 또는 “편집 재개” 버튼 표시
5. 사용자가 수정 버튼을 누르면 다시 편집 가능 상태로 전환
6. 수정 후 다시 확정 가능

즉,
“확정 → 잠금 → 필요 시 수정 모드 진입 → 다시 확정”
흐름이 가능해야 합니다.

[6. 수정 버튼 또는 편집 재개 버튼 추가]
확정 이후 상단 액션 영역에
아래 중 하나를 추가해주세요.

권장 버튼명:
- 수정
- 편집 재개
- 확정 해제 후 수정

가장 자연스러운 관리자용 표현으로 정리해주세요.

중요:
- 이 버튼은 확정 이후에만 보이게
- 누르면 셀 편집이 다시 가능해져야 함
- 확정 상태를 해제하고 작업 상태로 돌아가는 흐름이 직관적으로 보여야 함

[7. 상태 흐름을 명확히 보여주세요]
근무표 상단 상태는 아래 흐름이 읽히게 정리해주세요.

상태 흐름 예시:
- 작업 중
- 확정
- 수정 중

또는
- 확정됨
- 편집 재개됨

중요:
사용자가 지금 이 버전이
- 수정 가능한 상태인지
- 잠금 상태인지
- 다시 수정 중인지
즉시 알 수 있어야 합니다.

[8. 확정 후 수정 시 셀 잠금 해제]
확정 상태에서는 셀이 잠겨야 하지만,
사용자가 상단의 수정/편집 재개 버튼을 누르면
아래가 다시 가능해야 합니다.

- 셀 클릭 가능
- 수동 조정 팝업 열림
- 편집 가능한 커서/인터랙션 복구
- AI 자연어 조정 사용 가능

즉,
확정 상태와 수정 가능 상태가 명확히 전환되어야 합니다.

[9. 안내 문구도 같이 정리]
확정 상태에서는 예를 들어 아래 문구가 보이게 할 수 있습니다.
- 이 버전은 확정되어 직접 수정할 수 없습니다
- 수정이 필요하면 상단의 수정 버튼을 눌러 편집을 재개하세요

수정 모드로 전환되면 문구도 바뀌게 해주세요.
예:
- 현재 버전은 수정 중입니다
- 수정 후 다시 확정해주세요

[10. 최종 요청]
현재 근무표 화면을 아래 기준으로 수정해주세요.

- 확정 전의 락 셀과 확정 후의 확정 셀 인터랙션을 동일하게 통일
- 확정된 셀/락 처리된 셀은 손가락 커서가 뜨지 않게 하고, 동일한 잠금 안내 문구와 동일한 비활성 인터랙션 적용
- 확정 버튼을 누른 후 버튼이 사라지지 않게 수정
- 확정 이후에도 다시 수정할 수 있도록 상단에 수정/편집 재개 버튼 추가
- 수정 버튼을 누르면 셀 편집과 수동 조정 팝업이 다시 가능해지게 전환
- 확정 / 잠금 / 수정 재개 흐름이 화면에서 명확히 보이게 정리

핵심은,
확정 이후에도 운영상 다시 수정할 수 있어야 하며,
잠금 상태와 수정 가능 상태의 인터랙션이 일관되고 명확해야 한다는 점입니다.

추가 핵심: 확정 후 버튼이 사라지면 안 됩니다. 확정 상태를 유지하되, 상단에서 다시 수정 모드로 전환할 수 있는 버튼을 반드시 남겨주세요.
