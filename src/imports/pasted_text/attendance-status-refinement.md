현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS의 “근태 관리” 화면에서
상태 용어와 검토 기준이 헷갈리지 않도록 정리해줘.

이번 작업의 핵심은
“검토 필요”와 “미반영”을 같은 의미처럼 쓰지 않고,
관리자가 실제로 판단해야 하는 건과 근무표에 반영되지 않은 건을 명확히 구분하는 것이야.

--------------------------------------------------
[현재 문제]
--------------------------------------------------

현재 근태 관리 화면에서 아래 문제가 있음.

1) SL이 자동 반영된 건을 연차로 변경할지 관리자가 검토해야 하는데,
   이 건이 “검토 필요” 집계에 포함되지 않음

2) 현재 “검토 필요”가 사실상 “미반영” 건만 의미하는 것처럼 보임

3) 상세 검토 화면에 들어가면 상태에 “검토 필요”가 아니라
   미반영된 건만 나와서 용어가 헷갈림

4) 사용자가 보기에는
   “검토 필요”가 무엇을 의미하는지,
   “미반영”과 어떻게 다른지 명확하지 않음

--------------------------------------------------
[수정 목표]
--------------------------------------------------

근태 관리의 상태 체계를 아래처럼 명확히 분리해줘.

1) 자동 반영 완료
- 시스템이 정책에 따라 근무표에 자동 반영한 상태
- 예: 여직원 쉬는 날 희망 → SL 자동 반영
- 예: 연차로 자동 반영
- 예: 교육 승인 후 EDU 반영
- 예: 병가 승인 후 SICK 반영

2) 관리자 조정 필요
- 근무표에는 일단 자동 반영되었지만
  관리자가 최종 유형을 조정할 수 있는 상태
- 예: SL로 자동 반영되었지만 관리자가 연차로 바꿀지 검토해야 하는 건
- 예: SL 사용 가능 정책상 자동 반영되었지만 운영상 연차 처리 여부를 판단해야 하는 건

3) 미반영
- 신청은 존재하지만 근무표에 아직 실제 코드로 반영되지 않은 상태
- 예: 최소 인원 부족으로 OFF/VAC/SL/EDU/SICK 반영 실패
- 예: 충돌 때문에 자동 반영하지 못한 건

4) 승인 대기
- 관리자 승인 또는 확인이 아직 필요한 신청 접수 상태

5) 반려
- 관리자가 반려한 상태

6) 신청 취소
- 직원이 신청을 취소한 상태

--------------------------------------------------
[검토 필요의 의미 재정의]
--------------------------------------------------

“검토 필요”는 단순히 미반영만 뜻하면 안 됨.

검토 필요에 포함해야 하는 건:
- 관리자 조정 필요
- 미반영
- 충돌 발생
- 승인 대기 중 관리자가 봐야 하는 건
- SL 자동 반영 후 연차 전환 여부 판단이 필요한 건
- 최소 인원 부족으로 자동 반영 실패한 건
- 정책상 예외 여부 판단이 필요한 건

즉, “검토 필요”는 상위 카테고리이고
그 안에 여러 세부 상태가 있어야 함.

검토 필요 = 관리자 조정 필요 + 미반영 + 충돌 + 승인 대기 검토 대상

--------------------------------------------------
[상단 요약 카드 수정]
--------------------------------------------------

근태 관리 화면 상단 카드의 의미를 아래처럼 정리해줘.

기존 카드 예:
- 전체 신청
- 자동 반영 완료
- 검토 필요
- 미반영 건수

수정 방향:
1) 전체 신청
- 모든 신청 건수

2) 자동 반영 완료
- 근무표에 자동 반영이 끝난 건수

3) 검토 필요
- 관리자가 판단해야 하는 전체 건수
- 관리자 조정 필요 + 미반영 + 충돌 + 승인 대기 검토 대상 포함

4) 미반영 건수
- 실제 근무표에 아직 반영되지 않은 건수만 표시

중요:
- “검토 필요”와 “미반영 건수”는 숫자가 다를 수 있음
- SL 자동 반영 후 연차 변경 검토 건은 “검토 필요”에는 포함되지만 “미반영”에는 포함되지 않아야 함
- 미반영은 검토 필요의 일부일 수 있지만 같은 개념은 아님

--------------------------------------------------
[직원별 목록 컬럼 수정]
--------------------------------------------------

직원별 근태 관리 표에서 컬럼 의미를 명확히 해줘.

현재 컬럼 예:
- 직원명
- 성별
- 신청 건수
- 신청 날짜 요약
- 자동 반영
- 검토 필요
- 미반영
- 휴가 잔여
- 연차 잔여
- 관리

수정 방향:
- 자동 반영: 실제 자동 반영 완료 건수
- 조정 필요: 자동 반영은 되었지만 관리자가 유형 변경을 검토해야 하는 건수
- 미반영: 근무표에 아직 반영되지 않은 건수
- 검토 필요 합계: 조정 필요 + 미반영 + 충돌/승인 대기 건수

표 컬럼을 너무 많이 늘리기 어렵다면 아래처럼 정리해도 됨.

권장 컬럼:
- 직원명
- 성별
- 신청 건수
- 신청 날짜 요약
- 자동 반영
- 조정 필요
- 미반영
- 휴가 잔여
- 연차 잔여
- 관리

그리고 상단 요약 카드에서 “검토 필요”는 조정 필요 + 미반영 등을 합산한 값으로 표시.

--------------------------------------------------
[상세 검토 모달 수정]
--------------------------------------------------

직원 상세 검토 모달에서 상태 표시를 더 명확하게 바꿔줘.

현재 문제:
- 상세 검토에 들어가면 “검토 필요”라고 생각한 건이 실제로는 미반영만 보여서 헷갈림

수정 요구:
상세 검토 모달 안에서 각 신청 건의 상태를 아래처럼 구분해서 표시해줘.

상태 예:
- 자동 반영 완료
- 관리자 조정 필요
- 미반영
- 승인 대기
- 반려
- 신청 취소

SL 자동 반영 건 중 연차 변경 검토가 필요한 경우:
- 최종 반영: SL
- 상태: 관리자 조정 필요
- 반영 조정 사유:
  “여직원 월 1회 SL 사용 가능 정책에 따라 SL로 자동 반영되었습니다. 관리자가 필요 시 연차로 변경할 수 있습니다.”
- 조정 액션:
  “연차로 변경”

즉, SL로 이미 반영되었으므로 “미반영”이 아님.
하지만 관리자가 연차 전환 여부를 판단해야 하므로 “관리자 조정 필요”임.

--------------------------------------------------
[SL → 연차 변경 검토 로직]
--------------------------------------------------

SL 자동 반영 후 연차 변경 가능 건은 아래처럼 처리해줘.

조건:
- 직원 성별이 여
- 쉬는 날 희망 신청
- 해당 월 SL 사용 가능
- 시스템이 SL로 자동 반영
- 관리자가 연차로 변경할 수 있는 정책이 존재

표시:
- 최종 반영: SL
- 상태: 관리자 조정 필요
- 조정 액션: 연차로 변경

관리자가 “연차로 변경”을 누르면:
- 최종 반영 코드: SL → 연차
- 휴가/연차 잔여 수량 재계산
- 변경 이력에 기록
- 직원 화면에는 “관리자가 SL을 연차로 수정했습니다.” 정도로 표시

--------------------------------------------------
[미반영 상태 정의]
--------------------------------------------------

미반영은 아래 경우에만 사용해줘.

- 신청은 들어왔지만 근무표 셀에 아무 코드도 반영되지 않은 경우
- 최소 인원 부족으로 자동 반영하지 못한 경우
- 이미 다른 승인/확정 코드와 충돌해서 반영하지 못한 경우
- 운영 정책상 자동 반영이 보류된 경우

미반영 건의 상태:
- 상태: 미반영
- 사유: 최소 운영 인원 부족 / 충돌 / 정책 검토 필요 등
- 조정 액션: 수동 반영 / 반려 / 다른 날짜 검토 등

--------------------------------------------------
[용어 정리]
--------------------------------------------------

화면에서 용어를 아래처럼 통일해줘.

- 검토 필요: 관리자가 확인하거나 조정해야 하는 상위 상태
- 관리자 조정 필요: 이미 반영되었지만 유형 변경 등 관리자의 판단이 필요한 상태
- 미반영: 근무표에 아직 반영되지 않은 상태
- 자동 반영 완료: 시스템이 정책에 따라 근무표에 반영한 상태

혼동 방지를 위해 tooltip 또는 설명을 추가해도 좋음.

예:
검토 필요 tooltip:
“관리자가 확인해야 하는 건입니다. 미반영 건, 조정 필요 건, 충돌 건이 포함됩니다.”

미반영 tooltip:
“신청은 접수되었으나 아직 근무표에 반영되지 않은 건입니다.”

관리자 조정 필요 tooltip:
“근무표에는 반영되었지만 관리자가 최종 유형을 변경할 수 있는 건입니다.”

--------------------------------------------------
[필터 수정]
--------------------------------------------------

근태 관리 화면에 상태 필터가 있다면 아래처럼 정리해줘.

필터:
- 전체
- 자동 반영 완료
- 검토 필요
- 관리자 조정 필요
- 미반영
- 승인 대기
- 반려
- 신청 취소

“검토 필요” 필터를 선택하면:
- 관리자 조정 필요
- 미반영
- 충돌
- 승인 대기 검토 대상
이 모두 포함되어야 함.

“미반영” 필터를 선택하면:
- 실제 근무표 미반영 건만 보여야 함.

--------------------------------------------------
[대시보드와의 연결]
--------------------------------------------------

대시보드의 “근태 검토” 또는 “검토 필요” 숫자도 같은 기준으로 맞춰줘.

대시보드 검토 필요:
- 관리자 조정 필요
- 미반영
- 충돌
- 승인 대기 검토 대상
을 포함

대시보드 미반영:
- 실제 근무표에 반영되지 않은 건만 포함

즉, 대시보드와 근태 관리의 숫자 기준이 달라지면 안 됨.
