현재 LOTTE HOTELS & RESORTS 프런트 스케줄 SaaS의 “직원 관리” 화면에서
직원별 조 배치 구조를 수정해줘.

이번 작업의 핵심은
직원 관리 화면에서는 직원의 기본 배치 그룹을 “오전조 / 오후조 / 야간조” 3개만 관리하고,
중간조는 직원에게 고정 배치하지 않는 구조로 정리하는 것이야.

--------------------------------------------------
[현재 문제]
--------------------------------------------------

현재 직원 관리 화면의 “조 배치” 컬럼에 아래처럼 중간조가 직원별 배치값처럼 표시되고 있음.

예:
- 오전조 + 중간조
- 야간조 + 중간조
- 오후조 + 중간조

하지만 이 구조는 운영 의도와 맞지 않아.

중간조는 직원의 고정 배치 그룹이 아니라,
오전조/오후조/야간조의 기본 인원과 인차지 배치가 먼저 충족된 후
체크인/체크아웃 수요에 따라 자동으로 C08, C09, C10 등에 배정되는 보조 근무 코드여야 함.

--------------------------------------------------
[수정 목표]
--------------------------------------------------

직원 관리 화면에서는 조 배치 값을 아래 3개만 사용하게 해줘.

- 오전조
- 오후조
- 야간조

중간조는 직원 관리 화면의 기본 조 배치값에서 제거해줘.

즉, 직원 관리 표의 “조 배치” 컬럼에는 아래처럼만 보여야 함.

예:
- 오전조
- 오후조
- 야간조

아래처럼 보이면 안 됨.
- 오전조 + 중간조
- 오후조 + 중간조
- 야간조 + 중간조
- 중간조 단독

--------------------------------------------------
[중간조의 역할 정의]
--------------------------------------------------

중간조는 직원의 고정 그룹이 아니라
근무표 자동생성 시 수요예측에 따라 자동 배정되는 근무 코드로 처리해줘.

중간조 코드:
- C08
- C09
- C10
- C11

중간조 배정 순서:
1) 먼저 오전조 / 오후조 / 야간조의 기본 필요 인원 충족
2) 각 조별 인차지 필요 조건 충족
3) 최소 조별 인원 조건 충족
4) 그다음 체크인 / 체크아웃 수요예측을 보고 중간조 필요 여부 판단
5) 수요에 따라 C08 / C09 / C10 / C11 중 적절한 코드 자동 배정

즉:
- 중간조는 직원 관리에서 “소속 조”처럼 지정하는 값이 아님
- 중간조는 자동 스케줄링 결과로 특정 날짜에 배정되는 근무 코드임

--------------------------------------------------
[직원 관리 설정 팝업 수정]
--------------------------------------------------

직원 설정 팝업이나 직원 추가/수정 모달에서
주 배치 그룹 선택 항목을 아래 3개만 남겨줘.

주 배치 그룹:
- 오전조
- 오후조
- 야간조

만약 현재 “중간조 가능 여부” 같은 항목이 있다면,
그 의미를 명확히 분리해줘.

권장:
- 주 배치 그룹: 오전조 / 오후조 / 야간조
- 중간조 자동 배정 가능 여부: 가능 / 불가

단, “중간조 자동 배정 가능 여부”는 직원의 기본 조 배치가 아니라
수요예측 기반 자동편성에서 C08~C11 배정을 허용할지 여부를 나타내는 보조 옵션임.

직원 관리 표에는 기본적으로 “조 배치” 컬럼에는 주 배치 그룹만 표시하고,
필요하다면 상세 설정 안에서만 “중간조 자동 배정 가능” 여부를 관리해줘.

--------------------------------------------------
[직원 관리 표 표시 방식]
--------------------------------------------------

직원 관리 표의 “조 배치” 컬럼은 아래처럼 간결하게 표시해줘.

예:
- 김민준: 오전조
- 이서연: 오후조
- 박지호: 오전조
- 최유진: 야간조
- 정현우: 오후조

중간조 가능 여부를 표에서 꼭 보여줘야 한다면
조 배치 컬럼 안에 섞지 말고 별도 작은 보조 표시로 분리해줘.

예:
- 조 배치: 오전조
- 보조 가능: 중간조 가능

하지만 기본 권장안은
직원 관리 표에서는 오전조 / 오후조 / 야간조만 보이게 하는 것임.

--------------------------------------------------
[근무표 자동생성 로직과의 관계]
--------------------------------------------------

근무표 자동생성 시에는 아래 원칙으로 처리해줘.

1) 직원의 주 배치 그룹을 먼저 참고
- 오전조 직원은 M07 우선 후보
- 오후조 직원은 A13 우선 후보
- 야간조 직원은 N22 우선 후보

2) 기본 인원과 인차지 조건을 먼저 채움
- 최소 조별 인원
- 각 조별 인차지 1명 이상
- 운영정책 기준

3) 그다음 수요예측 결과를 기반으로 중간조 필요 여부 판단
- 체크인 수요
- 체크아웃 수요
- 피크 시간
- 중간조 필요 인원

4) 중간조가 필요하면 직원 풀에서 자동 배정
- C08
- C09
- C10
- C11

5) 중간조 배정 시 고려할 조건
- 해당 직원이 중간조 자동 배정 가능인지
- 전후 근무 강도
- A13 다음날 M07 회피
- 연속근무
- 휴가/병가/교육/SL/HO 충돌 여부
- 공정성
- 전월 근무강도 회복 반영

--------------------------------------------------
[근무표 범례는 유지]
--------------------------------------------------

중요:
직원 관리 화면에서 중간조를 기본 배치 그룹에서 제거하더라도
근무표 화면의 중간조 코드와 범례는 삭제하면 안 됨.

근무표 범례에는 아래 코드가 계속 유지되어야 함.

- C08 중간조
- C09 중간조
- C10 중간조
- C11 중간조

즉:
- 직원 관리의 조 배치에서는 중간조 제거
- 근무표의 근무 코드로서 중간조는 유지

이 둘을 혼동하지 말 것.

--------------------------------------------------
[연간 야간조 계획과의 관계]
--------------------------------------------------

연간 야간조 계획은 야간조 대상자 사전 계획 기능이므로
기본적으로 야간조 그룹과 연동하면 됨.

중간조는 연간 야간조 계획 대상이 아님.
중간조는 해당 월/일자의 수요예측에 따른 자동 배정 결과임.

따라서 연간 야간조 계획 화면이나 대상자 선택 팝업에
중간조를 고정 그룹처럼 표시하지 마.

--------------------------------------------------
[수요 예측과의 관계]
--------------------------------------------------

수요 예측 결과에서 체크인/체크아웃 수요가 많을 때
중간조 배치가 자동으로 필요해질 수 있음.

수요 예측 결과 예:
- 체크아웃 피크가 오전에 크면 C08 / C09 검토
- 체크인 피크가 오후에 크면 C10 / C11 검토
- 전체 객실 수요와 시간대별 부하에 따라 중간조 인원 자동 추천

이 로직은 직원 관리의 조 배치값과 별개로 동작해야 함.

--------------------------------------------------
[디자인 수정]
--------------------------------------------------

직원 관리 화면의 “조 배치” 컬럼에서
중간조 배지가 같이 붙어 있던 디자인을 제거해줘.

기존:
- 오전조 + 중간조
- 오후조 + 중간조
- 야간조 + 중간조

수정:
- 오전조
- 오후조
- 야간조

디자인 톤은 기존 LOTTE HOTELS & RESORTS 관리자 SaaS 톤 유지:
- 딥 네이비
- 웜 화이트
- 절제된 골드 포인트
- 얇은 테두리
- 정돈된 표 레이아웃

새로운 스타일로 만들지 말고 기존 배지 스타일을 유지하되
중간조 배지만 제거해줘.

--------------------------------------------------
[데이터 구조 정리]
--------------------------------------------------

직원 데이터에서 조 배치 관련 필드를 아래처럼 구분해줘.

권장:
- primary_shift_group: morning / afternoon / night
- mid_shift_eligible: true / false

표시:
- 직원 관리 표의 조 배치 컬럼: primary_shift_group만 표시
- 직원 상세/설정 팝업: mid_shift_eligible을 별도 옵션으로 관리 가능

중요:
- primary_shift_group에 mid 또는 middle 값을 넣지 말 것
- 중간조는 근무표 생성 결과의 shift_code로만 사용
- 예: schedule_cell.shift_code = C08 / C09 / C10 / C11
