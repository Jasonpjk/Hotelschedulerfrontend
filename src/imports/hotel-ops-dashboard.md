프리미엄 호텔 브랜드 무드의 관리자용 웹앱 대시보드를 디자인해줘.

중요 공통 방향:
- 일반 소비자용 예약 사이트가 아니라 호텔 운영 관리자용 SaaS 대시보드
- 롯데호텔 홈페이지가 주는 고급스럽고 정제된 톤앤매너를 참고
- 과장된 장식보다 절제된 럭셔리, 신뢰감, 깔끔한 정보 구조
- 컬러는 딥 네이비, 차콜, 웜 화이트, 골드 포인트를 사용
- 너무 번쩍이거나 과한 금색 느낌은 금지
- 표, 카드, 폼, 탭, 모달, 상태 배지 중심의 관리자 화면
- 가독성이 매우 좋아야 하고 실제 업무에 쓰기 편해야 함
- 넓은 여백, 정돈된 그리드, 얇은 선, 세련된 타이포그래피
- 데스크톱 우선 디자인
- 반응형을 고려하되 이번 시안은 데스크톱 중심
- 전체 분위기는 “호텔 브랜드급 운영 툴”
- 화려한 마케팅 랜딩페이지처럼 보이면 안 됨
- 호텔 예약 사이트나 멤버십 사이트처럼 보이면 안 됨
- 스타트업스럽고 캐주얼한 느낌보다 품격 있고 안정적인 엔터프라이즈 운영툴 느낌으로 설계

프로젝트 맥락:
- 이 서비스는 호텔 프런트 직원 스케줄 근무표를 생성하고 관리하는 운영 시스템
- 로그인 후 처음 진입하는 화면은 대시보드
- 대시보드는 메인 허브 화면이며, 핵심 기능 페이지로 자연스럽게 이동할 수 있어야 함
- 가장 중요한 핵심 기능은 월별 근무표 생성/조회/수정이므로, 대시보드는 그 핵심 화면으로 연결되는 구조여야 함
- 현재 1차 구현 범위는 호텔 관리자용 운영 웹앱 중심
- 향후 직원용 스케줄 조회/변경신청 기능이 추가될 수 있으므로 확장 가능한 관리자 구조 느낌이 있으면 좋음

디자인 목표:
- 로그인 직후 보는 고급 관리자 홈 화면
- 현재 운영 상태를 한눈에 보고, 바로 다음 액션을 할 수 있는 구조
- “오늘 무엇을 확인해야 하는지”가 명확하게 보여야 함
- 시각적으로 고급스럽지만 실무적인 밀도가 느껴져야 함
- 호텔 브랜드의 품격과 운영 시스템의 기능성이 동시에 느껴져야 함

화면 구성:
1. 상단 헤더
- 좌측: 호텔 또는 서비스 브랜드명 영역
- 중앙 또는 좌측: 페이지 타이틀 “Dashboard” 또는 “Operations Dashboard”
- 우측: 호텔 선택 드롭다운, 알림 아이콘, 사용자 프로필, 로그아웃 또는 계정 메뉴
- 헤더는 얇고 정돈된 스타일, 과하게 두껍지 않게

2. 좌측 사이드바
- 메뉴 항목:
  - Dashboard
  - Schedule
  - Employees
  - Leave Management
  - Demand Input
  - Hotel Settings
- 현재 페이지(Dashboard)는 선택 상태가 명확히 보이게
- 아이콘은 간결하고 고급스럽게
- 사이드바는 너무 넓지 않게, 프리미엄 관리자툴 느낌으로 절제되게

3. 상단 핵심 KPI 카드 영역
다음 카드들을 가로로 배치하되, 카드 수와 레이아웃은 보기 좋게 정리
- 이번 달 스케줄 상태
- 미확정 항목 수
- 락 예정 항목 수
- 전체 직원 수
- 피크 수요 예상일 수 또는 리스크 날짜 수
카드 디자인 조건:
- 단순하고 고급스럽게
- 숫자가 명확히 잘 보이게
- 카드마다 작은 상태 배지 또는 보조 텍스트 포함
- 너무 화려한 차트보다 요약 정보 중심

4. 메인 본문 1행
좌측 큰 카드:
- “이번 달 스케줄 상태” 또는 “Current Schedule Status”
- 현재 월, 현재 버전, 상태 배지(초안/확정/락), 마지막 수정 시간
- 주요 액션 버튼:
  - 스케줄 보기
  - 새로 생성
  - 다운로드
  - 확정하기
- 스케줄 생성 관련 빠른 진입 버튼 포함

우측 카드:
- “다음 달 생성 바로가기”
- 다음 달 기준 예상 준비 상태
- 생성 시작 버튼
- 최근 생성 히스토리 또는 마지막 생성 결과 요약

5. 메인 본문 2행
좌측 카드:
- “공정성 리포트 요약”
- 직원별 야간·주말 편차 요약 또는 작은 요약 차트/바
- 너무 복잡한 데이터 시각화보다 관리자에게 빠르게 이해되는 요약 중심

중앙 또는 우측 카드:
- “최근 변경 이력”
- 최근 누가 무엇을 수정했는지 3~5개 리스트
- 버전 생성, AI 조정, 수동 셀 편집, 확정 같은 이벤트가 보이게

또 다른 카드:
- “AI 조정 기록” 또는 “AI Schedule Adjustments”
- 최근 자연어 지시 예시와 반영 결과 요약
- 너무 기술적인 개발자 화면처럼 보이지 않게
- 관리자 입장에서 이해하기 쉬운 문장 구조

6. 경고/리스크 영역
- 인력 부족 위험일
- 확정되지 않은 일정
- 휴무 요청 미처리 건수
- 락 예정 항목 경고
이 영역은 경고 배너나 카드로 보이게 하되, 과하게 빨갛고 위협적인 느낌보다 세련된 주의 강조 방식으로 표현

7. 빠른 액션 영역
- 직원 추가
- 휴무 등록
- 수요 입력
- 근무표 바로 열기
- 호텔 설정 열기
이 버튼/카드는 실무적으로 자주 쓰는 액션 허브처럼 보여야 함

UX 방향:
- 관리자 입장에서 “오늘 할 일”이 바로 보여야 함
- 한 번에 너무 많은 정보가 몰리지 않게 정보 계층을 명확히
- 각 카드의 우선순위가 시선 흐름에 맞게 설계되어야 함
- 대시보드는 허브 화면이므로 각 카드가 실제 작업 페이지로 이동할 것 같은 구조여야 함
- 표보다는 카드 중심이지만, 운영툴의 정보 밀도는 유지
- 공정성 리포트, 변경 이력, 리스크는 요약형으로 보여주고 상세는 하위 페이지로 들어갈 것 같은 구조로 설계

시각적 스타일 세부 조건:
- 배경은 웜 화이트 또는 매우 밝은 고급 뉴트럴 톤
- 주요 네비게이션/헤더/강조 영역에 딥 네이비 사용
- 버튼이나 강조 배지에 절제된 골드 포인트 사용
- 차콜/네이비/웜 화이트 중심에 포인트 컬러만 최소 사용
- 카드 테두리는 얇고 정제되게
- 그림자는 매우 은은하게
- 과도한 그라데이션, 유리질감, 네온톤, 캐주얼 아이콘 금지
- 타이포그래피는 세련되고 안정적이어야 함
- 전체적으로 조용하지만 고급스럽게

텍스트/브랜드 처리:
- 실제 호텔명은 샘플 placeholder로 처리해도 괜찮음
- 너무 구체적인 가짜 브랜드 스토리보다 중립적인 관리자 시스템 텍스트 사용
- 영어 기반 UI로 만들어도 좋고, 일부 부제/보조 문구는 자연스러운 관리자용 문장으로 구성
- 지나치게 마케팅 문구처럼 과장하지 말고, 운영 시스템다운 신뢰감 있는 문구 사용

반드시 포함할 요소:
- 대시보드 메인 제목
- KPI 카드들
- 사이드바 메뉴
- 스케줄 상태 카드
- 다음 달 생성 바로가기
- 최근 변경 이력
- 공정성 리포트 요약
- AI 조정 기록 요약
- 경고/리스크 영역
- 빠른 액션 영역

금지사항:
- 예약 사이트처럼 객실 사진, 여행 감성, 프로모션 배너 중심으로 보이지 않게
- 소비자 대상 마이페이지처럼 보이지 않게
- 지나치게 화려한 호텔 브랜딩 쇼케이스처럼 보이지 않게
- 일반 스타트업 프로젝트 관리 툴처럼 가볍게 보이지 않게
- 차트 남발 금지
- 정보 밀도가 너무 낮아서 빈약해 보이지 않게

최종 결과물 목표:
- 로그인 후 바로 신뢰감이 드는 호텔 운영 관리자 대시보드
- 프리미엄 호텔 브랜드의 품격이 느껴지지만 실제 업무 효율이 높은 구조
- 이 화면을 기준으로 이후 Schedule, Employees, Demand, Settings 페이지까지 확장 가능한 디자인 시스템 느낌으로 설계
